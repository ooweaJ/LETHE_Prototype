# LETHE AI 알파테스트 시스템 v0.1

이 패키지는 `LETHE: 망각의 군주`의 코어 시스템을 빠르게 검증하기 위한 **봇 기반 알파테스트 시뮬레이터**입니다.
실제 HTML 플레이를 렌더링하는 도구가 아니라, 기획서의 핵심 규칙을 수치 모델로 옮겨 수백~수천 판을 빠르게 돌립니다.

검증 목적은 하나입니다.

> 망각이 “아쉬움”으로 작동하는가, 아니면 “짜증/압수감/무덤덤”으로 무너지는가?

단, `Q1/Q2 감정값`은 실제 인간 감정이 아니라 봇 프록시입니다. 이 시스템은 사람 테스트 전에 밸런스와 구조 위험을 걸러내는 1차 필터입니다.

---

## 포함된 검증 항목

- 무기 2종: `절단쌍검`, `장송대검`
- 기억 6종: `처형자의 섬광`, `굶주린 칼무리`, `추적자의 맹세`, `파쇄의 파문`, `피의 반사`, `멈춘 초침`
- 기억 슬롯 3개
- 기억 시너지 8개
- 의존도 기반 망각
- 망각 잔향 / 스탯 메아리
- 삭제 전 예측 일치율
- 삭제 직후 Q1/Q2 감정 프록시
- 망각 직후 전투력 딥
- 새 기억 획득 후 회복률
- 몰빵 / 분산-거미줄 / 분산-느슨 빌드 분류
- 삭제 기억 쏠림, 빌드 수렴, 잔향 수렴 위험

---

## 실행 방법

Node.js 18 이상이 필요합니다. 외부 라이브러리는 필요 없습니다.

```bash
npm run alpha
```

빠른 확인:

```bash
npm run alpha:quick
```

파라미터 직접 지정:

```bash
node src/run_alpha.js --runs 1000 --stages 2 --seed 20260601 --echo 0.50 --ui 0.78 --out outputs/my_test
```

스윕 테스트:

```bash
npm run sweep
```

---

## 주요 파라미터

| 파라미터 | 의미 | 추천값 |
|---|---|---:|
| `--runs` | 가상 플레이 수 | 1000 이상 |
| `--stages` | 한 런의 망각 반복 수 | 프로토타입 1~2, 풀게임 가정 3 |
| `--echo` | 잔향 강도 | 0.50~0.70 |
| `--ui` | 선명도/예고 UI 가시성 | 0.78 기본값 |
| `--bots` | 특정 봇만 실행 | 예: `focus_burst,balanced_web` |

예시:

```bash
node src/run_alpha.js --runs 2000 --stages 3 --echo 0.50 --ui 0.78 --bots focus_burst,balanced_web,echo_pivot --out outputs/tuning_050
```

## Go/No-Go 기준

기본 기준은 현재 검증 설계 문서의 합격선을 반영했습니다.

| 지표 | 목표 |
|---|---:|
| 아쉬움 분면 | 60% 이상 |
| 짜증 분면 | 20% 미만 |
| 무덤덤 분면 | 25% 미만 |
| 예측 일치율 | 60~65% 이상 |
| 삭제 직후 즉시 종료율 | 15% 미만 |
| 런 재시작 의향 | 60% 이상 |
| 첫 망각 전 사용 시간 | 8~10분 |
| 망각 직후 전투력 딥 | 30~50% |
| 새 기억 후 회복률 | 85~110% |
| 특정 빌드 분류 쏠림 | 80% 이하 |
| 특정 기억 삭제 쏠림 | 35% 이하 |

## 봇 프로필

| 봇 | 목적 |
|---|---|
| `random` | 기준선/노이즈 확인 |
| `survivor` | 생존·제어 중심 밸런스 확인 |
| `focus_burst` | 몰빵 버스트 처벌/보상 확인 |
| `melee_onhit` | 쌍검 온힛 조합 확인 |
| `balanced_web` | 분산-거미줄 시너지 함정 확인 |
| `memory_protector` | 기억을 지키려는 플레이가 가능한지 확인 |
| `echo_pivot` | 잔향 기반 피벗이 실제로 작동하는지 확인 |
| `optimizer` | 최적화/악용 가능성 탐지 |

## 결과 해석법

- `GO_CANDIDATE`: 사람 테스트로 넘길 수 있는 상태입니다.
- `ITERATE`: 큰 방향은 가능하지만 특정 튜닝이 필요합니다.
- `NO_GO_FIX_CORE`: 콘텐츠 추가 전에 의존도/잔향/망각 빈도부터 고쳐야 합니다.

주의할 점:

- 이 시스템이 “게임이 재밌다”를 최종 판정하지는 못합니다.
- 이 시스템은 “사람에게 보여주기 전에 명백한 밸런스/구조 결함이 있는가”를 찾는 도구입니다.
- 최종 감정 검증은 반드시 8~12명 플레이테스트로 확인해야 합니다.
