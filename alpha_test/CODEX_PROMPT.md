# Codex에 그대로 붙여넣을 요청문

아래 작업을 현재 LETHE HTML 프로토타입 프로젝트에 통합해줘.
목표는 사람이 플레이하기 전, AI/봇이 수백~수천 판을 자동으로 돌려서 밸런스와 재미 위험 신호를 뽑는 `AI 알파테스트 시스템`을 만드는 것이다.

## 핵심 목적

LETHE의 코어 질문은 “망각됐을 때 아쉬운가, 짜증나는가”다.
따라서 자동 테스트는 다음을 측정해야 한다.

1. 의존도 기반으로 어떤 기억이 삭제되는가
2. 봇이 삭제될 기억을 예측할 수 있는가
3. 삭제된 기억이 봇이 가장 지키고 싶어하던 기억인가
4. 망각 직후 전투력이 얼마나 떨어지는가
5. 잔향과 새 기억으로 얼마나 회복되는가
6. 몰빵 / 분산-거미줄 / 분산-느슨 빌드가 실제로 갈리는가
7. 특정 기억/잔향/빌드로 수렴하지 않는가
8. 삭제 후 즉시 종료 위험, 재시작 의향 프록시가 어떤가

## 구현 요구

프로젝트에 다음 구조를 추가해줘.

```text
alpha_test/
  package.json 또는 기존 package.json script 추가
  src/
    config.js              # 무기, 기억, 잔향, 시너지, 목표치
    rng.js                 # seed 고정 가능한 RNG
    bots.js                # 봇 프로필
    simulator.js           # 전투/망각/잔향/피벗 시뮬레이션
    metrics.js             # Go/No-Go 지표 계산
    report.js              # markdown/html 리포트 생성
    run_alpha.js           # CLI 실행기
    sweep.js               # echo/ui 파라미터 스윕
  outputs/
```

기본 명령어:

```bash
node alpha_test/src/run_alpha.js --runs 1000 --stages 2 --seed 20260601 --echo 0.60 --ui 0.62 --out alpha_test/outputs/default
node alpha_test/src/sweep.js --runs 300 --stages 2 --echo 0.40,0.50,0.60,0.70,0.80 --ui 0.45,0.62,0.78 --out alpha_test/outputs/sweep
```

## 게임 데이터

무기 2종:

- 절단쌍검: 빠른 근접/온힛. 공격속도, 추가타, 치명과 친화.
- 장송대검: 느린 강타/폭딜. 치명, 범위, 버스트와 친화.

기억 6종:

1. 처형자의 섬광: 버스트. 능동=주기적 강타. 잔향=치명타 확률/피해.
2. 굶주린 칼무리: 근접 도트. 능동=회전 칼날/오라. 잔향=공격속도/지속피해.
3. 추적자의 맹세: 추적 다중. 능동=유도 투사체. 잔향=투사체 수/유도력.
4. 파쇄의 파문: 광역/넉백. 능동=원형 충격파. 잔향=범위/넉백/생존.
5. 피의 반사: 온힛 증폭. 능동=추가타. 잔향=추가타 확률/온힛 피해.
6. 멈춘 초침: 제어. 능동=둔화/시간 균열. 잔향=쿨다운 감소/둔화 지속.

기억 슬롯은 최대 3개다.
삭제된 기억은 해당 런에서 다시 획득하지 않도록 한다.
망각된 기억은 능동 효과를 100% 잃고, 잔향 스탯만 남긴다.
잔향 강도 기본값은 잃은 능력의 60%로 시작하되, CLI에서 `--echo`로 바꿀 수 있어야 한다.

## 의존도 계산

플레이어에게 노출되는 규칙은 단순히 “레테는 네가 가장 의존한 기억을 가져간다”여야 한다.
내부 공식은 아래 비율로 시작한다.

```text
의존도 = 전투 기여 40% + 보스전 흔적 25% + 대체불가능성 20% + 존재감/발동 시간 15%
```

이 공식은 metrics에서 예측 일치율이 낮으면 바꿀 수 있도록 상수화한다.

## 봇 프로필

최소 8종을 구현한다.

- random: 기준선
- survivor: 생존/제어 우선
- focus_burst: 장송대검+처형자의 섬광 몰빵
- melee_onhit: 절단쌍검+굶주린 칼무리+피의 반사
- balanced_web: 시너지 연결이 촘촘한 분산 빌드
- memory_protector: 가장 지키고 싶은 기억을 잃지 않으려 분산
- echo_pivot: 삭제 후 잔향과 맞는 기억을 선택
- optimizer: 수치상 최적 선택만 해서 악용 가능성 탐지

## 산출물

실행 후 아래 파일을 만들어야 한다.

```text
outputs/default/summary.json
outputs/default/alpha_summary.md
outputs/default/alpha_report.html
outputs/default/runs.csv
outputs/default/raw_runs.json
```

`alpha_summary.md`에는 다음을 반드시 포함한다.

- verdict: GO_CANDIDATE / ITERATE / NO_GO_FIX_CORE
- Alpha Fun Score 0~1
- 아쉬움 분면 비율
- 짜증 분면 비율
- 무덤덤 분면 비율
- 예측 일치율
- 삭제 직후 즉시 종료율
- 런 재시작 의향
- 망각 직후 전투력 딥
- 새 기억 후 회복률
- 첫 망각 전 사용 시간
- 빌드 분류 분포
- 삭제된 기억 분포
- 봇별 결과
- 경고
- 튜닝 제안

## Go/No-Go 기준

하드 게이트:

- 아쉬움 분면 >= 60%
- 짜증 분면 < 20%
- 예측 일치율 >= 60~65%
- 삭제 직후 즉시 종료율 < 15%

소프트 게이트:

- 무덤덤 분면 < 25%
- 런 재시작 의향 >= 60%
- 첫 망각 전 사용 시간 8~10분
- 망각 직후 전투력 딥 30~50%
- 새 기억 후 회복률 85~110%
- 특정 빌드 분류 80% 초과 금지
- 특정 기억 삭제율 35% 초과 금지

## 중요한 제약

- 외부 의존성을 최소화해라. 가능하면 Node.js 표준 라이브러리만 써라.
- seed를 고정하면 같은 결과가 나와야 한다.
- 현재 HTML 게임 플레이와 충돌하지 않게 별도 폴더로 넣어라.
- 실제 인간 감정을 AI가 완전히 대체한다고 주장하지 말고, Q1/Q2는 “감정 프록시”임을 리포트에 명시해라.
- 리포트는 개발자가 바로 튜닝할 수 있게 “불합격 시 어떤 레버를 조정할지”를 제안해야 한다.

완료 후 기본 명령어를 실행해서 샘플 리포트를 생성하고, `summary.json`의 주요 값을 콘솔에 요약 출력해줘.
