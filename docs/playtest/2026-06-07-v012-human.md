# LETHE v0.12 Human Playtest Sheet

Purpose: test whether the v0.12 automated balance baseline produces a fun early run, meaningful memory choices, and a regrettable rather than irritating forgetting loop.

Automated baseline before this sheet:

- Full `qa:balance`: `GO_BALANCE_BASELINE`.
- First boss clear: `80%`.
- Full clear: `60%`.
- Death: `40%`.
- First boss TTK median: `25.79s`.

## Setup

- Build: LETHE HTML alpha v0.12.
- Package command: `npm run playtest:package`.
- Open: `dist/lethe-v0.12-playtest/index.html`.
- Testers: 5-8 controlled human sessions.
- Session length: 12-18 minutes, or one complete run if shorter.
- Required path: first boss, first forgetting, 2-memory deficit segment, and first memory refill.

## Stop Rules

- If most testers are bored before the first boss, revise early pressure/growth before adding content.
- If most testers click level-up choices randomly, revise choice readability before more tests.
- If two or more testers call forgetting unfair, revise prediction/readability before more tests.
- If two or more testers call the deficit segment impossible, soften post-loss pressure after human evidence.
- If loss feels like no big deal, inspect refill/echo visibility before adding systems.
