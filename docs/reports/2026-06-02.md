## 결정 사항

- GPT/Claude에게 보내는 프롬프트는 기본적으로 한국어로 작성한다.
- 답변 형식에는 반드시 `앞으로 해야 할 일` 섹션을 넣는다.
- Discord 보고는 긴 원문 대신 `작업`, `완료`, `문제`, `기획질문`만 짧게 보여준다.
- GPT는 구현 우선순위와 테스트 기준 검토에 사용한다.
- Claude는 감정선, 문구, 플레이어 경험 검토에 사용한다.
- Claude 자동 검토 때는 `--tools ""`로 도구를 비활성화해 파일을 직접 수정하지 못하게 한다.
- Claude 답변 원문은 Discord가 아니라 `docs/review_responses/`에 저장한다.

## 문제와 리스크

- 예전 문서 일부에는 깨진 한국어 인코딩이 남아 있다.
- 앞으로 새 문서는 깨끗한 UTF-8 한국어로 작성해야 한다.
- Discord는 권한 승인을 대신 누를 수 없고, Codex UI 승인이 필요하면 작업이 멈춘다.
- Claude Code 인증이나 네트워크 문제가 있으면 자동 검토가 멈출 수 있다.
- PowerShell `Get-Content` 출력에서는 Claude 응답 한국어가 깨져 보일 수 있지만, Node로 읽으면 UTF-8 파일은 정상이다.
- 아직 v0.2 구현은 시작하지 않았다.

## 기획 검토 요약

- GPT verdict: `ITERATE_BEFORE_TEST`.
- Claude verdict: `ITERATE_BEFORE_TEST`.
- v0.1은 바로 사람 테스트로 보내기보다 좁은 v0.2 튜닝 후 테스트하는 것이 좋다.
- 첫 망각 타이밍을 8-10분, 가능하면 8.0-9.5분 범위로 늦춘다.
- `처형자의 섬광` 삭제율을 25-35%로 낮춘다.
- JSON 로그에 선택 기억, 예측 기억, 실제 삭제 기억, 삭제 weight 또는 score를 남긴다.
- `echo=0.50`과 기존 `echo=0.60`을 비교 가능하게 로그를 남긴다.
- 망각 후 피드백 화면은 “왜 잃었고 무엇이 남았는지” 쉽게 이해되게 만든다.
- 사람 테스트에는 사라진 기억 이름을 말할 수 있는지 확인하는 자유응답을 추가한다.
