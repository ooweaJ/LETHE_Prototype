4. 결정 사항

  • v0.8 Gate C는 당장 다음 제품 단계가 아니다.
  • 다음 제품 단계는 v0.9 release-feel HTML prototype이다.
  • 사람 테스트는 빌드 정체성, 압박 고저차, post-loss challenge, 자동전투 안의 작은 전술 선택이 보강된 뒤 요청한다.
  • 밤샘 루프의 기본값은 외부 AI가 파일을 직접 수정하지 않는 planning/verification loop로 둔다.
  • 구현 명령은 신뢰된 로컬에서 명시적으로 넘길 때만 실행한다.
  • Claude와 Codex CLI 모두 BUILD_V09_RELEASE_FEEL_LOOP를 선택했다.
  • 다음 구현 범위는 v0.9 Work Package 1로 제한한다: 기존 6개 기억의 빌드 정체성과 시너지 체감을 90초 안에 읽히게 만든다.

5. 문제 또는 리스크

  • 완전한 무인 구현 루프는 아직 위험하다. 외부 모델이 파일을 직접 수정하려면 명시적인 --implement-cmd가 필요하다.
  • 루프는 preflight와 clean tree를 기본으로 요구하므로, 작업 중 검증은 --allow-dirty가 필요하다.
  • 레퍼런스 조사는 방향을 제공하지만, LETHE의 재미는 실제 브라우저/사람 플레이로 다시 확인해야 한다.

6. GPT/Claude 인계 요약
