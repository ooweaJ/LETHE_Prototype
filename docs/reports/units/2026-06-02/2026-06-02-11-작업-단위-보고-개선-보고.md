# 2026-06-02-11 - 작업 단위 보고 개선 보고

## 1. 현재 빌드 상태

- LETHE HTML alpha는 v0.5 core-fun human-test candidate 상태를 유지한다.
- 보고서 원본은 계속 날짜 단위 Markdown/HTML로 보관한다.
- Discord 전송은 이제 일일 보고와 작업 단위 보고를 구분할 수 있다.

## 2. 오늘 바뀐 것

- `scripts/send_discord_report.js`에 `--section` 옵션을 추가했다.
- `scripts/send_discord_report.js`에 `--latest-section` 옵션을 추가했다.
- `package.json`에 `report:discord:unit`과 `report:discord:unit:dry`를 추가했다.
- `docs/DISCORD_REPORTING.md`와 `docs/CODEX_RUNBOOK.md`에 날짜 단위 보관과 작업 단위 전송의 차이를 정리했다.
- `scripts/ask_claude_review.js`가 Claude 401 인증 오류 때 로컬 로그인 후 재시도하라고 안내하도록 바꿨다.

## 3. 테스트 결과와 근거

- `node --check scripts/send_discord_report.js`: 통과.
- `node --check scripts/ask_claude_review.js`: 통과.
- `package.json` JSON parse 검사: 통과.
- `node scripts/send_discord_report.js docs/reports/2026-06-02.md --dry-run --section "v0.5 초반 재미 보완 보고"`: v0.5 섹션만 작업 보고로 요약됨.

## 4. 결정 사항

- 큰 버전업, AI 테스트 milestone, 자동화 변경처럼 보고할 만한 작업은 `report:discord:unit`으로 따로 보낸다.
- 날짜 단위 보고서는 전체 기록용으로 유지한다.
- Claude 401 오류는 스크립트 실패로만 남기지 않고 사용자에게 인증 조치를 안내한다.

## 5. 문제 또는 리스크

- 실제 Discord 전송은 외부 webhook 전송이라 이 세션에서는 차단될 수 있다.
- 작업 단위 보고는 Markdown의 `# 제목` 섹션 기준이므로, 큰 작업은 보고서에 별도 최상위 제목으로 남겨야 한다.

## 6. GPT/Claude 인계 요약

- 기획 검토 자동화는 Claude Code 우선, Codex CLI fallback 순서다.
- Claude 인증 실패 시 로컬 터미널에서 `claude` 로그인/인증 후 `npm run review:claude`를 다시 실행한다.

## 7. 다음 Codex 작업

- 다음 큰 작업 완료 시 `npm run report:discord:unit:dry`로 먼저 메시지를 확인한다.
- 문제가 없으면 로컬에서 `npm run report:discord:unit`으로 작업 단위 보고를 보낸다.

## 8. 포트폴리오 메모

- 문제: 하루 보고서 안에 큰 작업 여러 개가 쌓이면 Discord 알림이 작업 단위를 제대로 전달하지 못했다.
- 방향: 기록은 날짜 단위로 유지하되, 공유는 작업 단위로 잘라 보낼 수 있게 한다.
- 실행: Discord 보고 스크립트에 섹션 선택 기능과 작업 단위 npm 명령을 추가했다.
- 결과: v0.5 같은 큰 버전업을 별도 작업 보고로 보낼 수 있게 됐다.
