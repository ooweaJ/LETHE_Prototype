2026-06-02-25 - v0.7 밸런스 판정 실패 정정 보고

1. 현재 빌드 상태

  • v0.7의 밸런스 판정은 사용자 실제 플레이 피드백으로 무효화됐다.
  • 현재 상태는 GO_TO_SOLO_PLAYTEST가 아니라 FIX_BALANCE_MODEL_FIRST에 가깝다.
  • 다음 목표는 수치 보정 전에 자동 테스트가 실제 브라우저 전투 체감과 왜 어긋났는지 고치는 것이다.

2. 오늘 바뀐 것

  • v0.7 Claude 피드백이 실전 밸런스 피드백이 아니라 AI proxy 기반 판단이었다는 점을 정정했다.
  • AI 실패 사례에 v0.7 balance proxy 실패를 추가했다.
  • v0.7.1 현실 검증 프롬프트를 추가했다: docs/review_prompts/2026-06-02-v071-balance-reality-check.md.
  • docs/NEXT_TASKS.md와 docs/CODEX_STATUS.md를 사용자 직접 피드백 기준으로 갱신했다.

3. 테스트 결과와 근거

  • 사용자 직접 피드백: “밸런스가 1도 안 맞는다.”
  • 코드 확인 결과 실제 브라우저 결손 구간은 기억 2개일 때 스폰 간격이 더 짧아지고 pack size가 3으로 유지된다.
  • 기존 AI 시뮬레이터는 이 실제 전투 압박을 추상 power ratio로만 처리했다.
  • 기존 브라우저 QA는 boss -> forgetting -> deficit -> refill 흐름 완료를 확인했지만, 실제 전투 밸런스를 플레이하지 않았다.

4. 결정 사항

  • AI proxy 통과를 밸런스 통과로 보고하지 않는다.
  • Claude에게 전달할 다음 질문은 “v0.7이 괜찮은가”가 아니라 “왜 AI 지표가 실제 플레이를 못 잡았는가”다.
  • v0.7.1은 수치부터 만지기보다 결손 구간 압박, 적 HP/피해, 무기 역할, 위기 탈출 수단, 실제 브라우저 밸런스 QA를 함께 본다.

5. 문제 또는 리스크

  • 지금 수치를 감으로 또 바꾸면 같은 실패가 반복된다.
  • 자동화가 신뢰를 얻으려면 실제 브라우저 전투를 어느 정도라도 플레이하는 balance QA가 필요하다.
  • Claude/GPT도 잘못된 입력을 받으면 그럴듯한 잘못된 결론을 낸다.

6. GPT/Claude 인계 요약

  • v0.7.1 질문은 docs/review_prompts/2026-06-02-v071-balance-reality-check.md를 사용한다.
  • 사용자 직접 플레이 피드백을 최상위 근거로 둔다.
  • AI 지표는 참고 자료일 뿐, 결론 근거로 과대표현하지 않는다.

7. 다음 Codex 작업

  • 실제 브라우저 전투 압박을 잡는 balance QA 또는 보정 지표를 추가한다.
  • 그 결과와 사용자 피드백을 함께 Claude/GPT에 보낸다.
  • v0.7.1에서 결손 구간 스폰/적 스펙/무기 역할을 좁게 수정한다.

8. 포트폴리오 메모

  • 문제: 자동화 루프가 AI 지표를 실제 밸런스 검증처럼 착각했다.
  • 방향: 실패를 숨기지 않고, AI proxy와 human signal을 분리한다.
  • 실행: 실패 사례, 현실 검증 프롬프트, 다음 작업 목록을 갱신했다.
  • 결과: LETHE 자동화는 “AI가 맞다고 했다”가 아니라 “AI가 틀린 지점을 발견하고 테스트 설계를 고친다”는 방향으로 보정된다.
