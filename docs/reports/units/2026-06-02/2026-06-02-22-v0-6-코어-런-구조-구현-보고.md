# 2026-06-02-22 - v0.6 코어 런 구조 구현 보고

## 1. 현재 빌드 상태

- LETHE HTML alpha는 v0.6 core run-structure solo-test candidate 상태다.
- v0.5의 9분 첫 망각 구조를 버리고, 4분 첫 보스/첫 망각으로 LETHE의 고유 훅을 앞당겼다.
- 넓은 사람 테스트는 아직 보류하지만, 사용자 1인 체감 테스트 후보까지는 올라왔다.

## 2. 오늘 바뀐 것

- 브라우저 표기와 experiment version을 `v0.6`으로 변경했다.
- 총 런 구조를 20분 기준으로 바꿨다.
- 보스 스케줄을 `4 / 8 / 12 / 16 / 20분`으로 설정했다.
- 보스 처치 후 의존도 기반으로 기억 1개를 잃고, 즉시 결과 종료가 아니라 기억 2개 결손 생존 구간에 들어가게 했다.
- 결손 생존 구간 이후 새 기억 후보 3개 중 1개를 선택해 다시 3개 슬롯으로 복귀하게 했다.
- `runTimeline` JSON payload에 boss schedule, cycle, forgotten history, refill choices를 추가했다.
- AI 테스트에 v0.6 cycle 지표를 추가했다:
  - first cycle completion,
  - two-memory survival,
  - echo pivot score,
  - refill reached rate.
- `?qa=fast,v06` 브라우저 QA 모드를 추가했다.
- v0.6 결과 판단 프롬프트 `docs/review_prompts/2026-06-02-v06-cycle-eval.md`를 작성했다.

## 4. 결정 사항

- v0.6은 자동 지표 기준으로는 `GO_TO_SOLO_PLAYTEST_CANDIDATE`로 둔다.
- Claude v0.6 피드백은 현재 Codex 세션에서 외부 전송 정책으로 차단되어 받지 못했다.
- Claude 피드백이 필요하면 사용자의 신뢰된 로컬 터미널에서 같은 프롬프트를 실행한다.
- Claude 없이 진행한다면 다음 단계는 1인 체감 테스트다.

## 5. 문제 또는 리스크

- 실제 플레이 감각은 아직 확인하지 않았다.
- quick 기준 prediction match가 `92.5%`로 높아, 예측이 너무 정답 맞히기처럼 느껴질 수 있다.
- 결손 생존 105초가 “다른 모드로 버틴다”인지 “그냥 기다린다”인지는 실제 체감이 필요하다.
- 잔향은 기존 echo 기반이므로, 기억별 변형이 눈에 충분히 띄는지는 더 확인해야 한다.
- Claude 피드백 자동 루프는 이 세션에서 외부 전송 정책에 막혔다.
