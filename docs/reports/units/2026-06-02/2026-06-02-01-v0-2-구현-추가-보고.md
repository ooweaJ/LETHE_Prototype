# 2026-06-02-01 - v0.2 구현 추가 보고

## 1. 현재 빌드 상태

- LETHE HTML alpha는 v0.2 narrow tuning 후보 상태다.
- AI 1000런 기준 verdict는 `GO_CANDIDATE`다.
- 사람 테스트 전에 브라우저 화면 QA만 한 번 더 보면 된다.

## 2. 오늘 바뀐 것

- 첫 망각 타이밍을 평균 9분으로 늦췄다.
- 기본 실험값을 `echo=0.50`, `ui=0.62`로 바꿨다.
- `처형자의 섬광` 삭제 쏠림을 낮추고 삭제 weight를 로그에 남기게 했다.
- 망각 결과 화면에 사라진 기억, 예측 결과, 삭제 weight, 잔향, 다음 방향을 한 번에 보여주게 했다.
- Q3 자유응답으로 “방금 사라진 기억 이름을 떠올릴 수 있는지”를 추가했다.
- AI 테스트 기준에 예측 상한, 삭제 분포, 회복률 기준을 추가했다.

## 3. 테스트 결과와 근거

- `node --check src/game.js`: 통과.
- `node --check alpha_test/src/config.js`: 통과.

## 4. 결정 사항

- 새 기억, 새 보스, 상점, 메타 성장 같은 확장은 아직 하지 않는다.
- v0.2는 “더 만들기”가 아니라 “망각이 이해 가능한 상실인지 확인하는 버전”으로 둔다.
- `echo=0.50`, `ui=0.62`를 기본값으로 유지한다.

## 5. 문제 또는 리스크

- 망각 직후 전투력 딥은 `29.6%`라 목표 `30%`보다 아주 조금 낮다.
- sweep에서는 `ui=0.78` 조합이 더 높은 점수를 냈다. 다만 이번 구현 범위에서는 기획 답변의 기본값을 유지했다.
- 아직 실제 브라우저 화면 QA는 하지 않았다.

## 6. GPT/Claude 인계 요약

- GPT와 Claude 모두 `ITERATE_BEFORE_TEST`를 권장했다.
- Codex는 그 답변을 기준으로 타이밍, 삭제 분포, 피드백, 로그, 자유응답 문항만 구현했다.
- 추가 기획 질문은 지금 당장 필요하지 않다.

## 7. 다음 Codex 작업

- 브라우저에서 v0.2 결과 화면을 직접 확인한다.
- 화면 QA가 통과하면 5-8명용 human playtest 가이드를 만든다.
- 사람 테스트에서는 “아까움 / 짜증 / 무감정” 중 어떤 반응인지와 그 이유를 가장 중요하게 본다.

## 8. 포트폴리오 메모

- 문제: v0.1은 감정 루프는 좋았지만 첫 망각이 빠르고 삭제 결과 해석이 어려웠다.
- 방향: 기능 추가 없이 테스트 결과를 해석 가능한 상태로 보정했다.
- 실행: 타이밍, 삭제 weight, 결과 피드백, JSON 로그, 자유응답 문항을 좁게 구현했다.
- 결과: AI 기준 human test 후보까지 올라왔고, 다음 단계는 실제 플레이어 감정 반응 검증이다.
