# 2026-06-02-20 - 런 구조 재설계 프롬프트 보고

## 1. 현재 빌드 상태

- LETHE HTML alpha v0.5는 더 이상 그대로 사람 테스트로 보내지 않는다.
- 사용자의 직접 플레이 결과, 보스와 첫 망각까지 가는 시간이 너무 멀어 현재 테스트는 의미가 약하다고 판단했다.
- 다음 gate는 사람 테스트가 아니라 v0.6 런 구조 재설계 판단이다.

## 2. 오늘 바뀐 것

- `docs/review_prompts/2026-06-02-run-structure-redesign.md`를 추가했다.
- 프롬프트는 25분/5분 보스 주기안과 20분/큰 망각+마지막 각인안을 비교하도록 구성했다.
- `docs/NEXT_TASKS.md`를 “사람 테스트 진행”에서 “런 구조 재설계 판단”으로 바꿨다.
- `docs/CODEX_STATUS.md`에 사람 테스트 보류와 새 기획 프롬프트 경로를 기록했다.

## 3. 테스트 결과와 근거

- 이번 작업은 문서/기획 인계 작업이라 게임 로직 테스트는 새로 실행하지 않았다.
- 근거는 사용자의 직접 플레이 피드백이다:
  - 보스까지 너무 멀다.
  - 기억 상실 감정 이전에 게임이 먼저 재미있어야 한다.
  - 망각은 런 후반 이벤트가 아니라 반복되는 핵심 리듬이어야 한다.

## 4. 결정 사항

- v0.5 사람 테스트는 보류한다.
- Claude/GPT에게 먼저 20-25분 런 구조, 보스 주기, 기억 상실 후 치환 보상, 기억 보충 타이밍을 묻는다.
- 답변을 받은 뒤 Codex가 v0.6 작업 목록으로 바꾸고 구현한다.

## 5. 문제 또는 리스크

- 기존 AI 지표는 9분 첫 망각 기준이라 새 5분/반복 보스 구조에는 그대로 적용할 수 없다.
- 잦은 기억 상실은 LETHE다움을 강화할 수 있지만, 너무 자주 일어나면 짜증으로 변할 수 있다.
- 마지막 기억 각인은 매력적이지만 HTML 프로토타입에서 지금 테스트할지, Unity 이후로 넘길지 결정이 필요하다.

## 6. GPT/Claude 인계 요약

- 새 프롬프트의 핵심 질문은 “LETHE의 다음 HTML 프로토타입은 어떤 런 구조로 가야 하는가”다.
- 답변은 `FIX_CORE_RUN_STRUCTURE`, `ITERATE_V05_SMALL`, `GO_TO_HUMAN_TEST` 중 하나를 골라야 한다.
- Codex가 바로 구현할 수 있게 v0.6 작업 목록과 테스트 기준을 요구했다.

## 7. 다음 Codex 작업

- 사용자가 `docs/review_prompts/2026-06-02-run-structure-redesign.md`를 Claude/GPT에 직접 보낸다.
- 답변이 오면 `docs/review_responses/`에 저장한다.
- Codex는 답변을 읽고 `docs/NEXT_TASKS.md`에 v0.6 구현 작업으로 변환한다.

## 8. 포트폴리오 메모

- 문제: AI 지표는 통과했지만 실제 플레이에서는 핵심 이벤트가 늦게 열려 테스트 가치가 낮았다.
- 방향: 사람 테스트를 강행하지 않고, 직접 플레이 피드백으로 기획 gate를 되돌린다.
- 실행: 런 구조 후보안을 정리하고 Claude/GPT 판단용 프롬프트로 문서화했다.
- 결과: LETHE는 이제 “망각 감정 튜닝”보다 먼저 “반복되는 상실/치환/생존 리듬”을 검증하는 v0.6 단계로 이동한다.
