2026-06-02-10 - 자동화 정리 보고

1. 현재 빌드 상태

  • LETHE HTML alpha는 v0.5 core-fun human-test candidate 상태를 유지한다.
  • 자동 기획 검토 경로는 Claude Code → Codex CLI fallback 두 단계로 정리했다.
  • OpenAI API fallback은 제거했다.

2. 오늘 바뀐 것

  • scripts/ask_openai_review.js를 삭제했다.
  • package.json에서 review:openai, review:openai:dry를 제거했다.
  • docs/CODEX_RUNBOOK.md, docs/review_prompts/README.md, docs/CODEX_STATUS.md에서 OpenAI API fallback 안내를 제거했다.
  • Claude 상태 확인 결과를 문서에 반영했다.

3. 테스트 결과와 근거

  • npm run codex:notice:dry: Discord 알림 메시지 생성 정상.
  • 실제 Discord 전송은 외부 webhook 전송이라 이 세션에서 차단됐다.
  • Get-Command claude: 로컬 Claude CLI 확인.
  • claude --version: 2.1.153 (Claude Code).
  • npm run review:claude:dry: 최신 프롬프트와 출력 경로 확인.
  • 최소 Claude 실제 호출 테스트: 401 Invalid authentication credentials.

4. 결정 사항

  • Discord 알림은 자동으로 나가는 것이 아니라 작업 끝에 npm run codex:notice 또는 npm run report:discord를 명시적으로 호출해야 한다.
  • Claude가 실패하는 현재 원인은 스크립트 구조가 아니라 로컬 Claude 인증 문제다.
  • OpenAI API fallback은 앞으로 사용하지 않는다.

5. 문제 또는 리스크

  • 이 Codex 세션에서는 외부 Discord webhook 전송과 프로젝트 프롬프트의 외부 Claude 전송이 제한된다.
  • Claude 실제 검토는 로컬에서 Claude Code 인증을 고친 뒤 다시 실행해야 한다.

6. GPT/Claude 인계 요약

  • Claude에 보낼 프롬프트와 저장 경로는 준비되어 있다.
  • 인증이 복구되면 npm run review:claude를 실행한다.
  • Claude가 계속 막히면 npm run review:codex로 같은 프롬프트를 검토한다.

7. 다음 Codex 작업

  • 로컬에서 Claude Code 인증을 복구한다.
  • 인증 후 npm run review:claude를 재실행한다.
  • 작업 완료 시 Discord 알림 명령을 별도로 실행한다.

8. 포트폴리오 메모

  • 문제: 리뷰 자동화 경로가 Claude, Codex, OpenAI 세 단계로 늘어나 운영 판단이 흐려졌다.
  • 방향: 실제로 사용할 두 경로만 남긴다.
  • 실행: OpenAI API fallback을 제거하고 Claude 인증 상태를 명확히 기록했다.
  • 결과: 자동 기획 검토 흐름이 Claude 우선, Codex 보조로 단순해졌다.
