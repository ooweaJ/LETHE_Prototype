# 2026-06-02-05 - v0.2 브라우저 QA 및 사람 테스트 게이트 보고

## 1. 현재 빌드 상태

- LETHE HTML alpha v0.2는 브라우저 QA 기준으로 결과 화면과 JSON payload 검증을 통과했다.
- 사람 테스트는 바로 진행하지 않는다. 전투 연출과 LETHE 고유 시스템 체감이 더 강해진 뒤 진행한다.

## 2. 오늘 바뀐 것

- 브라우저 표기를 `v0.2`로 고쳤다.
- 브라우저 실제 보스/망각 타이밍을 v0.2 목표인 9분 기준으로 맞췄다.
- `?qa=fast` 모드를 추가해 결과 화면과 JSON payload를 빠르게 검증할 수 있게 했다.
- 결과 화면 문구의 조사 오류를 피하도록 문장을 고쳤다.
- 결과 카드 레이아웃을 데스크톱 2열로 바꿔 다운로드 버튼이 QA viewport 안에 보이게 했다.
- 정적 CSS/JS에 버전 query를 붙여 캐시된 파일이 남는 문제를 줄였다.

## 3. 테스트 결과와 근거

- `node --check src/game.js`: 통과.
- `npm run ai:test:quick`: `GO_CANDIDATE`, Alpha Fun Score `0.7798`.
- `npm run ai:test`: `GO_CANDIDATE`, Alpha Fun Score `0.7737`.
- 브라우저 QA:
  - `?qa=fast`에서 질문 화면까지 도달했다.
  - 결과 화면 문구가 `처형자의 섬광 기억이 망각되었습니다.`처럼 표시됐다.
  - 결과 패널은 QA viewport에서 내부 스크롤 없이 표시됐다.
  - 설문 입력 후 다운로드 버튼이 활성화됐다.
  - JSON payload에 `memoryNames`, `questionNames`, `forgottenName`, `deletionWeights`, `survey`, `echo`, `experiment`가 들어갔다.

## 4. 결정 사항

- AI 기준으로는 사람 테스트 후보지만, 아직 사람에게 보여줄 완성감은 부족하다고 판단한다.
- 사람 테스트 전 다음 단계는 콘텐츠 확장이 아니라 전투 연출과 망각/잔향 시스템 가시성 강화다.
- `?qa=fast`는 QA용으로만 사용하고, 기본 빌드는 9분 망각 타이밍을 유지한다.

## 5. 문제 또는 리스크

- 현재 공격 연출은 기능 확인 수준이고, “화려한 공격”이나 기억별 개성이 충분히 강하지 않다.
- 자동 기억 발동은 작동하지만, 플레이어가 어떤 기억이 빌드를 이끌고 있는지 순간적으로 알아보기 어렵다.
- 사람 테스트를 지금 하면 시스템보다 프로토타입의 밋밋함에 반응이 섞일 수 있다.

## 6. GPT/Claude 인계 요약

- v0.2 QA는 통과했다.
- 다음 기획 질문은 “사람 테스트 전 전투 연출/시스템 가시성을 어디까지 올릴 것인가”다.
- Claude Code 실제 호출은 로컬 로그인 후 다시 실행해야 한다.

## 7. 다음 Codex 작업

- 기억별 공격 이펙트를 더 구분되게 만든다.
- 의존도 상승, 망각 위험, 잔향 변형이 플레이 중 더 눈에 띄게 만든다.
- QA fast 모드로 결과 화면과 JSON payload를 계속 회귀 검증한다.

## 8. 포트폴리오 메모

- 문제: AI 지표는 통과했지만, 실제 유저에게 보여줄 프로토타입 완성감은 아직 낮다.
- 방향: 사람 테스트를 서두르지 않고, LETHE의 핵심 체감인 화려한 자동 기억 공격과 상실 후 변형을 먼저 강화한다.
- 실행: 브라우저 QA에서 타이밍 불일치, 표기, 결과 문구, 레이아웃, JSON 검증 경로를 고쳤다.
- 결과: v0.2는 검증 가능한 상태가 됐고, v0.3 전 단계의 목표는 사람 테스트용 완성감 확보다.
