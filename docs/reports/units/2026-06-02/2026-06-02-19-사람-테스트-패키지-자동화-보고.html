2026-06-02-19 - 사람 테스트 패키지 자동화 보고

1. 현재 빌드 상태

  • LETHE HTML alpha v0.5는 사람 테스트 준비 상태다.
  • 사람 테스트용 정적 폴더를 한 명령으로 생성할 수 있게 됐다.

2. 오늘 바뀐 것

  • scripts/prepare_playtest_build.js를 추가했다.
  • playtest:package, playtest:package:dry npm 명령을 추가했다.
  • dist/를 .gitignore에 추가했다.
  • npm run playtest:package가 dist/lethe-v0.5-playtest에 정적 게임 파일, 가이드, 노트 템플릿, README를 복사한다.
  • docs/HUMAN_PLAYTEST_GUIDE.md와 docs/CODEX_RUNBOOK.md에 package 절차를 추가했다.
  • doctor deep에 package dry-run 검사를 추가했다.

3. 테스트 결과와 근거

  • node --check scripts/prepare_playtest_build.js: 통과.
  • node --check scripts/check_local_pipeline.js: 통과.
  • npm run playtest:package:dry: 출력 파일 목록 확인.
  • npm run playtest:package: dist/lethe-v0.5-playtest 생성.
  • npm run doctor:deep: 38 pass, 0 warn, 0 fail.

4. 결정 사항

  • 배포용 dist/ 산출물은 git에 올리지 않는다.
  • 테스트 공유 전에는 npm run playtest:package로 최신 정적 폴더를 다시 만든다.
  • 테스트 후 JSON 로그는 playtest_logs/에 모으고 npm run playtest:summary로 다음 Claude/GPT 판단으로 넘긴다.

5. 문제 또는 리스크

  • 현재 package는 zip을 만들지 않고 폴더만 만든다.
  • 브라우저 보안 설정에 따라 file URL 실행이 막히는 환경에서는 로컬 서버가 필요할 수 있다.
  • 실제 사람 테스트가 없으면 패키지 자동화만으로 product gate가 통과되지는 않는다.

6. GPT/Claude 인계 요약

  • 사람 테스트 준비는 실행 패키지, 원본 로그 폴더, 요약 스크립트, human-test prompt 생성까지 연결됐다.
  • 다음 Claude/GPT 인계는 실제 사람 테스트 결과가 모인 뒤 진행한다.

7. 다음 Codex 작업

  • 5-8명 사람 테스트를 진행한다.
  • 필요한 경우 테스트 실행용 로컬 서버 명령을 추가한다.
  • 테스트 로그가 쌓이면 npm run playtest:summary를 실행한다.

8. 포트폴리오 메모

  • 문제: 사람 테스트를 할 때 매번 어떤 파일을 공유하고 어떤 로그를 받아야 하는지 흐려질 수 있다.
  • 방향: 실행 패키지, 테스트 가이드, 로그 수집, 결과 요약을 명령으로 연결한다.
  • 실행: playtest package script와 doctor 검증을 추가했다.
  • 결과: 다른 로컬에서도 doctor -> package -> test -> summary -> planning 흐름을 재현할 수 있다.
