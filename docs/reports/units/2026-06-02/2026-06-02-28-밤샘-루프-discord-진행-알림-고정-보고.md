# 2026-06-02-28 - 밤샘 루프 Discord 진행 알림 고정 보고

## 1. 현재 빌드 상태

- v0.9 release-feel loop 준비는 완료됐고, 다음 구현 범위는 Work Package 1이다.
- 밤샘 루프는 이제 실제 실행 시 Discord 진행 알림을 기본으로 보낸다.
- `--no-discord`를 명시하지 않는 한 시작/진행/체크포인트/막힘/완료 알림과 work-unit report upload가 실행된다.

## 2. 오늘 바뀐 것

- `scripts/run_overnight_loop.js`의 기본 Discord 동작을 실제 전송으로 바꿨다.
- 루프 시작 시 `start` notice를 보낸다.
- 각 반복 시작 시 `status` notice를 보낸다.
- 각 반복 완료 시 `checkpoint` notice를 보낸다.
- 필수 단계 실패 시 `blocked` notice를 보내고 blocker prompt를 남긴다.
- 루프 종료 시 `done` notice를 보낸다.
- 보고서 단계에서는 `node scripts/send_discord_report.js --latest-section`을 실행한다.
- `--discord-dry-run`은 메시지를 미리 보기만 하고, `--no-discord`는 deliberate local debugging에서만 알림을 끈다.

## 3. 테스트 결과와 근거

- `node --check scripts/run_overnight_loop.js`: 통과.
- `npm run overnight:loop:dry`: 기본 루프가 실제 Discord notice/report 경로를 preview함.
- `node scripts/run_overnight_loop.js --allow-dirty --skip-preflight --provider none --test none --skip-post-ai --skip-report --discord-dry-run --iterations 1`: 통과.
- Notification smoke log: `docs/loop_runs/2026-06-02-overnight-164700.md`.

## 4. 결정 사항

- 밤샘 루프에서 Discord는 선택 기능이 아니라 기본 진행 추적 기능이다.
- 사용자가 진행 상황을 파악해야 하므로 실제 루프에서는 알림을 끄지 않는다.
- 알림을 끄는 옵션은 로컬 디버깅용으로만 둔다.

## 5. 문제 또는 리스크

- Discord 알림은 상태 채널일 뿐이고, Codex UI 승인이나 실제 구현 판단을 대신하지 않는다.
- `--latest-section`은 보고서의 마지막 top-level 섹션을 보내므로, 최신 작업 보고는 반드시 보고서 맨 끝에 추가해야 한다.
