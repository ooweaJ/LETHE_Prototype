2026-06-02-15 - 테스트 결과 기반 기획 파이프라인 보고

1. 현재 빌드 상태

  • LETHE HTML alpha는 v0.5 core-fun human-test ready 상태다.
  • 다음 방향은 “그냥 기능을 더 만들기”가 아니라 AI/사람 테스트 결과를 보고하고, 그 판단으로 다음 구현을 정하는 흐름이어야 한다.

2. 오늘 바뀐 것

  • scripts/run_planning_pipeline.js를 추가했다.
  • planning:pipeline, planning:pipeline:dry, planning:pipeline:prompt, planning:pipeline:codex, planning:pipeline:claude npm 명령을 추가했다.
  • planning:pipeline:prompt를 추가해 외부 모델 호출 없이 AI 테스트와 프롬프트 생성까지만 수행할 수 있게 했다.
  • 파이프라인은 quick AI test를 기본 실행하고 docs/review_prompts/YYYY-MM-DD-pipeline.md를 생성한다.
  • 생성된 프롬프트를 Claude에 먼저 보내고, 실패하면 Codex CLI fallback으로 응답을 저장한다.
  • Claude/Codex wrapper 문구를 “기획 검토자”에서 “테스트 결과 기반 기획 파트너”로 바꿨다.
  • 날짜 뒤 suffix가 붙은 프롬프트 파일도 자동 선택할 수 있게 했다.
  • Discord 작업 보고도 같은 날짜의 최신 프롬프트를 첨부하도록 바꿨다.

3. 테스트 결과와 근거

  • node --check scripts/run_planning_pipeline.js: 통과.
  • node --check scripts/ask_claude_review.js: 통과.
  • node --check scripts/ask_codex_review.js: 통과.
  • node --check scripts/send_discord_report.js: 통과.
  • npm run planning:pipeline:dry: 생성될 프롬프트와 호출 경로 확인.
  • npm run planning:pipeline:prompt: quick AI test 통과, docs/review_prompts/2026-06-02-pipeline.md 생성.
  • npm run review:claude:dry: 최신 pipeline 프롬프트와 docs/review_responses/2026-06-02-pipeline-claude.md 출력 경로 확인.
  • npm run review:codex:dry: 최신 pipeline 프롬프트와 docs/review_responses/2026-06-02-pipeline-codex.md 출력 경로 확인.
  • 실제 npm run planning:pipeline: 이 Codex 세션에서는 외부 Claude/Codex 서비스로 프로젝트 프롬프트를 전송하는 단계가 차단됐다.

4. 결정 사항

  • v0.5 이후 판단은 npm run planning:pipeline으로 최신 AI 결과를 포함한 프롬프트를 만든 뒤 Claude/Codex 답변을 받는다.
  • 답변 파일이 생기면 Codex가 읽고 docs/NEXT_TASKS.md에 작업화한 뒤 구현을 이어간다.
  • 실제 Claude 호출은 프로젝트 내용을 외부 Claude 서비스로 보내는 작업이므로 로컬 인증과 사용자의 명시적 실행 환경이 필요하다.
  • 이 Codex 세션에서 외부 전송이 막히면 npm run planning:pipeline:prompt로 프롬프트까지만 만들고, 사용자의 로컬 터미널에서 npm run planning:pipeline을 실행한다.

5. 문제 또는 리스크

  • Claude 인증이 깨져 있으면 자동으로 Codex CLI fallback으로 넘어간다.
  • Codex CLI fallback도 로컬 Codex 로그인 세션과 네트워크 상태에 의존한다.
  • 모델 응답을 코드로 자동 적용하는 것은 위험하므로, 구현 단계는 Codex가 응답을 읽고 선택 작업을 적용하는 방식으로 유지한다.

6. GPT/Claude 인계 요약

  • 새 파이프라인 프롬프트는 v0.5 AI 결과, Chrome headless QA 결과, Unity 전환 판단 질문을 함께 전달한다.
  • Claude/GPT에게 묻는 핵심은 “바로 사람 테스트인가, HTML v0.6을 먼저 작게 만들 것인가”다.

7. 다음 Codex 작업

  • 파이프라인 dry-run과 문법 검사를 통과시킨다.
  • 사용자가 실제 외부 호출을 진행할 수 있는 환경에서 npm run planning:pipeline을 실행한다.
  • 응답이 저장되면 그 내용을 기준으로 다음 구현 또는 사람 테스트 준비를 진행한다.
