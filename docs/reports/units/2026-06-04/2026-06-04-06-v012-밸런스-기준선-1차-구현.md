# 2026-06-04-06 - v012 밸런스 기준선 1차 구현

## 1. 현재 빌드 상태

- LETHE HTML Alpha는 `v0.12` 첫 밸런스 기준선 구현 상태다.
- v0.11의 기억 획득, 가중 망각, 무기 잔향, 무기 진화 구조는 유지했다.
- 이번 패스는 새 기능 확장이 아니라 `굶주린 칼무리`, 적 스케일링, telemetry, 전술 집중 가중치, `피의 늪` 폭주 위험을 먼저 정리했다.

## 2. 오늘 바뀐 것

- 브라우저 라벨과 QA 기대 버전을 `v0.12`로 올렸다.
- `굶주린 칼무리`를 `28 DPS * dt`로 교정하고 가까운 타깃 우선 soft cap을 적용했다.
- 일반 적 HP/피해에 시간+레벨 하이브리드 스케일링을 추가했다.
- JSON 로그에 `balance`와 `telemetry`를 추가했다.
- telemetry는 피해원별 누적 피해/DPS, 보스 TTK/집중 DPS, 레벨업 시각, 3슬롯 완성 시각, 적 스케일링, 5초 샘플을 남긴다.
- 전술 집중의 망각 가중치 반영을 12배에서 3배로 낮췄다.
- `피의 늪`은 쌍검 30%, 대검 20% 발동률로 낮추고, 지속 시간/틱/피해를 줄였으며 최대 활성 장판 5개 제한을 넣었다.
- `docs/CODEX_STATUS.md`, `docs/NEXT_TASKS.md`, `docs/devlog/2026-06-04.md`를 v0.12 기준으로 갱신했다.

## 4. 결정한 것

- 보스 HP는 이번 패스에서 올리지 않는다.
- 먼저 실제 브라우저 로그의 `telemetry.bossFights[].ttk`, `focusedDps`, `dpsBySource`를 보고 첫 보스 HP 목표를 역산한다.
- 순수 레벨 비례 스케일링은 성장 벌칙처럼 느껴질 수 있어 시간+레벨 하이브리드로 간다.
- 실제 Discord 전송은 사용자 명시 요청이 있을 때만 실행하고, 이번 작업은 dry-run 확인 대상으로 둔다.

## 5. 문제 또는 리스크

- 브라우저 QA는 현재 환경에서 Chrome/`CHROME_PATH` 문제로 막혀 있어 실제 플레이 assertion은 아직 없다.
- AI 시뮬레이터는 v0.12 코드 telemetry를 직접 실행하는 검증이 아니므로 밸런스 증거로 과신하면 안 된다.
- `굶주린 칼무리` DPS 교정으로 초반 체감이 급격히 약해질 수 있다.
- 보스 HP와 XP 곡선은 telemetry 로그를 본 뒤 2차 패스로 조정해야 한다.
