2026-06-04-05 - v011 밸런스 수치표 정리

1. 현재 빌드 상태

  • LETHE HTML Alpha는 v0.11 구조 구현 이후 첫 밸런스 점검 단계다.
  • 별도 밸런스 수치표는 없었고, 실제 수치는 src/game.js와 기획서에 흩어져 있었다.
  • 현재 코드 기준 수치표는 docs/BALANCE_TABLE_v0_11.md에 새로 정리했다.

2. 오늘 바뀐 것

  • docs/BALANCE_TABLE_v0_11.md를 추가했다.
  • 수치표에 런 구조, 플레이어, 무기, 기억, 적, 스폰, 보스, 시너지, 망각 확률, 잔향, 무기 진화를 정리했다.
  • README, docs/CODEX_STATUS.md, docs/NEXT_TASKS.md에 밸런스 수치표 위치와 다음 밸런스 범위를 연결했다.
  • 첫 번째 의심 원인으로 굶주린 칼무리가 프레임마다 피해를 주는 문제를 기록했다.

3. 테스트 결과와 근거

  • 코드 검색으로 현재 실제 수치가 독립 문서가 아니라 src/game.js에 흩어져 있음을 확인했다.
  • src/game.js 기준으로 무기/기억/적/보스/잔향/진화 수치를 수치표에 옮겼다.
  • 굶주린 칼무리는 dt 없이 업데이트마다 7.4 피해를 적용하고 있어, 60fps 기준 단일 대상 약 444 DPS까지 튈 수 있음을 확인했다.
  • npm run report: 일일 HTML과 단위 보고서 5개 생성.
  • npm run report:check: 통과.
  • node scripts/send_discord_report.js docs/reports/2026-06-04.md --dry-run --section "2026-06-04-05 - v011 밸런스 수치표 정리": 한국어 Discord 본문과 수치표 보고 단위 HTML 첨부를 확인.
  • 실제 Discord 전송은 이번 새 밸런스 수치표 작업에 대한 사용자 명시 요청이 없어 보류했다.

4. 결정한 것

  • 밸런스 조정 기준 문서는 docs/BALANCE_TABLE_v0_11.md로 둔다.
  • 다음 구현은 새 기능보다 첫 밸런스 패스를 우선한다.
  • 첫 밸런스 패스는 굶주린 칼무리, 첫 180초 성장 속도, 피의 늪, 보스 HP를 우선 본다.

5. 문제 또는 리스크

  • 현재 수치는 v0.11 구조 검증용 임시값이라 실제 플레이 감각과 맞지 않을 가능성이 높다.
  • 굶주린 칼무리가 다른 기억 대비 압도적으로 강하면 기억 획득/망각/진화 테스트가 왜곡된다.
  • 피의 늪은 쌍검 발동률과 틱 피해가 높아 후반 화면 장악으로 이어질 수 있다.
  • Chrome/CHROME_PATH 문제 때문에 자동 브라우저 QA는 아직 막혀 있다.

6. GPT/Claude 인계 요약

  • 기획 구조는 v0.11로 어느 정도 잡혔지만, 수치 밸런스는 아직 정리/검증 전이었다.
  • 현재 가장 먼저 의심할 수치는 굶주린 칼무리의 프레임 단위 피해다.
  • GPT/Claude는 밸런스 리뷰 시 수치표를 기준으로 기억별 역할 대비 DPS/유틸/리스크가 맞는지 봐야 한다.

7. 다음 Codex 작업

  • 굶주린 칼무리를 초당 피해 기준으로 고친다.
  • 첫 보스 전 목표를 평균 레벨업 2-3회, 기억 2-3개 완성으로 맞춘다.
  • 피의 늪 발동률/틱 피해를 낮춰 기준선을 만든다.
  • v0.11 전용 브라우저/AI 밸런스 체크를 추가한다.

8. 포트폴리오 노트

  • 문제: 게임 구조는 잡혔지만 수치가 흩어져 있어 밸런스 판단 기준이 없었다.
  • 방향: 코드 기준 실제 수치를 표로 모아 조정 가능한 기준선을 만든다.
  • 행동: v0.11 수치표를 만들고 최우선 밸런스 위험을 표시했다.
  • 결과: 다음 작업은 감으로 튜닝하는 대신 수치표 기준으로 밸런스 패스를 시작할 수 있다.
