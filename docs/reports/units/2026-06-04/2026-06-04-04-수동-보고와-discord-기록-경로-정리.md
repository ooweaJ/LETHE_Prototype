# 2026-06-04-04 - 수동 보고와 Discord 기록 경로 정리

## 1. 현재 빌드 상태

- 보고 스크립트는 일일 보고, 작업 단위 보고, 최신 섹션 첨부, Discord dry-run을 이미 지원한다.
- 자동 루프는 Discord 작업 단위 보고를 호출하지만, 일반 수동 Codex 구현 턴은 마지막에 보고/Discord 명령을 명시적으로 실행해야 한다.
- v0.11 구현 보고는 `dry-run`만 됐던 상태에서 실제 Discord 전송까지 보정 완료했다.

## 2. 오늘 바뀐 것

- `docs/DISCORD_REPORTING.md`에 수동 보고 절차를 문서화했다.
- `docs/ai/`는 매 작업 자동 누적 로그가 아니라 AI 협업 규칙/실패 사례를 보관하는 정적 문서라고 기록했다.
- 작업별 AI 활용 기록은 devlog, daily report, review prompt, review response에 남긴다고 정리했다.
- `## 2. 오늘 바뀐 것`처럼 번호가 붙은 섹션도 Discord 요약이 인식하도록 `scripts/send_discord_report.js`를 고쳤다.
- v0.11 구현 보고와 수동 보고 경로 정리 보고서를 한국어로 다시 작성했다.

## 3. 테스트 결과와 근거

- 스크립트 검색으로 `run_autonomous_dev_loop.js`가 `sendWorkUnitReport()`를 호출하는 것을 확인했다.
- `report:discord:unit:dry`와 `report:discord:unit` 명령이 이미 존재하는 것도 확인했다.
- `node scripts/send_discord_report.js docs/reports/2026-06-04.md --dry-run --section "2026-06-04-03 - v0.11 기억 획득과 무기 진화 구현"`이 실제 작업/상태/문제/인계 요약을 뽑고 v0.11 단위 HTML을 첨부하는 것을 확인했다.
- `node scripts/send_discord_report.js docs/reports/2026-06-04.md --section "2026-06-04-03 - v0.11 기억 획득과 무기 진화 구현"`로 v0.11 구현 보고 실제 Discord 전송을 완료했다.

## 4. 결정한 것

- 수동 구현 후 Discord 업로드가 없었다고 해서 스크립트 자체가 자동 실패한 것으로 보지 않는다.
- Codex 관리 세션에서는 기본 증거로 `npm run report:discord:unit:dry`를 실행한다.
- 실제 webhook 전송은 의도된 trusted local 환경에서 `npm run report:discord:unit` 또는 특정 섹션 전송으로 수행한다.
- v0.11 구현처럼 사용자가 완료 알림을 기대하는 큰 단위는 `dry-run`에서 멈추지 않고 실제 전송 결과까지 확인한다.

## 5. 문제 또는 리스크

- 실제 Discord 업로드는 `.env`의 `DISCORD_WEBHOOK_URL`과 네트워크 접근이 필요하다.
- 수동 작업 후 Codex가 dry-run에서 멈추면 Discord/latest-section 증거가 구현보다 뒤처진다.
- `docs/ai/`를 작업별 로그로 기대하면 문서가 오래된 것처럼 보일 수 있다.
- latest-section은 항상 마지막 보고 단위를 잡기 때문에, v0.11 구현 보고처럼 특정 단위를 보내려면 `--section`을 명시해야 한다.

## 6. GPT/Claude 인계 요약

- 보고 시스템은 존재하지만 자동 전송 경로는 자동 루프에 묶여 있다.
- 수동 구현은 보고서 생성, 보고서 체크, Discord dry-run 또는 실제 전송까지 닫아야 한다.
- 작업별 AI 활용 증거는 reports/devlogs와 review prompt/response 파일에 남긴다.
- 이번 누락은 dry-run을 실제 전송으로 착각한 Codex 운용 실수이며, v0.11 구현 보고는 실제 전송으로 보정됐다.

## 7. 다음 Codex 작업

- 매 수동 구현 단위 후 보고서 생성/체크/dry-run 또는 실제 전송을 실행한다.
- 같은 실수가 반복되면 수동 종료용 wrapper 명령을 추가한다.
