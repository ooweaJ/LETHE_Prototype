## 4. Decisions made

- Do not treat missing Discord upload after manual implementation as automatic failure of the script.
- Use `npm run report:discord:unit:dry` as the default proof in Codex-managed sessions.
- Use `npm run report:discord:unit` only in trusted local environments where real webhook sending is intended.

- Real Discord upload needs `.env` with `DISCORD_WEBHOOK_URL` and network access.
- If Codex forgets to run the dry-run after manual work, Discord/latest-section evidence will lag behind the actual implementation.
- AI collaboration docs can look stale if readers expect them to be per-work logs.
