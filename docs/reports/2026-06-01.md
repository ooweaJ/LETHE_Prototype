## Portfolio Notes

### Problem

The project needed to verify whether losing a relied-on ability creates regret rather than irritation.

### Direction

The prototype was scoped around one complete loop: build attachment, beat a boss, predict the loss, forget the most relied-on memory, gain an echo, and answer Q1/Q2.

### Action

Implemented a browser alpha and an AI alpha test system that can run hundreds to thousands of simulated runs, measure deletion patterns, estimate regret/irritation proxies, and produce reports.

### Result

The first run produced high regret proxy, low irritation proxy, and high prediction match. The system also exposed concrete tuning issues: early forgetting, `처형자의 섬광` deletion bias, and slightly weak loss impact.

### Reusable Sentence

I converted an abstract roguelike design question into a playable HTML prototype plus an automated validation loop that measured whether the core loss mechanic felt regrettable, predictable, and recoverable.
