  • 화면 QA가 통과하면 5-8명용 human playtest 가이드를 만든다.
  • 사람 테스트에서는 “아까움 / 짜증 / 무감정” 중 어떤 반응인지와 그 이유를 가장 중요하게 본다.

v0.2 브라우저 QA 및 사람 테스트 게이트 보고

1. 현재 빌드 상태

  • LETHE HTML alpha v0.2는 브라우저 QA 기준으로 결과 화면과 JSON payload 검증을 통과했다.
  • 사람 테스트는 바로 진행하지 않는다. 전투 연출과 LETHE 고유 시스템 체감이 더 강해진 뒤 진행한다.

2. 오늘 바뀐 것

  • 브라우저 표기를 v0.2로 고쳤다.
  • 브라우저 실제 보스/망각 타이밍을 v0.2 목표인 9분 기준으로 맞췄다.
  • ?qa=fast 모드를 추가해 결과 화면과 JSON payload를 빠르게 검증할 수 있게 했다.
  • 결과 화면 문구의 조사 오류를 피하도록 문장을 고쳤다.
  • 결과 카드 레이아웃을 데스크톱 2열로 바꿔 다운로드 버튼이 QA viewport 안에 보이게 했다.
  • 정적 CSS/JS에 버전 query를 붙여 캐시된 파일이 남는 문제를 줄였다.

3. 테스트 결과와 근거

  • node --check src/game.js: 통과.
  • npm run ai:test:quick: GO_CANDIDATE, Alpha Fun Score 0.7798.
  • npm run ai:test: GO_CANDIDATE, Alpha Fun Score 0.7737.
  • 브라우저 QA:
  • ?qa=fast에서 질문 화면까지 도달했다.
  • 결과 화면 문구가 처형자의 섬광 기억이 망각되었습니다.처럼 표시됐다.
  • 결과 패널은 QA viewport에서 내부 스크롤 없이 표시됐다.
  • 설문 입력 후 다운로드 버튼이 활성화됐다.
  • JSON payload에 memoryNames, questionNames, forgottenName, deletionWeights, survey, echo, experiment가 들어갔다.

4. 결정 사항

  • AI 기준으로는 사람 테스트 후보지만, 아직 사람에게 보여줄 완성감은 부족하다고 판단한다.
  • 사람 테스트 전 다음 단계는 콘텐츠 확장이 아니라 전투 연출과 망각/잔향 시스템 가시성 강화다.
  • ?qa=fast는 QA용으로만 사용하고, 기본 빌드는 9분 망각 타이밍을 유지한다.

5. 문제 또는 리스크

  • 현재 공격 연출은 기능 확인 수준이고, “화려한 공격”이나 기억별 개성이 충분히 강하지 않다.
  • 자동 기억 발동은 작동하지만, 플레이어가 어떤 기억이 빌드를 이끌고 있는지 순간적으로 알아보기 어렵다.
  • 사람 테스트를 지금 하면 시스템보다 프로토타입의 밋밋함에 반응이 섞일 수 있다.

6. GPT/Claude 인계 요약

  • v0.2 QA는 통과했다.
  • 다음 기획 질문은 “사람 테스트 전 전투 연출/시스템 가시성을 어디까지 올릴 것인가”다.
  • Claude Code 실제 호출은 로컬 로그인 후 다시 실행해야 한다.

7. 다음 Codex 작업

  • 기억별 공격 이펙트를 더 구분되게 만든다.
  • 의존도 상승, 망각 위험, 잔향 변형이 플레이 중 더 눈에 띄게 만든다.
  • QA fast 모드로 결과 화면과 JSON payload를 계속 회귀 검증한다.

8. 포트폴리오 메모

  • 문제: AI 지표는 통과했지만, 실제 유저에게 보여줄 프로토타입 완성감은 아직 낮다.
  • 방향: 사람 테스트를 서두르지 않고, LETHE의 핵심 체감인 화려한 자동 기억 공격과 상실 후 변형을 먼저 강화한다.
  • 실행: 브라우저 QA에서 타이밍 불일치, 표기, 결과 문구, 레이아웃, JSON 검증 경로를 고쳤다.
  • 결과: v0.2는 검증 가능한 상태가 됐고, v0.3 전 단계의 목표는 사람 테스트용 완성감 확보다.

v0.3 기획 자동화 fallback 및 전투 가시성 개선 보고

1. 현재 빌드 상태

  • LETHE HTML alpha는 v0.3 pre-human-test polish 후보 상태다.
  • AI 1000런 기준 verdict는 계속 GO_CANDIDATE다.
  • 사람 테스트는 아직 진행하지 않는다. 전투 연출과 시스템 가시성을 먼저 더 끌어올리는 단계다.

2. 오늘 바뀐 것

  • scripts/ask_codex_review.js를 추가해 Codex CLI로도 같은 기획 답변을 파일로 받을 수 있게 했다.
  • npm run review:codex와 npm run review:codex:dry를 추가했다.
  • 브라우저 표기를 v0.3으로 올렸다.
  • 기억 발동 시 기억 이름이 전투 화면에 플로팅 텍스트로 뜨게 했다.
  • 피해 숫자, 타격 스파크, 투사체 trail, 보스 등장/페이즈 impact를 추가했다.
  • 기억 슬롯에 레테의 시선 태그와 의존도 퍼센트를 표시해 어떤 기억이 삭제 위험에 가까운지 더 잘 보이게 했다.

3. 테스트 결과와 근거

  • node --check src/game.js: 통과.
  • node --check scripts/ask_codex_review.js: 통과.
  • npm run review:codex:dry: 최신 v0.3 프롬프트와 출력 경로 확인.
  • npm run ai:test:quick: GO_CANDIDATE, Alpha Fun Score 0.7798.
  • npm run ai:test: GO_CANDIDATE, Alpha Fun Score 0.7737.
  • 브라우저 QA:
  • HTML Alpha Prototype v0.3 표시 확인.
  • ?qa=fast에서 질문 화면과 결과 화면까지 도달했다.
  • 레테의 시선 태그와 의존도 퍼센트가 기억 슬롯에 표시됐다.
  • 결과 패널은 내부 스크롤 없이 표시됐다.
  • JSON payload에 memoryNames, questionNames, forgottenName, deletionWeights, survey, echo, experiment가 들어갔다.
  • payload의 experiment.version은 v0.3으로 기록됐다.

4. 결정 사항

  • Claude Code는 설치됐지만 로컬 로그인/승인이 필요하므로, 내일 로컬 Mac에서 claude auth login을 진행해야 한다.
  • Claude가 막힐 때는 먼저 Codex CLI fallback을 사용한다.
  • 외부 기획 검토가 막혀도, 현재 범위에서는 새 콘텐츠를 추가하지 않고 전투 가시성/시스템 전달력만 개선한다.

5. 문제 또는 리스크

  • Codex CLI 방식도 로컬 Codex 로그인 세션과 플랜 사용 한도에 의존한다.
  • Claude Code는 설치됐지만 인증 흐름에서 멈추므로, 원격으로는 브라우저 승인까지 진행할 수 없다.
  • 공격 연출은 개선됐지만, 잔향이 “상실 후 변형”처럼 느껴지는지는 아직 결과 화면 중심이라 추가 개선 여지가 있다.

6. GPT/Claude 인계 요약

  • v0.3/version-up 기획 질문 Markdown은 준비되어 있다.
  • Claude 실행은 로컬 로그인 후 가능하다.
  • Codex CLI 실행은 npm run review:codex로 가능하며, 답변은 docs/review_responses/YYYY-MM-DD-codex.md에 저장된다.

7. 다음 Codex 작업

  • 잔향 보상이 결과 화면뿐 아니라 다음 플레이 감각으로 더 잘 이어지게 만든다.
  • 공격 이펙트가 기억별로 더 구분되는지 브라우저에서 계속 확인한다.
  • 사람 테스트는 v0.3 polish가 충분히 보인 뒤 docs/HUMAN_PLAYTEST_GUIDE.md로 진행한다.

8. 포트폴리오 메모

  • 문제: Claude 인증이 막히면 기획 자동화가 멈추고, v0.2는 기능은 통과했지만 전투 체감이 약했다.
  • 방향: Claude가 막힐 때 Codex CLI fallback으로 같은 프롬프트를 검토하고, 사람 테스트 전 핵심 전투 체감을 강화한다.
  • 실행: Codex CLI 검토 스크립트와 v0.3 전투 가시성 개선을 추가하고 브라우저 QA로 회귀를 확인했다.
  • 결과: v0.3 프로토타입은 기억 의존도와 망각 위험이 더 눈에 띄는 상태가 됐다.

v0.4 사람 테스트 후보 보고

1. 현재 빌드 상태

  • LETHE HTML alpha는 v0.4 human-test candidate 상태다.
  • AI 1000런과 heavy 5000런/3스테이지 모두 GO_CANDIDATE다.
  • 다음 큰 단계는 더 많은 기능 추가가 아니라 5-8명 사람 테스트다.

2. 오늘 바뀐 것

  • 브라우저 표기를 v0.4로 올렸다.
  • 기본 AI 테스트의 UI 선명도 값을 ui=0.78로 올렸다.
  • 결과 화면에서 사라진 행동과 잔향 변형을 분리했다.
  • 잔향 변형 카드에 요약 문장과 stat badge를 추가했다.
  • JSON payload에 echoTransformation을 추가했다.
  • AI 테스트 기본 명령과 alpha test 문서 예시를 echo=0.50, ui=0.78 기준으로 맞췄다.

3. 테스트 결과와 근거

  • node --check src/game.js: 통과.
  • node --check alpha_test/src/config.js: 통과.
  • node --check alpha_test/src/run_alpha.js: 통과.
  • npm run ai:test:quick: GO_CANDIDATE, Alpha Fun Score 0.8190.
  • npm run ai:test: GO_CANDIDATE, Alpha Fun Score 0.8261, regret 85.6%, irritation 0.4%, prediction 85.8%.
  • npm run ai:test:heavy: GO_CANDIDATE, Alpha Fun Score 0.8369, regret 87.6%, irritation 0.3%, prediction 87.7%.
  • npm run ai:sweep: echo=0.50, ui=0.78이 현재 가장 안정적인 GO_CANDIDATE 프리셋임을 확인했다.
  • 브라우저 QA:
  • HTML Alpha Prototype v0.4 표시 확인.
  • ?qa=fast에서 질문 화면과 결과 화면까지 도달했다.
  • 잔향 변형 카드와 echo badge가 표시됐다.
  • 결과 패널은 내부 스크롤 없이 표시됐다.
  • JSON payload에 echoTransformation, uiClarity: 0.78, experiment.version: v0.4가 들어갔다.

4. 결정 사항

  • 지금부터는 Codex가 더 큰 콘텐츠를 추가하기보다 사람 테스트를 위한 안정화 상태로 본다.
  • 상점, 메타 진행, 최종 보스, 기억 추가, 다중 지역 구조는 여전히 만들지 않는다.
  • 남은 29.4-29.6% power drop soft warning은 코드로 과보정하기보다 사람 테스트에서 실제 체감으로 확인한다.

5. 문제 또는 리스크

  • Alpha Fun Score는 올랐지만 목표 0.89+에는 아직 못 미친다.
  • 망각 직후 전투력 딥은 목표 30%보다 아주 조금 낮다.
  • AI 지표가 좋아져도 실제 사람이 잔향을 “상실 후 변형”으로 느낄지는 테스트가 필요하다.

6. GPT/Claude 인계 요약

  • 외부 기획 검토 없이도 v0.4는 사람 테스트 후보까지 도달했다.
  • Claude/GPT에게 다음에 물을 질문은 “사람 테스트 결과를 어떻게 해석해 v0.5 방향을 잡을지”다.
  • Codex CLI fallback은 준비되어 있지만, 최종 기획 판단은 사람 테스트 후 GPT/Claude에 다시 묻는 것이 좋다.

7. 다음 Codex 작업

  • docs/HUMAN_PLAYTEST_GUIDE.md 기준으로 5-8명 테스트를 진행한다.
  • JSON 로그와 관찰 메모를 분리해서 수집한다.
  • 테스트 후 regret/irritation/neutral/unclear 비율과 발화 내용을 요약한다.

8. 포트폴리오 메모

  • 문제: v0.3는 전투 가시성은 좋아졌지만, 잔향이 단순 보상처럼 보일 위험이 있었다.
  • 방향: 결과 화면에서 “잃은 행동”과 “남은 변형”을 분리해 LETHE의 핵심 감정을 명확히 한다.
  • 실행: v0.4 결과 화면, 로그 payload, AI 테스트 기본값, 브라우저 QA를 한 번에 맞췄다.
  • 결과: AI 기준 주요 게이트를 통과했고, 이제 실제 사람 테스트만 남은 후보 상태가 됐다.

Claude 자동화 오프라인 검증 보고

1. 현재 빌드 상태

  • LETHE HTML alpha는 계속 v0.4 human-test candidate 상태다.
  • 사람 테스트가 다음 주요 검증 단계라는 판단은 유지한다.
  • Claude 자동 리뷰 스크립트는 dry-run과 로컬 mock 저장 검증을 통과했다.

2. 오늘 바뀐 것

  • scripts/ask_claude_review.js에 --mock-response 옵션을 추가했다.
  • 실제 Claude 호출 동작은 유지하면서, 외부 전송 없이 출력 파일 생성 경로를 검증할 수 있게 했다.
  • docs/CODEX_STATUS.md와 docs/NEXT_TASKS.md에 Claude 자동화 검증 상태를 반영했다.

3. 테스트 결과와 근거

  • Get-Command claude: 로컬 claude 명령 설치 확인.
  • npm run review:claude:dry: 최신 프롬프트와 출력 경로 확인.
  • node --check scripts/ask_claude_review.js: 통과.
  • node scripts/ask_claude_review.js --output alpha_test/outputs/claude-review-mock.md --mock-response ...: mock 응답 파일 생성 성공.
  • 실제 npm run review:claude는 저장소 프롬프트를 외부 Claude 서비스로 보내는 작업이라 이 세션에서는 실행하지 않았다.

4. 결정 사항

  • Claude 자동화 배관은 정상으로 본다.
  • 실제 Claude 기획 검토는 외부 전송을 명시적으로 승인할 수 있는 로컬 환경에서만 실행한다.
  • 외부 호출이 어려울 때는 Codex CLI fallback이나 mock 검증으로 자동화 상태만 먼저 확인한다.

5. 문제 또는 리스크

  • mock 검증은 응답 저장과 경로 생성만 보장하며, 실제 Claude 품질이나 로그인 상태는 보장하지 않는다.
  • 실제 Claude 호출은 프로젝트 프롬프트가 외부 서비스로 전송되는 작업이므로 별도 승인과 주의가 필요하다.

6. GPT/Claude 인계 요약

  • 지금 당장 새 기획 질문을 보내기보다 5-8명 사람 테스트 결과를 먼저 모으는 것이 좋다.
  • 사람 테스트 후에는 regret/irritation/neutral/unclear 반응 분포와 대표 발화를 Claude/GPT에 넘겨 v0.5 방향을 묻는다.

7. 다음 Codex 작업

  • 사람 테스트 중에는 자동화 추가 구현보다 테스트 기록과 JSON 로그 수집을 우선한다.
  • 테스트 결과가 모이면 docs/review_prompts/에 새 해석 프롬프트를 만들고 Claude/GPT 검토를 요청한다.

8. 포트폴리오 메모

  • 문제: 외부 Claude 호출이 막히면 자동화 검증 전체가 멈춘 것처럼 보일 수 있다.
