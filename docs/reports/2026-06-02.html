  • 화면 QA가 통과하면 5-8명용 human playtest 가이드를 만든다.
  • 사람 테스트에서는 “아까움 / 짜증 / 무감정” 중 어떤 반응인지와 그 이유를 가장 중요하게 본다.

v0.2 브라우저 QA 및 사람 테스트 게이트 보고

1. 현재 빌드 상태

  • LETHE HTML alpha v0.2는 브라우저 QA 기준으로 결과 화면과 JSON payload 검증을 통과했다.
  • 사람 테스트는 바로 진행하지 않는다. 전투 연출과 LETHE 고유 시스템 체감이 더 강해진 뒤 진행한다.

2. 오늘 바뀐 것

  • 브라우저 표기를 v0.2로 고쳤다.
  • 브라우저 실제 보스/망각 타이밍을 v0.2 목표인 9분 기준으로 맞췄다.
  • ?qa=fast 모드를 추가해 결과 화면과 JSON payload를 빠르게 검증할 수 있게 했다.
  • 결과 화면 문구의 조사 오류를 피하도록 문장을 고쳤다.
  • 결과 카드 레이아웃을 데스크톱 2열로 바꿔 다운로드 버튼이 QA viewport 안에 보이게 했다.
  • 정적 CSS/JS에 버전 query를 붙여 캐시된 파일이 남는 문제를 줄였다.

3. 테스트 결과와 근거

  • node --check src/game.js: 통과.
  • npm run ai:test:quick: GO_CANDIDATE, Alpha Fun Score 0.7798.
  • npm run ai:test: GO_CANDIDATE, Alpha Fun Score 0.7737.
  • 브라우저 QA:
  • ?qa=fast에서 질문 화면까지 도달했다.
  • 결과 화면 문구가 처형자의 섬광 기억이 망각되었습니다.처럼 표시됐다.
  • 결과 패널은 QA viewport에서 내부 스크롤 없이 표시됐다.
  • 설문 입력 후 다운로드 버튼이 활성화됐다.
  • JSON payload에 memoryNames, questionNames, forgottenName, deletionWeights, survey, echo, experiment가 들어갔다.

4. 결정 사항

  • AI 기준으로는 사람 테스트 후보지만, 아직 사람에게 보여줄 완성감은 부족하다고 판단한다.
  • 사람 테스트 전 다음 단계는 콘텐츠 확장이 아니라 전투 연출과 망각/잔향 시스템 가시성 강화다.
  • ?qa=fast는 QA용으로만 사용하고, 기본 빌드는 9분 망각 타이밍을 유지한다.

5. 문제 또는 리스크

  • 현재 공격 연출은 기능 확인 수준이고, “화려한 공격”이나 기억별 개성이 충분히 강하지 않다.
  • 자동 기억 발동은 작동하지만, 플레이어가 어떤 기억이 빌드를 이끌고 있는지 순간적으로 알아보기 어렵다.
  • 사람 테스트를 지금 하면 시스템보다 프로토타입의 밋밋함에 반응이 섞일 수 있다.

6. GPT/Claude 인계 요약

  • v0.2 QA는 통과했다.
  • 다음 기획 질문은 “사람 테스트 전 전투 연출/시스템 가시성을 어디까지 올릴 것인가”다.
  • Claude Code 실제 호출은 로컬 로그인 후 다시 실행해야 한다.

7. 다음 Codex 작업

  • 기억별 공격 이펙트를 더 구분되게 만든다.
  • 의존도 상승, 망각 위험, 잔향 변형이 플레이 중 더 눈에 띄게 만든다.
  • QA fast 모드로 결과 화면과 JSON payload를 계속 회귀 검증한다.

8. 포트폴리오 메모

  • 문제: AI 지표는 통과했지만, 실제 유저에게 보여줄 프로토타입 완성감은 아직 낮다.
  • 방향: 사람 테스트를 서두르지 않고, LETHE의 핵심 체감인 화려한 자동 기억 공격과 상실 후 변형을 먼저 강화한다.
  • 실행: 브라우저 QA에서 타이밍 불일치, 표기, 결과 문구, 레이아웃, JSON 검증 경로를 고쳤다.
  • 결과: v0.2는 검증 가능한 상태가 됐고, v0.3 전 단계의 목표는 사람 테스트용 완성감 확보다.

v0.3 기획 자동화 fallback 및 전투 가시성 개선 보고

1. 현재 빌드 상태

  • LETHE HTML alpha는 v0.3 pre-human-test polish 후보 상태다.
  • AI 1000런 기준 verdict는 계속 GO_CANDIDATE다.
  • 사람 테스트는 아직 진행하지 않는다. 전투 연출과 시스템 가시성을 먼저 더 끌어올리는 단계다.

2. 오늘 바뀐 것

  • scripts/ask_openai_review.js를 추가해 Claude Code가 안 될 때 GPT 기획 검토를 터미널에서 호출할 수 있게 했다.
  • npm run review:openai와 npm run review:openai:dry를 추가했다.
  • 브라우저 표기를 v0.3으로 올렸다.
  • 기억 발동 시 기억 이름이 전투 화면에 플로팅 텍스트로 뜨게 했다.
  • 피해 숫자, 타격 스파크, 투사체 trail, 보스 등장/페이즈 impact를 추가했다.
  • 기억 슬롯에 레테의 시선 태그와 의존도 퍼센트를 표시해 어떤 기억이 삭제 위험에 가까운지 더 잘 보이게 했다.

3. 테스트 결과와 근거

  • node --check src/game.js: 통과.
  • npm run review:openai:dry: 최신 v0.3 프롬프트와 출력 경로 확인.
  • npm run review:openai: OPENAI_API_KEY 없음으로 중단. 실패 원인이 명확하게 출력된다.
  • npm run ai:test:quick: GO_CANDIDATE, Alpha Fun Score 0.7798.
  • npm run ai:test: GO_CANDIDATE, Alpha Fun Score 0.7737.
  • 브라우저 QA:
  • HTML Alpha Prototype v0.3 표시 확인.
  • ?qa=fast에서 질문 화면과 결과 화면까지 도달했다.
  • 레테의 시선 태그와 의존도 퍼센트가 기억 슬롯에 표시됐다.
  • 결과 패널은 내부 스크롤 없이 표시됐다.
  • JSON payload에 memoryNames, questionNames, forgottenName, deletionWeights, survey, echo, experiment가 들어갔다.
  • payload의 experiment.version은 v0.3으로 기록됐다.

4. 결정 사항

  • Claude Code는 설치됐지만 로컬 로그인/승인이 필요하므로, 내일 로컬 Mac에서 claude auth login을 진행해야 한다.
  • Claude가 막힐 때는 OpenAI fallback을 사용한다.
  • OpenAI fallback도 실제 호출에는 OPENAI_API_KEY가 필요하다.
  • 외부 기획 검토가 막혀도, 현재 범위에서는 새 콘텐츠를 추가하지 않고 전투 가시성/시스템 전달력만 개선한다.

5. 문제 또는 리스크

  • 현재 로컬에는 OpenAI API key가 없어 밤새 GPT 기획 검토 자동 실행은 아직 불가능하다.
  • Claude Code는 설치됐지만 인증 흐름에서 멈추므로, 원격으로는 브라우저 승인까지 진행할 수 없다.
  • 공격 연출은 개선됐지만, 잔향이 “상실 후 변형”처럼 느껴지는지는 아직 결과 화면 중심이라 추가 개선 여지가 있다.

6. GPT/Claude 인계 요약

  • v0.3/version-up 기획 질문 Markdown은 준비되어 있다.
  • Claude 실행은 로컬 로그인 후 가능하다.
  • GPT 실행은 .env 또는 셸 환경에 OPENAI_API_KEY를 넣은 뒤 npm run review:openai로 가능하다.

7. 다음 Codex 작업

  • 잔향 보상이 결과 화면뿐 아니라 다음 플레이 감각으로 더 잘 이어지게 만든다.
  • 공격 이펙트가 기억별로 더 구분되는지 브라우저에서 계속 확인한다.
  • 사람 테스트는 v0.3 polish가 충분히 보인 뒤 docs/HUMAN_PLAYTEST_GUIDE.md로 진행한다.

8. 포트폴리오 메모

  • 문제: Claude 인증이 막히면 기획 자동화가 멈추고, v0.2는 기능은 통과했지만 전투 체감이 약했다.
  • 방향: OpenAI fallback으로 기획 검토 경로를 하나 더 만들고, 사람 테스트 전 핵심 전투 체감을 강화한다.
  • 실행: GPT 호출 스크립트와 v0.3 전투 가시성 개선을 추가하고 브라우저 QA로 회귀를 확인했다.
  • 결과: 외부 기획 검토는 키/로그인만 남았고, v0.3 프로토타입은 기억 의존도와 망각 위험이 더 눈에 띄는 상태가 됐다.
