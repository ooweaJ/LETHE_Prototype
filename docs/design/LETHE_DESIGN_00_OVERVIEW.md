# 00. LETHE 기획 개요

최종 갱신: 2026-06-12 · 기준: HTML v0.12 코어 + 잔향/공명/궁극 확장

## 한 줄 소개

LETHE는 **기억을 잃을수록 다른 힘(잔향)이 남는** 로그라이크 액션 게임이다.

## 제품 비전

핵심 재미는 콘텐츠 수가 아니라 아래 순서에서 나온다.

```text
기억으로 강해진다
-> 가장 키운 기억을 잃는다 (망각)
-> 잃은 기억이 잔향으로 형태를 바꾼다
-> 잔향 조합이 새 빌드 목표가 된다
-> 잃었던 기억을 다시 얻으면 공명한다
-> 두 잔향을 +5로 키우면 궁극 잔향이 열린다
```

## 핵심 명제

- **활성 기억**: 지금 사용하는 능력. 플레이어/전장에 독립적으로 존재한다.
- **잔향(Echo)**: 잃어버린 기억이 무기·몸에 남긴 다른 형태. 활성 기억의 약화판이 아니다.
- **망각**: 익숙한 능력을 제거하지만, 그 레벨을 잔향으로 전환한다. 손실이 단순 삭제로 끝나지 않는다.
- **공명**: 잔향이 남은 기억을 다시 얻으면 강화된다.
- **성장 = 위험 관리**: 가장 높게 키운 기억이 다음 망각 후보이므로, 강하게 키우는 것은 강한 잔향을 준비하는 선택이기도 하다.

## 코어 루프 (한 판)

1. 무기 1개와 시작 기억 1개를 선택한다.
2. 사냥하며 레벨업으로 기억을 얻거나 강화한다(활성 슬롯 최대 3).
3. 문지기(보스) 이후 가장 레벨 높은 기억을 잃는다(망각).
4. 잃은 기억 레벨만큼 같은 잔향이 성장한다.
5. 결손 상태(기억 2개 이하)로 생존하며 바뀐 빌드를 적응한다.
6. 새 기억 또는 잃었던 기억을 다시 얻는다(공명).
7. 잔향 +5(각성), 두 각성 잔향으로 궁극 잔향을 완성한다.

> 타임라인·수치는 [01_RUN_LOOP](LETHE_DESIGN_01_RUN_LOOP.md), 망각/잔향 규칙은 [03_MEMORY_ECHO](LETHE_DESIGN_03_MEMORY_ECHO.md).

## 플레이어 감정 목표

망각은 짜증이 아니라 아쉬움과 기대를 동시에 만들어야 한다.

- "아, 이 기억 잃기 싫었는데."
- "근데 잔향이 남아서 다른 빌드가 되네?"
- "이걸 다시 얻으면 더 강해지겠는데?"
- "두 잔향을 만렙으로 만들면 뭔가 터지겠다."

단, 망각까지 도달하려면 **초반 전투·성장 루프가 먼저 재미있어야 한다**. 검증 우선순위는 항상 "초반 손맛 → 망각 도달 → 망각이 후회로 느껴짐" 순서다.

## 확정 상수 (요약)

| 항목 | 값 | 출처 |
| --- | --- | --- |
| 시작 활성 기억 슬롯 | 1 | `startingMemorySlots` |
| 최대 활성 기억 슬롯 | 3 | `maxActiveMemorySlots` |
| 기억 레벨 max | +5 | `maxMemoryLevel` |
| 잔향 레벨 max | +5 | `maxEchoLevel` |
| 망각 대상 | 활성 기억 중 최고 레벨(동률 시 플레이어 선택) | 03 문서 |
| 한 판 목표 길이 | 600초(HTML 압축) / Unity 6~8분 | `runDurationSec` |

## 범위

- **포함**: 무기 2, 기억 8, 잔향 8, 궁극 4, 적 4 + 보스, 런 1판, UI 전체, 밸런스 수치.
- **보류**: 상점, 메타 성장, 다중 지역, 최종 보스 확장, 출시급 사운드/UI 전체, 빌드 배포, `Assets/Lethe` 승격.

## 검증 게이트 (이 세트로 무엇을 증명하는가)

1. 초반 전투·성장이 망각 전까지 충분히 재미있다.
2. 기억을 잃는 순간이 납득되고 아쉽다.
3. 잔향이 전투에서 약화판이 아니라 형태 변화로 보인다.
4. 잃었던 기억을 다시 얻는 것이 설렌다(공명).
5. 4궁극 중 최소 하나가 빌드 목표가 된다.
6. **Unity 빌드가 HTML 수준의 게임 셸(HUD·레벨업·망각 결과 화면)을 갖춘다.**
