# LETHE 밸런스 기준 문서

최종 갱신: 2026-06-09

## 목적

이 문서는 HTML v0.12에서 얻은 수치 근거와 Unity 첫 slice의 초기 목표 범위를 분리해서 관리한다.

세부 과거 수치는 다음 문서를 근거로 본다.

- `docs/BALANCE_TABLE_v0_12.md`
- `docs/LETHE_v0.12_밸런스_개선_제안서.md`
- `docs/balance/2026-06-09-v012-balance-qa.md`

## 현재 HTML evidence build

채택된 최신 조정:

- 플레이어 최대 HP `180 -> 190`.
- 새 기억, 상점, 메타, 지역, 최종보스 확장은 하지 않음.

검증 결과:

- `npm run balance:loop` 1회차: `GO_BALANCE_BASELINE`.
- `npm run balance:loop` 2회차: `GO_BALANCE_BASELINE`.
- first boss clear: `100%`.
- full clear: `60%`.
- death: `40%`.
- first boss TTK median: 약 `18.97s` / `20.18s`.

## Unity 첫 slice 초기 목표

Unity는 HTML 수치를 그대로 복사하지 않는다. 아래는 첫 체감 기준이다.

| 지표 | 초기 목표 |
| --- | --- |
| 첫 성장 선택 | 30~50초 안 |
| 첫 기억 2~3개 완성 | 2~3분 안 |
| 첫 보스/문지기 TTK | 20~35초 |
| 첫 망각 후 전투력 하락 | 체감상 20~35% |
| 결손 생존 | 45~60초 |
| 보충 후 회복 | 이전 전투력의 80~110% |
| 첫 각성 잔향 도달 | 첫 slice 테스트 안에서 가능 |

## 잔향 수치 원칙

- 잔향 `+1~+2`: 보조 효과.
- 잔향 `+3`: 빌드 방향으로 고려 가능.
- 잔향 `+4`: 플레이 스타일 변화.
- 잔향 `+5`: 강한 각성 효과.

잔향 `+5`는 단순 피해 `+n%`가 아니라 화면에서 보이는 효과여야 한다.

## 수치 운영 원칙

- 메인 기획서에는 상세 수치를 넣지 않는다.
- 수치 변경은 이 문서 또는 별도 balance report에 기록한다.
- Unity 첫 slice에서는 재미가 확인되기 전까지 대규모 밸런스 표를 만들지 않는다.
- 먼저 전투 감각, 잔향 가시성, 공명 보상감을 본 뒤 수치를 좁힌다.
