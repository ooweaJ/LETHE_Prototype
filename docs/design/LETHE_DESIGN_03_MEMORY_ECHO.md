# 03. 기억 / 망각 / 잔향 / 공명 / 궁극

최종 갱신: 2026-06-12 · 출처: `src/game.js` (`memories`, `synergyRules`) + 잔향/공명/궁극 확장 개념

이 문서는 LETHE의 정체성 시스템이다. **HTML 코어의 8기억은 그대로 이식**하고, 그 위에 **망각→잔향→공명→각성→궁극** 확장을 얹는다.

## 용어 구분 (중요)

- HTML `memories[].echo` 필드(예: "공격속도 +18%")는 **활성 기억의 레벨 보너스**다. 이 문서에서는 "기억 레벨 보너스"로 부른다.
- **잔향(Echo)**은 기억을 잃었을 때 생기는 별도 빌드 상태다. HTML의 `state.echo`/`synergyRules` 보조 시스템을 확장·대체한다.
- 즉 Unity에서는 "기억 레벨 보너스"와 "잔향"을 **서로 다른 두 데이터**로 구현한다.

## 8기억 (활성 기억) — HTML 이식

출처: `memories`. 모든 기억은 레벨 1에서도 화면에서 역할이 읽혀야 한다.

| 기억 ID (HTML) | 이름 | 역할 | 쿨다운 | 태그 | 레벨 보너스 방향 | 잔향 ID |
| --- | --- | --- | --- | --- | --- | --- |
| `hungry_blades` | 굶주린 칼무리 | 근접 도트 | 0.34s | area, dot | 공속+18%, 지속피해+35% | `echo_kalmuri` |
| `blood_reflection` | 피의 반사 | 온힛 증폭 | 0s(상시) | survival, dot | 추가타+12%, 온힛+22% | `echo_blood` |
| `execution_flash` | 처형자의 섬광 | 버스트 | 3.15s | burst | 치명+12%, 치명피해+35% | `echo_execution` |
| `stalker_oath` | 추적자의 맹세 | 추적 다중 | 2.2s | burst | 투사체+1, 속도+25% | `echo_homing` |
| `shatter_ripple` | 파쇄의 파문 | 광역/넉백 | 4.55s | area, control | 범위+18%, 넉백+25%, 감소+6% | `echo_shockwave` |
| `stopped_second` | 멈춘 초침 | 제어 | 7.25s | control, dot | 쿨감+12%, 둔화+35% | `echo_timestop` |
| `ashen_guard` | 잿빛 보호막 | 보호/반격 | 8.4s | survival, area | 보호막+20%, 반격+10% | `echo_ashen` |
| `oblivion_brand` | 망각의 낙인 | 표식/증폭 | 5.2s | burst, control | 낙인+20%, 표식+10% | `echo_brand` |

### 기억 레벨 수락 기준 (+1 / +3 / +5)

- **+1**: 역할이 화면에서 보인다.
- **+3**: 범위/대상/지속 중 하나가 확장된다.
- **+5**: 행동 자체가 바뀐다.
- 쌍검·대검에서 최소 하나의 표현 차이를 가진다.

예) 굶주린 칼무리: 활성 시 DPS 28, 반경 72, 대상 소프트캡 4, 초과 피해 ×0.55(`balance.hungryBlades`). 레벨업 시 공속/지속피해가 위 보너스만큼 상승.

## 망각 → 잔향 변환

활성 기억이 망각될 때:

1. 활성 기억 중 **가장 레벨 높은 기억**을 대상으로 고른다(동률 시 플레이어 선택, 디버그는 최근 강화 기억 우선).
2. 활성 슬롯에서 제거한다.
3. 그 기억의 현재 레벨을 **같은 잔향에 더한다**.
4. 잔향 레벨은 **+5에서 멈춘다**.
5. +5 초과분은 버리지 않고 **잔향 과부하**로 전환한다.
6. 잔향이 +5에 도달하면 **각성 잔향** 효과가 열린다.

예시:
- `굶주린 칼무리 +3` 상실, 잔향 없음 → `칼무리 잔향 +3`.
- `칼무리 잔향 +3` 보유 중 `굶주린 칼무리 +2` 상실 → `칼무리 잔향 +5`.
- `칼무리 잔향 +2` 보유 중 `굶주린 칼무리 +5` 상실 → `칼무리 잔향 +5` + 과부하 2.

### 잔향 과부하

```text
초과분 = max(0, 기존 잔향 레벨 + 잃은 기억 레벨 - 5)
```

- 초과분 1마다 즉시 전투 폭발 **또는** 궁극 잔향 게이지 충전.
- 첫 Unity 구현에서는 둘 중 하나로 고정(권장: 즉시 폭발 + 게이지 충전 동시).
- 이 규칙 덕분에 잔향이 +5인 뒤에도 강한 기억을 잃는 선택이 의미를 가진다.

## 잔향 레벨 체감

| 레벨 | 의미 | 체감 목표 |
| --- | --- | --- |
| +1 | 희미한 흔적 | 낮은 확률 무기 보정 / 작은 수치 |
| +2 | 눈에 띄는 흔적 | 전투 중 가끔 보이는 부가 효과 |
| +3 | 빌드 재료 | 플레이어가 계획하기 시작 |
| +4 | 강한 잔향 | 범위/타겟팅/생존 방식이 바뀜 |
| +5 | 각성 잔향 | 화면에서 확실히 보이는 강한 효과 |

잔향 수락 기준: +1 작은 형태 변화 / +3 다중 대상·linger·전파·조건 강화 중 하나 / +5 각성 행동. HUD에 level과 awakened 상태 표시, 디버그에서 +1/+3/+5 즉시 전환.

## 8잔향 (각성 방향)

| 원본 기억 | 잔향 | 각성(+5) 전투 판타지 |
| --- | --- | --- |
| 굶주린 칼무리 | 칼무리 잔향 | 무기 타격에 칼날 잔흔, 주변 온힛/출혈 |
| 피의 반사 | 혈반 잔향 | 빠른 타격이 피의 추가타·흡혈 압박 |
| 처형자의 섬광 | 처형 잔향 | 치명/처형 조건에서 하얀 폭발 |
| 추적자의 맹세 | 추적 잔향 | 타격이 유도 잔탄/그림자 투사체 생성 |
| 파쇄의 파문 | 파문 잔향 | 타격이 주기적 충격파·넉백 |
| 멈춘 초침 | 정지 잔향 | 타격이 짧은 둔화/정지감 |
| 잿빛 보호막 | 잿빛 잔향 | 방어/피격 시 반격 파동·짧은 가드 |
| 망각의 낙인 | 낙인 잔향 | 타격이 표식을 남기고 후속 폭딜 증폭 |

> 활성 기억과 잔향은 **화면 형태가 달라야** 한다. 잔향은 무기/몸에 남은 흔적이고, 활성 기억은 독립 객체다.

## 공명 (잔향이 남은 기억 재획득)

잔향이 이미 있는 기억을 다시 얻으면 기억이 공명으로 강화된다.

```text
재획득 기억 시작 레벨 = 기본 제시 레벨 + floor(잔향 레벨 / 2)   (최대 +5)
잔향은 소비하지 않는다.
```

예시:
- `칼무리 잔향 +3` 보유 중 `굶주린 칼무리 +1` 획득 → 기억 `+2`로 시작.
- `칼무리 잔향 +5` 보유 중 `굶주린 칼무리 +1` 획득 → 기억 `+3` 시작 + 공명 보너스.

이유: 다시 얻는 보상은 있되 잃기 전 레벨을 완전 복구하지 않아 망각의 의미가 남고, 잔향을 소비하지 않으므로 "상실이 남긴 힘" 판타지가 유지된다.

### 공명 보너스 (기억+같은 잔향 동시 보유)

- 굶주린 칼무리 + 칼무리 잔향: 칼무리 타격이 잔향 칼날의 온힛 판정으로도 계산.
- 처형자의 섬광 + 처형 잔향: 섬광이 낙인/저체력 적 우선.
- 추적자의 맹세 + 추적 잔향: 유도 투사체가 짧은 잔탄을 남김.
- 파쇄의 파문 + 파문 잔향: 주 파문 뒤 작은 2차 파문.

> Unity 첫 구현은 공명 보너스 전부가 아니라 2~3개만 제대로 만든다.

## 궁극 잔향 (각성 잔향 2개 조합)

`+5` 각성 잔향 2개가 모이면 궁극 잔향이 열린다. 활성 슬롯을 요구하지 않으며, 두 잔향보다 확실히 강하고 후반 빌드 정체성을 책임진다.

| 필요 잔향 (+5 둘) | 궁극 ID | 이름 | 핵심 상호작용 루프 |
| --- | --- | --- | --- |
| 칼무리 + 혈반 | `synergy_blood_blade_storm` | 피의 칼폭풍 | 칼날이 혈반을 묻히고 혈반이 회복 실을 돌려준다 |
| 처형 + 낙인 | `synergy_execution_brand` | 처형 각인 | 낙인 적 처형 시 하얀 폭발과 검은 파편이 연쇄 |
| 추적 + 정지 | `synergy_frozen_hunt` | 정지 추적 | 둔화 적을 추적 잔탄이 우선 공격하고 분열 |
| 파문 + 잿빛 | `synergy_bastion_wave` | 성채 파문 | 방어 반응이 큰 파문과 안전지대를 만든다 |

궁극 수락 기준:
- 두 required 잔향이 +5일 때만 unlock.
- 단순 1+1 강화가 아니라 두 잔향이 서로를 먹여 살리는 루프.
- 텍스트 없이 화면/SFX/피드백으로 "궁극이 켜졌다"가 보인다.
- 회복량/처치/전파는 cap을 가진다(화면 삭제 방지).
- 10초 디버그 쇼케이스에서 화면 변화가 명확하다.

> Unity 첫 구현은 궁극 1개(피의 칼폭풍)만 완성한 뒤 나머지로 확장한다.

## HTML 시너지(태그 기반) — 참고/보조

HTML은 잔향과 별개로 활성 기억 태그 조합 시너지를 갖는다(출처 `synergyRules`). 잔향 시스템과 충돌하지 않으면 보조로 유지, 충돌하면 잔향 우선.

| ID | 이름 | 조건 태그 | 효과 |
| --- | --- | --- | --- |
| `area_control` | 압축 파문 | area+control | 둔화/밀린 적에 광역 피해 증가 |
| `dot_control` | 느린 출혈 | dot+control | 둔화 적에 지속 피해 강화 |
| `burst_survival` | 피 묻은 결의 | burst+survival | 큰 피해 시 짧게 회복 |
| `brand_chain` | 각인 연쇄 | burst+control | 약화/낙인 적에 단발 폭딜 증가 |

## 빌드 판단 루프 (플레이어가 반복하는 질문)

1. 지금 어떤 기억에 의존하나?
2. 어떤 기억을 최고 레벨로 키워 다음 망각 후보로 만들까?
3. 이 기억을 잃으면 어떤 잔향/과부하가 생기나?
4. 다시 얻으면 공명이 얼마나 강해질까?
5. 어떤 두 잔향을 +5로 만들어 궁극을 열까?

## 트리거 패밀리 (Unity 구현용)

| TriggerFamily | 용도 | 대표 잔향 |
| --- | --- | --- |
| `OnWeaponHit` | 무기 타격 후 | 칼무리, 혈반, 추적, 파문, 정지, 낙인 |
| `OnKill` | 처치/처형 후 | 처형, 낙인 전파 |
| `OnDamageTaken` | 피격 후 | 잿빛 |
| `OnShieldBreak` | 보호막 파괴 후 | 잿빛 각성, 성채 파문 |
| `Periodic` | 주기적 독립 | 활성 칼무리, 활성 혈반, 정지장 |

무한 루프 제한: `WeaponHit`은 일반 잔향 발동 가능 / `EchoHit`은 일반 온힛 잔향 재발동 불가 / `UltimateHit`은 궁극 전용 전파·회복만 / 프레임당 전파·처형·회복은 target cap·recursion cap을 가진다.
