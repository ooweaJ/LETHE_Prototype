# LETHE 무기·기억·잔향 전체 구체화 설계서

최종 갱신: 2026-06-10

## 0. 이 문서의 위치

`LETHE_WEAPON_MEMORY_ECHO_SPEC.md`가 "각 잔향이 무엇을 하는가"를 글로 적은 문서라면, 이 문서는 그걸 **레벨 단위·동작 단위·무기별 표현**까지 내려서 바로 구현할 수 있게 만든 설계서다.

활성 기억과 잔향의 화면 형태 차이, 망각 변환 연출, 온힛 루프 제한은 `LETHE_ECHO_FORM_TRANSFORMATION_SPEC.md`를 따른다. Unity 클래스, ScriptableObject, 프리팹, 에셋 구조는 `LETHE_UNITY_ECHO_SYSTEM_PRD.md`를 따른다.

수치와 동작은 `EchoDefinition` / `EchoLevelData` / `EchoBehavior`(앞서 만든 ScriptableObject 구조)와 1:1로 매칭된다. 모든 수치는 튜닝 시작점이며, `LETHE_BALANCE_BASELINE`의 "감각 먼저, 수치 나중" 원칙을 따른다.

## 1. 설계 3원칙

- 잔향은 수치가 아니라 **행동**이다. (`확률 +n%` 금지)
- 레벨은 세 축으로만 자란다: **빈도(얼마나 자주) → 개수(몇 개) → 새 행동(+5에서 동작 자체가 바뀜)**.
- 무기마다 같은 잔향이 **다르게 보여야** 한다.

## 2. 잔향 트리거 패밀리 (가장 중요)

8개 잔향이 안 겹치게 하려면, "무엇으로 발동하는가"와 "무슨 역할인가"를 다르게 둬야 한다. 대부분 온힛이지만 역할을 갈라야 하고, 처형·잿빛은 트리거 자체가 다르다.

| 잔향 | 핵심 동사 | 트리거 | 차별점 |
| --- | --- | --- | --- |
| 칼무리 | 영역 베기 | 온힛 | 플레이어 주변 공간을 점유 |
| 혈반 | 회복 압박 | 온힛 | 지속력(생존) 축 |
| 추적 | 원거리 정리 | 온힛 → 유도 발사 | 내가 안 때린 적을 정리 |
| 파문 | 포위 해제 | 온힛(위치) / 주기 | 넉백·군중 제어 |
| 낙인 | 표식 증폭 | 온힛 → 후속타 | 단일 대상 폭딜 셋업 |
| 정지 | 시간 제어 | 온힛(둔화) | 다른 잔향이 터질 시간을 벌어줌 |
| 처형 | 조건부 폭발 | **처치 / 저체력** | 상시 발동이 아닌 조건 발동 |
| 잿빛 | 피격 반응 | **피격 / 보호막 파괴** | 유일하게 방어 트리거 |

원칙: 온힛 잔향 6개는 각각 "무엇을 스폰하는가"가 달라야 하고(영역/회복/유도/넉백/표식/둔화), 처형과 잿빛은 트리거 패밀리 자체로 구분된다.

## 3. 무기 정체성

| 무기 | 전투 리듬 | 잔향 변환 규칙 |
| --- | --- | --- |
| 절단쌍검 | 빠른 연타, 온힛, 주변 압박 | 잔향을 **작게·자주·연쇄**로 처리. 도장처럼 묻힌다 |
| 장송대검 | 느린 큰 타격, 처형, 범위 제어 | 잔향을 **모아 크게·조건부**로 처리. 균열처럼 남긴다 |

모든 잔향은 아래 5종 구체화에서 쌍검/대검 표현을 각각 가진다. 같은 잔향이라도 쌍검은 "자주 작게", 대검은 "느리게 크게"가 기본 변환 공식이다.

## 4. EchoBehavior 플래그 (확장)

코드의 `[Flags] EchoBehavior`에 아래를 추가한다. 레벨이 오르면 플래그가 **누적**된다.

```text
// 공통/칼무리/혈반 (기존)
DelayedSlash, OrbitingBlade, OnHitLaunch, KillAccelerate,
BleedApply, HealThread, BleedSpread, BloodBurst,

// 처형
ExecuteThreshold,   // 저체력/조건 시 폭발
ChainShatter,       // 처치 시 파편 연쇄

// 추적
Homing,             // 유도 잔탄
ShadowTrail,        // 그림자 분리 추적(각성)

// 파문
Shockwave, Knockback,
FloorRune,          // 주기 바닥 문양(각성)

// 정지
Slow,
TimeCrack,          // 둔화 적 재타격 시 시간 균열(각성)

// 잿빛 (피격 트리거)
ShieldReact,
GuardPulse,         // 보호막 파괴 시 장벽 파동(각성)

// 낙인
Mark, Amplify,
MarkSpread          // 처치 시 표식 전파(각성)
```

전투 코드는 `echo.HasBehavior(level, EchoBehavior.OnHitLaunch)`처럼 플래그로 분기한다. 트리거 패밀리(온힛/처치/피격)는 이벤트 구독 단계에서 갈라서, 같은 시스템이 모든 잔향을 처리하지 않게 한다.

## 5. 기억·잔향 8종 구체화

### 5.1 굶주린 칼무리 → 칼무리 잔향 (영역 베기 / 온힛)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 30%에 지연 칼자국 1 | DelayedSlash |
| +2 | 2타마다 확정 | DelayedSlash |
| +3 | 칼자국이 ~1초 궤도, 2~3명 벰 | DelayedSlash + linger |
| +4 | 칼날 1개 상시 회전 시작 | OrbitingBlade |
| +5 | 고리 2~3개 + 타격마다 칼날 발사 + 처치 가속 | OrbitingBlade·OnHitLaunch·KillAccelerate |

- 쌍검: 고리가 자주 반응하고 발사 칼날이 작고 빠르다.
- 대검: 발사 칼날이 크고 적을 관통한다.
- 각성: 플레이어 주변 칼날 고리 유지, 온힛 발사.
- 공명: 활성 칼날과 잔향 칼날이 같은 온힛 판정 공유.

### 5.2 피의 반사 → 혈반 잔향 (회복 압박 / 온힛)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 혈반 부착 + 낮은 확률 회복 실 | BleedApply·HealThread |
| +2 | 혈반 자주, 약한 출혈 | BleedApply·HealThread |
| +3 | 처치 시 인접 적에 혈반 전파 | +BleedSpread |
| +4 | 회복 실 여러 가닥, 저체력일수록 선명 | spawnCount↑ |
| +5 | 피꽃 폭발 → 광역 출혈 + 회복 다발 | +BloodBurst |

- 쌍검: 빠른 타수로 회복 실이 끊이지 않는다.
- 대검: 1회 폭발의 범위·회복량이 크다.
- 공명: 활성 추가타가 혈반 표식을 우선 터뜨린다.

### 5.3 처형자의 섬광 → 처형 잔향 (조건부 폭발 / 처치·저체력)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 치명/저체력(≤20%) 타격에 하얀 금(25%), 처치 시 작은 섬광 | ExecuteThreshold |
| +2 | 금 확률·섬광 범위↑ | ExecuteThreshold |
| +3 | 임계 ≤30%, 섬광이 인접 1명에 파편 | +ChainShatter |
| +4 | 처형 가능 적 머리 위 하얀 선 표시(가독성), 임계 ≤35% | ExecuteThreshold·ChainShatter |
| +5 | 조건 처치 시 화면 번쩍 + 주변 다수 파편 연쇄 | ExecuteThreshold·ChainShatter(강화) |

- 쌍검: 작은 처형이 낮은 확률로 자주 연쇄.
- 대검: 처형 한 방의 화면 정리력이 크다.
- 각성: 처형 표시 + 연쇄 폭발이 무리를 정리.
- 공명: 섬광이 낙인/저체력/엘리트를 우선 대상.
- 주의: 상시 proc이 아니라 **조건 발동**이라 칼무리·혈반과 체감이 확실히 다르다.

### 5.4 추적자의 맹세 → 추적 잔향 (원거리 정리 / 온힛 → 유도)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 30%에 작은 잔탄 1개가 멀거나 약한 적 추적 | Homing |
| +2 | 50% 또는 2타마다 확정 | Homing |
| +3 | 잔탄 관통 1회 또는 재추적 | Homing |
| +4 | 잔탄 2개 | Homing, spawnCount↑ |
| +5 | 그림자 칼날이 따라다니다 일정 타격마다 분리 추적, 잔탄 3개 | Homing·ShadowTrail |

- 쌍검: 잔탄이 자주 나가지만 작다.
- 대검: 잔탄이 관통하거나 더 오래 따라간다.
- 각성: 플레이어 뒤 그림자 투사체가 원거리 적을 자동 정리.
- 공명: 활성 투사체가 잔탄을 남기고, 잔탄은 둔화/표식 적 우선.

### 5.5 파쇄의 파문 → 파문 잔향 (포위 해제 / 온힛 위치·주기)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 타격 위치 작은 파문(30%), 약한 넉백 | Shockwave·Knockback |
| +2 | 50%, 넉백↑ | Shockwave·Knockback |
| +3 | 반경↑, 2명 이상 밀어냄 | Shockwave·Knockback |
| +4 | 주기적으로 발밑 문양, 다음 타격에 큰 파문 | +FloorRune |
| +5 | 주기 문양 + 큰 파문이 포위를 한 번에 해제, 짧은 안전지대 | Shockwave·Knockback·FloorRune |

- 쌍검: 작은 파문이 자주 생겨 포위를 흐트러뜨린다.
- 대검: 강타 지점에서 큰 파문이 늦게 터진다.
- 각성: 정기적 안전지대 생성(생존 유틸).
- 공명: 활성 충격파 뒤 작은 2차 파문.

### 5.6 멈춘 초침 → 정지 잔향 (시간 제어 / 온힛 둔화)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 타격 시 짧은 둔화(15%, 0.5s, 30%) | Slow |
| +2 | 둔화율·지속↑ | Slow |
| +3 | 둔화장 잔상, 여러 적에 묻힘 | Slow + linger |
| +4 | 둔화 적 재타격 시 추가 피해 | Slow |
| +5 | 둔화 적 재타격 시 시간 균열 → 주변도 둔화, 짧은 정지 | Slow·TimeCrack |

- 쌍검: 여러 적에게 얇게 둔화를 묻힌다.
- 대검: 맞은 적 주변에 느린 시간 장판.
- 각성: 적 무리가 느려진 사이 다른 잔향이 안전하게 터진다(콤보 enabler).
- 공명: 활성 시간 제어가 잔향 둔화 적에게 추가 피해/쿨다운 환급.

### 5.7 잿빛 보호막 → 잿빛 잔향 (피격 반응 / 피격·보호막 파괴)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 피해/보호막 파괴 시 잿빛 파편(약한 반격) | ShieldReact |
| +2 | 파편 범위↑, 피격 후 짧은 공속/반격↑(쌍검) | ShieldReact |
| +3 | 보호막 파괴 시 작은 넉백 파동 | ShieldReact·Knockback |
| +4 | 피격 후 다음 강타에 잿빛 파문 | ShieldReact |
| +5 | 보호막 파괴 시 회색 원형 장벽이 적을 밀어냄(생존 리셋) | ShieldReact·GuardPulse |

- 쌍검: 피격 후 공속/반격 잔향이 짧게 오른다.
- 대검: 피격 후 다음 강타에 잿빛 파문이 붙는다.
- 각성: 위험 순간 보호막 파괴 → 포위 해제.
- 공명: 활성 보호막이 깨질 때 잔향 반격을 확정 발동.
- 주의: **유일한 피격 트리거**. 공격 안 해도 발동하므로 다른 7개와 구현 경로(이벤트 구독)가 다르다.

### 5.8 망각의 낙인 → 낙인 잔향 (표식 증폭 / 온힛 → 후속타)

| 레벨 | 변화 | 동작 |
| --- | --- | --- |
| +1 | 검은 표식(25%), 표식 적 재타격 시 피해 증폭(+15%) | Mark·Amplify |
| +2 | 표식 확률·증폭↑ | Mark·Amplify |
| +3 | 표식 적 처치 시 인접 적에 표식 전파 | +MarkSpread |
| +4 | 표식 폭발(대검) / 빠른 소비(쌍검) | Mark·Amplify·MarkSpread |
| +5 | 강한 적 자동 낙인, 처치 시 검은 파편이 가장 가까운 적에 옮겨감 | Mark·Amplify·MarkSpread(강화) |

- 쌍검: 표식을 빨리 터뜨린다.
- 대검: 표식 폭발이 크다.
- 각성: 엘리트 사이를 표식이 옮겨다니며 폭딜 셋업.
- 공명: 활성 낙인이 처형 잔향/섬광과 우선 연계.

## 6. 궁극 잔향 4종 구체화

각 궁극은 단순 1+1이 아니라 **두 효과가 서로를 먹여 살리는 루프**여야 한다.

### 6.1 피의 칼폭풍 — 칼무리 +5 & 혈반 +5

- 회전 칼날 고리가 적에게 혈반을 묻히고, 혈반이 터지면 회복 실이 돌아온다. 처치가 이어지면 고리가 가속.
- 쌍검: 고리가 자주 반응, 피 실이 빠르게 돌아온다.
- 대검: 강타 때 고리가 크게 확장되어 넓은 피 칼날 폭풍.

### 6.2 처형 각인 — 처형 +5 & 낙인 +5

- 낙인 적이 처형 조건에 들면 하얀 균열 표시. 처형 시 하얀 폭발 + 검은 파편이 동시에 터지고, 파편이 다음 위협 적에게 낙인을 옮긴다.
- 쌍검: 작은 처형 폭발이 자주 연쇄.
- 대검: 처형 한 방의 화면 정리력이 크다.

### 6.3 정지 추적 — 추적 +5 & 정지 +5

- 유도 잔탄이 둔화 적을 우선 추적하고, 둔화 적에게 명중하면 잔탄이 한 번 더 분열한다. 어두운 잔탄 + 보라색 시간 잔상.
- 쌍검: 잔탄 분열이 잦다.
- 대검: 분열 잔탄이 크고 오래 추적.

### 6.4 성채 파문 — 파문 +5 & 잿빛 +5

- 보호막이 있거나 피격 직후 강타하면 큰 성채 파문이 퍼져 적을 밀어내고 잠깐 안전지대를 만든다. **방어 → 공격 전환**이 핵심.
- 쌍검: 파문이 자주, 안전지대 갱신이 빠르다.
- 대검: 파문 1회가 크고 멀리 민다.

## 7. 구현 순서

slice 우선순위와 트리거 패밀리 다양성을 함께 고려한 순서다.

1. **쌍검 + 칼무리 + 혈반 + 피의 칼폭풍** — 이미 데이터 존재. 가장 가독성 좋은 영역/회복 루프로 "뽕맛"을 먼저 증명.
2. **대검 + 처형 + 낙인 + 처형 각인** — 두 번째 대비축. 조건부(처치) 트리거 패밀리를 추가해 잔향이 다양하다는 걸 보여준다.
3. **추적 + 파문** — 원거리 정리와 군중 제어 검증. 온힛이지만 역할이 확실히 다른 두 축.
4. **정지 + 잿빛 + 나머지 궁극** — 정지는 콤보 enabler라 다른 잔향이 갖춰진 뒤 가치가 보이고, 잿빛은 유일한 피격 트리거라 시스템이 가장 다르므로 마지막에 붙인다.

이 순서로 가면 첫 두 단계 안에 **온힛·조건부 두 트리거 패밀리**가 다 들어와서, 적은 콘텐츠로도 "잔향이 단조롭지 않다"는 인상을 빠르게 줄 수 있다.
