# LETHE 런 구조 기획서

최종 갱신: 2026-06-09

## 목적

이 문서는 한 판의 흐름, 성장, 망각, 결손 생존, 기억 보충 구조를 정의한다.

## 기본 목표

한 판은 "강해짐 -> 잃음 -> 잔향으로 변형 -> 다시 강해짐"을 반복해야 한다.

HTML v0.12의 600초 압축 런은 근거 자료로 남기고, Unity 첫 slice는 더 짧은 검증용 루프로 시작한다.

## Unity 첫 slice 권장 길이

- 목표 길이: 6~8분.
- 첫 보스/망각: 2~3분.
- 첫 결손 생존: 45~60초.
- 첫 보충: 결손 생존 종료 직후.
- 첫 각성 잔향 목표: 1회 플레이 안에서 도달 가능하게 조정.

## 단계별 흐름

| 단계 | 목표 | 플레이어 경험 |
| --- | --- | --- |
| 시작 | 무기 1개와 기억 1개 선택 | 즉시 빌드 방향을 느낌 |
| 초반 성장 | 레벨업과 기억 획득 | 2~3개의 능력이 돌아가기 시작 |
| 첫 보스 | 현재 빌드 시험 | 내가 만든 빌드가 통하는지 확인 |
| 첫 망각 | 핵심 기억 상실 | 아쉬움과 불안 |
| 결손 생존 | 2개 이하 기억으로 버티기 | 잃은 능력의 빈자리를 체감 |
| 기억 보충 | 새 기억 또는 잃었던 기억 재획득 | 피벗 또는 공명 선택 |
| 잔향 목표 | `+5` 잔향 또는 궁극 조합 추구 | 장기 빌드 목표 형성 |

## 레벨업 선택

레벨업 선택지는 세 종류를 섞는다.

- 새 기억 획득.
- 활성 기억 강화.
- 런 스탯 강화.

Unity 첫 slice에서는 선택지를 과하게 늘리지 않는다. 기억/잔향 구조가 읽히는 것이 먼저다.

## 망각 규칙

망각은 플레이어가 의존한 기억과 연결되어야 한다.

- 완전 랜덤 삭제처럼 보이면 안 된다.
- 전투 사용량, 피해 기여, 생존 기여, 전술 사용 등을 설명할 수 있어야 한다.
- 결과 화면은 "사라진 기억"과 "남은 잔향"을 함께 보여준다.

## 결손 생존

결손 생존은 단순 약화 구간이 아니라 적응 구간이다.

- 잃은 기억의 빈자리를 느끼게 한다.
- 잔향 효과가 처음으로 보이게 한다.
- 너무 길면 짜증이 되므로 첫 slice는 짧게 둔다.

## 보충 선택

보충은 세 방향을 제공한다.

- 잃었던 기억 재획득: 공명으로 강화.
- 새 기억 획득: 새로운 빌드 방향.
- 기존 기억 강화: 안정적 성장.

## Unity 첫 slice 완료 기준

- 첫 망각이 발생한다.
- 잃은 기억 레벨이 잔향으로 전환된다.
- 잔향이 남은 기억을 다시 얻으면 강화된다.
- 결손 생존 후 보충 선택이 작동한다.
- 플레이어가 현재 기억/잔향 상태를 UI에서 확인할 수 있다.
