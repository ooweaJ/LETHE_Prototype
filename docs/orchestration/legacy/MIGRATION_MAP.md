# Orchestration Migration Map

Last updated: 2026-06-08

This file records where project-management documents moved after adopting `docs/orchestration/` as the shared local AI-orchestration interface.

| Old path | New path | Action | Source of truth after migration |
| --- | --- | --- | --- |
| `docs/devlog/YYYY-MM-DD.md` | `docs/orchestration/devlog/YYYYMMDD.md` | moved and renamed by compact date | new path |
| `docs/reports/YYYY-MM-DD.md` | `docs/orchestration/reports/YYYYMMDD/index.md` | moved into date folder | new path |
| `docs/reports/YYYY-MM-DD.html` | `docs/orchestration/reports/YYYYMMDD/index.html` | moved into date folder, regenerated by `npm run report` | generated new path |
| `docs/reports/units/YYYY-MM-DD/` | `docs/orchestration/reports/YYYYMMDD/units/` | moved into each date folder | generated new path |
| `docs/review_prompts/*.md` | `docs/orchestration/review_prompts/*.md` | moved | new path |
| `docs/review_responses/*.md` | `docs/orchestration/review_responses/*.md` | moved | new path |
| `docs/review_prompts/README.md` | `docs/review_prompts/README.md` | replaced with legacy pointer | no |
| `docs/review_responses/README.md` | `docs/review_responses/README.md` | replaced with legacy pointer | no |
| `docs/NEXT_TASKS.md` | `docs/orchestration/state/NEXT_TASKS.md` | summarized; original remains detailed legacy archive | orchestration state for current work |
| `docs/CODEX_STATUS.md` | `docs/orchestration/state/STATUS.md` | summarized; original remains detailed legacy archive | orchestration state for current work |

Legacy folders may remain if external tooling or historical links reference them. New project-management work should not create source-of-truth files under legacy paths.
