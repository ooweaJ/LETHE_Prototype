# 2026-07-06 Devlog

## Gatekeeper Heal Diagnosis

- Jaewoo reported that the first boss did not seem to die after boss spawn, while normal adds kept spawning during a dual-blades + Hungry Blades run.
- Code inspection found no dedicated Gatekeeper self-heal.
- Actual issue found:
  - `VoidPriest` healed nearby `V1Enemy` targets every 1.4 seconds.
  - The target filter did not exclude `V1EnemyKind.Gatekeeper`.
  - Therefore a priest near the boss could restore boss HP, which matched the playtest feeling.

## Enemy Soft Separation

- Jaewoo asked whether normal enemies are supposed to overlap, and whether Vampire Survivors also allows overlap.
- Direction chosen:
  - keep survivor-style dense crowd pressure,
  - avoid hard collision/body blocking,
  - add a soft visual/gameplay separation nudge so enemies do not collapse into one unreadable stack.
- Implemented `V1GameManager.EnemySeparationForce(V1Enemy self)`.
- Applied separation after normal enemy behavior:
  - Gatekeeper is pushed less to keep boss presence stable.
  - Normal enemies are pushed more strongly away from Gatekeeper.
  - DriftingEye separates even while holding its ranged casting position.
  - exact same-position overlaps use a deterministic fallback direction.
- Added `DebugRunEnemySeparationMatrix()`.
- Added Unity menu `LETHE/V1 QA/Enemy Separation Matrix`.
