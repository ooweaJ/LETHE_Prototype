# 2026-06-16 Devlog

## Pause / hitstop movement fix

- User review:
  - Monsters kept moving while choosing cards.
  - Greatsword/basic attack/Kalmuri echo hitstop made the character feel like movement briefly stopped.
- Diagnosis:
  - `V1GameManager` returned during overlays, but `V1Enemy.Update()` and projectile updates still ran independently.
  - `hitstopTimer` returned before player/camera/weapon visual updates, so player-side responsiveness felt interrupted.
- Implemented:
  - Added `V1GameManager.GameplayPaused` for blocking overlays:
    - start weapon selection.
    - level-up card selection.
    - forgetting result.
    - resonance refill.
    - death overlay.
  - Added `V1GameManager.HitstopActive` for combat impact freeze.
  - `V1Enemy`, `V1Projectile`, and `V1EnemyShot` now early-return when gameplay is paused or hitstop is active.
  - Hitstop now still updates player, camera, and weapon visuals before returning from the manager loop.
- Verification:
  - `dotnet build LETHE/Assembly-CSharp.csproj --nologo`: pass, 7 warnings, 0 errors.
  - Unity compilation errors: `count=0`.

    - `hitstopActive=True`
- Reporting:
