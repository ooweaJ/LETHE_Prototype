# 2026-06-08-06 - 프로젝트 대시보드 압축

## 1. 현재 빌드 상태

- 메인 대시보드는 짧은 상태 요약판으로 정리됐다.

## 2. 오늘 바뀐 것

- 긴 프롬프트와 상세 설명을 대시보드에서 제거했다.
- 다음 지시와 운영 절차는 별도 화면이 맡도록 했다.

## 3. 테스트 결과와 근거

- `npm run report`: 통과.
- `npm run report:check`: 통과.
- `npm run report:discord:unit:dry`: 통과.
- `npm run doctor`: 통과.

## 4. 결정한 것

- 대시보드는 요약, command 화면은 지시, runbook 화면은 절차, reports는 상세 설명을 맡는다.

## 5. 문제 또는 리스크

- 보고서 목록 구조가 아직 완전히 정규화되지 않았다.

## 6. GPT/Claude 인계 요약

메인 화면은 길게 설명하지 않고 현재 판단에 필요한 핵심만 보여줘야 한다.

## 7. 다음 Codex 작업

- 보고서가 너무 많아질 때 날짜별로 볼 수 있게 구조화한다.

## 8. 포트폴리오 메모

- 문제: 한 화면에 너무 많은 역할이 섞였다.
- 방향: 화면별 책임을 줄인다.
- 행동: 대시보드를 요약 중심으로 압축했다.
- 결과: 사용자가 현재 판단만 빠르게 볼 수 있다.
