# 2026-06-08-12 - 개발 문서 플러그인 최신 기준 정렬

## 1. 현재 빌드 상태

- 게임 빌드와 밸런스 코드는 변경하지 않았다.
- 현재 플레이/검증 상태는 여전히 v0.12 `ITERATE_BALANCE` 이후의 밸런스 리뷰 단계다.
- 이번 작업은 `docs/orchestration/MIGRATION_PROMPT.md`의 최신 development-docs plugin 기준을 문서 구조에 적용한 것이다.

## 2. 오늘 바뀐 것

- `AGENTS.md`의 오케스트레이션 섹션을 `Development Docs Plugin` 기준으로 정리했다.
- `docs/orchestration/templates/HTML_INTERFACE_TEMPLATE.md`를 추가했다.
- `docs/orchestration/README.md`, `state/RUNBOOK.md`, `state/PROMPT_CONTEXT.md`, `state/NEXT_TASKS.md`, `state/DECISION_LOG.md`, `legacy/MIGRATION_MAP.md`를 새 기준에 맞췄다.
- `docs/review_prompts/README.md`와 `docs/review_responses/README.md`의 깨진 legacy pointer 문장을 교체했다.
- `docs/DISCORD_REPORTING.md`를 Project Orchestrator 중앙 intake 우선, 로컬 Discord 스크립트는 trusted fallback 기준으로 바꿨다.

## 3. 테스트 결과와 근거

- `npm run report`: 통과, 12개 단위 리포트 생성.
- `npm run report:check`: 순서 정리 전 1회 실패. 원인은 12번 단위를 파일 상단에 넣어 번호 순서가 깨진 것이었다.
- `npm run report:check`: 순서 정리 후 통과, 12개 단위 확인.
- 커밋은 사용자가 금지했으므로 하지 않는다.

## 4. 결정한 것

- `docs/orchestration/`은 LETHE 전용 임시 문서 묶음이 아니라 shared personal development-docs plugin으로 본다.
- 사람은 HTML을 보고, AI는 `state/`와 `devlog/` Markdown을 우선 읽는다.
- Discord 실제 전송은 Project Orchestrator가 우선 담당한다.
- Project Orchestrator intake가 아직 없을 때만 이 저장소의 직접 Discord 전송 스크립트를 trusted-local fallback으로 사용한다.

## 5. 문제 또는 리스크

- Project Orchestrator intake 명령/API는 이 저장소 안에 아직 없다.
- `package.json`과 `scripts/send_discord_report.js`의 직접 전송 기능은 남아 있으므로, 문서상 기본 정책과 실행 fallback을 구분해야 한다.
- 과거 보고서 본문에는 이전 경로와 이전 Discord 정책이 역사 기록으로 남아 있다.

## 6. GPT/Claude 인계 요약

최신 마이그레이션 프롬프트 기준은 대부분 적용됐다. 다음 검토자는 `docs/orchestration/MIGRATION_PROMPT.md`, `AGENTS.md`, `docs/orchestration/templates/HTML_INTERFACE_TEMPLATE.md`, `docs/orchestration/legacy/MIGRATION_MAP.md`를 기준으로 보면 된다.

## 7. 다음 Codex 작업

- Project Orchestrator intake가 준비되면 `npm run report:discord:unit` 직접 전송을 중앙 intake 제출로 대체한다.
- 이후 원래 작업인 v0.12 밸런스 실패 리뷰로 돌아간다.

## 8. 포트폴리오 메모

- 문제: 새 plugin 기준 문서는 있었지만 실제 LETHE 문서 구조와 일부 정책이 덜 맞았다.
- 방향: 기존 LETHE 규칙을 보존하면서 공통 development-docs plugin 규약만 정렬한다.
- 행동: 규칙, 템플릿, legacy map, Discord 정책, pointer README를 정리했다.
- 결과: 다음 프로젝트에도 복사 가능한 문서 플러그인 기준에 더 가까워졌다.
