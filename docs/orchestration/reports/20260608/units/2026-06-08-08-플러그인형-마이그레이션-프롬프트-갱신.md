# 2026-06-08-08 - 플러그인형 마이그레이션 프롬프트 갱신

## 1. 현재 빌드 상태

- 다른 프로젝트에 적용할 공통 마이그레이션 프롬프트를 갱신했다.

## 2. 오늘 바뀐 것

- `interface/`, `state/`, `reports/`, `devlog/`, `legacy/` 역할을 정의했다.
- 사람은 HTML을 보고, AI는 Markdown을 읽는 방향을 명시했다.

## 3. 테스트 결과와 근거

- `npm run report`: 통과.
- `npm run report:check`: 통과.
- `npm run report:discord:unit:dry`: 통과.
- `npm run doctor`: 통과.

## 4. 결정한 것

- 오케스트레이션은 단순 문서 묶음이 아니라 재사용 가능한 개인 AI 플러그인 구조로 다룬다.

## 5. 문제 또는 리스크

- 이 시점에는 프롬프트만 바뀌었고 LETHE 실제 폴더는 아직 완전히 옮겨지지 않았다.

## 6. GPT/Claude 인계 요약

다음 단계는 규약을 문서로만 두지 말고 실제 LETHE 폴더 구조에 적용하는 것이다.

## 7. 다음 Codex 작업

- 기존 `docs/reports/`와 `docs/devlog/`를 오케스트레이션 내부로 옮긴다.

## 8. 포트폴리오 메모

- 문제: 여러 프로젝트에 적용할 구조가 아직 말뿐이었다.

- 행동: 마이그레이션 프롬프트를 갱신했다.
- 결과: 플러그인형 오케스트레이션의 기준이 생겼다.
