오케스트레이션 로컬 보고서

현재 빌드 상태

LETHE v0.12는 controlled human-test candidate입니다. 자동 밸런스 기준은 통과했지만, Unity 전환이나 다음 튜닝 판단에는 아직 사람 반응 증거가 필요합니다.

오늘 바꾼 것

  • docs/orchestration/index.html을 한글 보고형 대시보드로 정규화했습니다.
  • 보고서 목록과 개발로그 목록을 오케스트레이션에서 직접 열 수 있게 연결했습니다.
  • Codex 앱과의 차별점인 “프로젝트 폴더에서 개발 문서를 직접 보는 구조”를 대시보드에 명시했습니다.

검증과 증거

  • npm run report: pass
  • npm run report:check: pass
  • npm run report:discord:unit:dry: pass
  • npm run doctor: 50 pass / 0 warn / 0 fail

결정

오케스트레이션 HTML은 단순 문서 덤프가 아니라 사람이 다음 프롬프트를 결정하기 위한 프로젝트 인터페이스로 다룹니다. Markdown은 계속 source of truth입니다.

문제와 리스크

  • 실제 Discord 전송은 외부 webhook 정책으로 차단되었습니다.
  • HTML은 아직 수동 유지입니다. 나중에 Markdown 기반 생성기가 필요합니다.
  • 사람 플레이테스트 로그가 없어서 다음 제품 방향 판단은 보류입니다.

다음 Codex 작업

  • 사람 테스트 JSON 로그를 playtest_logs/에 모읍니다.
  • npm run playtest:summary를 실행합니다.
  • 요약 결과를 STATUS, CURRENT_TASK, NEXT_TASKS, devlog, reports, DECISION_LOG에 반영합니다.

포트폴리오 메모

문제: Codex 대화만으로는 프로젝트 상태와 개발 기록을 장기적으로 보기 어렵습니다. 방향: 프로젝트 폴더 안의 오케스트레이션 HTML을 사람용 개발 문서 인터페이스로 만듭니다. 결과: 사용자가 대시보드와 보고서 목록을 보고 다음 프롬프트를 직접 결정할 수 있습니다.
