# 2026-06-10-13 - Unity 런타임 기반 스크립트

## 1. 현재 빌드 상태

Unity `_dev`에는 기본 sprite와 폴더 구조가 있고, 이번 작업으로 첫 slice 런타임 골격이 들어갔다. 아직 scene/prefab 조립은 하지 않았고, 목표는 이후 기본 전투와 잔향 디버그 루프가 붙을 수 있는 컴파일 가능한 C# 기반을 만드는 것이다.

## 2. 오늘 바뀐 것

- `WeaponDefinition`, `MemoryDefinition`, `EchoDefinition`, `EchoSynergyDefinition`, `FeedbackProfile` ScriptableObject 정의를 추가했다.
- `EchoLevelData`, `TriggerFamily`, `HitSourceType`, `EchoBehavior`, `EchoStyle`을 추가했다.
- `IWeaponRuntime`, `IActiveMemoryRuntime`, `IEchoRuntime`, `IUltimateEchoRuntime`, `IPooledObject` 계약을 추가했다.
- Unity 직렬화/컴포넌트 연결용 abstract base class를 추가했다.
- `RunBuildState`에 active memory level, echo level, synergy unlock, highest memory lookup, echo +5 cap/overflow 처리를 넣었다.
- `HitEvent`, `KillEvent`, `DamageTakenEvent`, `HitResolver`, `EchoTriggerRouter`, `EchoProcLimiter`를 추가했다.
- `Health`, `WeaponHitEmitter`, `PoolService`, `PooledBehaviour`, `FeedbackService`, `SpriteFlashReceiver`를 추가했다.

- 아직 ScriptableObject asset 자체와 prefab 연결은 없다.
- Project Orchestrator API 보고는 endpoint 미응답으로 제출되지 않았다.
