1. 현재 빌드 상태

Dev_Prototype_v0는 이제 쌍검 1종/기억 2종이 아니라, 압축 디버그 루프로 쌍검 + 대검 + 8기억 + 8잔향 + 4궁극을 모두 켜고 확인할 수 있는 상태다.

  • EnemyDefinition, RewardPoolDefinition, WeaponRhythm, 추가 EchoBehavior 플래그를 만들었다.
  • MemoryInventory, EchoInventory, ForgetService, ResonanceService, UltimateEchoService, RewardService, DebugStateInjector를 추가했다.
  • PrototypeWeaponController가 쌍검/대검을 전환하게 했다.
  • PrototypeEnemySpawner가 근접, 원거리, 분열, 엘리트 역할 id를 가진 적을 생성하게 했다.
  • PrototypeGameManager를 complete prototype 카탈로그 기반으로 교체했다.
  • 8기억, 8잔향, 4궁극 잔향을 F1-F8과 debug 버튼으로 강제 테스트할 수 있게 했다.
  • 전투 중 kill/forget이 Dictionary 순회를 깨던 예외를 snapshot 순회로 고쳤다.

4. 결정한 것

  • 완성형 기준을 잡기 위해 모든 기억/잔향/궁극을 먼저 한 씬에서 호출 가능하게 만든다.
  • 지금 단계에서는 나머지 6개 기억/잔향이 전용 sprite를 갖지 않아도 된다.
  • 단, 역할과 행동 차이는 반드시 보여야 하므로 색, 반경, 타이밍, 타겟팅, 회복/처형/파문/낙인 로직을 분리했다.

5. 문제 또는 리스크

  • PrototypeGameManager에는 아직 switch 기반 effect routing이 많다.
  • 데이터는 먼저 code catalog로 들어갔고, 아직 _dev/Data ScriptableObject asset으로 완전히 빠지지 않았다.
  • 처형, 추적, 파문, 정지, 잿빛, 낙인은 전용 VFX가 아니라 기존 칼무리/혈반 sprite를 재사용한다.
  • 밸런스는 아직 GO 기준이 아니라 기능 smoke 기준이다.

6. GPT/Claude 인계 요약

새 PRD 기준의 첫 구현이 들어갔다. Unity에서 Play 후 F6으로 대검 전환, F7로 8기억, F8로 8잔향+4궁극을 즉시 켤 수 있다. 다음 리뷰는 어떤 기억/잔향/궁극이 실제로 방향성이 보이고, 어떤 것이 아직 색만 다른 placeholder처럼 보이는지 판단하면 된다.
