## 4. 결정한 것

- 완성형 기준을 잡기 위해 모든 기억/잔향/궁극을 먼저 한 씬에서 호출 가능하게 만든다.
- 지금 단계에서는 나머지 6개 기억/잔향이 전용 sprite를 갖지 않아도 된다.
- 단, 역할과 행동 차이는 반드시 보여야 하므로 색, 반경, 타이밍, 타겟팅, 회복/처형/파문/낙인 로직을 분리했다.

## 5. 문제 또는 리스크

- `PrototypeGameManager`에는 아직 switch 기반 effect routing이 많다.
- 데이터는 먼저 code catalog로 들어갔고, 아직 `_dev/Data` ScriptableObject asset으로 완전히 빠지지 않았다.
- 처형, 추적, 파문, 정지, 잿빛, 낙인은 전용 VFX가 아니라 기존 칼무리/혈반 sprite를 재사용한다.
- 밸런스는 아직 GO 기준이 아니라 기능 smoke 기준이다.

## 6. GPT/Claude 인계 요약
