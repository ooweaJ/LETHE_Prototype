2026-06-24-07 - VFX / 공격 범위 전체 확대

1. 현재 빌드 상태

Dev_Prototype_v1은 무기별 베기 VFX와 8잔향 VFX가 이미 연결된 상태였지만, 실제 플레이 피드백상 화면에서 작게 읽히고 공격 범위도 좁게 느껴질 가능성이 있었다. 이번 작업은 가시성 문제를 단순 색/시간만이 아니라 실제 VFX 크기와 타격 범위 자체를 키워 해결하는 방향이다.

2. 오늘 바뀐 것

  • 공통 전투 VFX 배율 CombatVfxVisibilityScale = 1.18f를 추가했다.
  • transient 전투 VFX, 베기 잔상, 시계 바늘, 타격 스파크, 잔향 아이콘형 VFX가 공통으로 더 크게 나온다.
  • 쌍검 공격 범위를 키웠다:
  • range 2.35 -> 2.8
  • arc 119 -> 132
  • max targets 6 -> 7
  • echo size scale 0.8 -> 1.05
  • 대검 공격 범위를 키웠다:
  • range 3.15 -> 3.75
  • arc 82 -> 96
  • max targets 5 -> 6
  • echo size scale 1.8 -> 2.15
  • 쌍검/대검 VFX 프로필에서 반달 베기, 컷 플래시, 히트 스파크, 칼무리 후속타, 대검 피의 칼폭풍 베기 스케일을 키웠다.
  • 칼무리, 파쇄의 파문, 멈춘 1초 계열의 반경도 같이 키워 보이는 범위와 실제 기능 범위가 덜 따로 놀게 했다.

3. 테스트 결과와 근거

  • dotnet build LETHE/Assembly-CSharp.csproj --nologo: 통과, warning 0개, error 0개.
  • Unity Assets/Refresh: 성공.
  • Unity compile error count: 0.
  • Unity Play Mode 진입: isPlaying=true 확인.
  • Unity console에는 MCP 서버 정보 로그만 있었고 게임 런타임 오류/경고는 없었다.

4. 결정한 것

이번에는 VFX를 조금만 다듬는 방식보다 전투 전체의 판독 크기 기준을 올렸다. LETHE의 목표가 시원시원한 뽕맛이라면, 작은 이펙트를 여러 번 보강하는 것보다 기본 전투 스케일 자체가 한 단계 커져야 한다.

5. 문제 또는 리스크

이제 남은 리스크는 반대 방향이다. 이전에는 안 보이는 것이 문제였다면, 지금은 여러 잔향이 동시에 터질 때 너무 복잡하거나 배경을 덮을 수 있다. 특히 Echo All과 대검 조합은 직접 플레이로 과밀도를 봐야 한다.
