2026-06-24-01 - 출시 준비형 맵/배경 1차 확장

1. 현재 빌드 상태

Dev_Prototype_v1은 계속 _dev 검증 빌드다. 다만 이번 작업부터는 작은 테스트 박스 느낌을 줄이는 쪽으로 방향을 바꿨고, 맵/배경을 실제 출시 준비용 월드 베이스에 가깝게 키우기 시작했다.

2. 오늘 바뀐 것

  • 새 맵 스프라이트를 추가했다:
  • tile_lethe_stone_01.png
  • tile_lethe_stone_02.png
  • tile_lethe_stone_03.png
  • tile_lethe_stone_04.png
  • spr_lethe_arena_backdrop_01.png
  • 같은 이미지를 다시 만들 수 있게 scripts/generate_world_sprites.ps1를 추가했다.
  • 플레이어 이동 범위를 기존 x +/-12, y -8.5..8.5에서 x +/-24, y +/-16으로 키웠다.
  • 바닥 타일 배치도 11x9에서 21x15로 늘렸다.
  • 카메라 크기를 6.1에서 6.8로 키우고, 커진 맵 안에서만 따라가도록 클램프했다.
  • 적 스폰 반경을 늘려서 커진 맵에서 바로 몸에 붙는 느낌을 줄였다.
  • 배경, 경계선, 균열, 외곽 표식을 더 큰 맵 기준으로 다시 배치했다.

3. 테스트 결과와 근거

  • 새 타일과 배경 PNG를 로컬에서 직접 열어 비어 있지 않고 어두운 전투 배경으로 읽히는지 확인했다.
  • dotnet build LETHE/Assembly-CSharp.csproj --nologo: 통과, legacy warning 7개, error 0개.
  • Unity Assets/Refresh: 성공.
  • Unity compile error count: 0.
  • Unity Play Mode 진입: isPlaying=true 확인.
  • Unity console error count: 0.

4. 결정한 것

이제 배경/맵 작업은 단순한 임시 장식이 아니라 출시 준비형 월드 베이스로 본다. 먼저 맵 크기와 카메라 감각을 잡고, 그 다음에 배경 스프라이트를 더 강한 아트 방향으로 교체하는 순서가 맞다.

5. 문제 또는 리스크

이번 배경은 직접 제작한 1차 생성 이미지라 최종 아트는 아니다. 맵이 커진 만큼 전투가 시원해질 수 있지만, 반대로 초반 180초가 더 루즈하게 느껴질 수도 있다. 직접 플레이로 크기, 밀도, 이동 시간, 적 접근감까지 봐야 한다.

6. GPT/Claude 인계 요약

Codex가 Dev_Prototype_v1의 월드 크기를 약 두 배로 확장하고, 새 바닥/배경 스프라이트를 연결했다. 다음 판단은 "이 크기가 출시 준비용 기본 전투장으로 맞는가"이며, 맞다면 배경은 생성형 1차 이미지에서 더 의도된 랜드마크/경계/테마 아트로 업그레이드하면 된다.

7. 다음 Codex 작업

1. jaewoo가 대검/쌍검으로 커진 맵을 직접 플레이한다.

2. 맵 크기가 맞으면 배경 스프라이트 2차 제작으로 넘어간다.

3. 맵이 너무 넓거나 루즈하면 적 스폰 밀도, 카메라 크기, 이동 속도, 보스 타이밍을 같이 조정한다.

4. 큰 맵에서 적/보스 스프라이트가 충분히 읽히는지 다시 확인한다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 프로토타입 맵이 작고 배경이 임시 장식처럼 보여 실제 출시 준비 단계로 보기 어려웠다.
  • 방향: 먼저 플레이 공간과 카메라 기준을 키워 실제 게임 공간의 기본 감각을 잡는다.
  • 행동: 새 맵 스프라이트를 만들고, 플레이 범위/타일 수/카메라/스폰 반경을 함께 확장했다.
  • 결과: 기술 검증상 빌드와 Unity Play Mode가 통과했고, 이제 직접 플레이로 맵 크기와 배경 방향을 판단할 수 있다.
