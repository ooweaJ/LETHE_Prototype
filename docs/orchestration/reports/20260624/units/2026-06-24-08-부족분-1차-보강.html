1. 현재 빌드 상태

VFX와 공격 범위는 커졌지만, 그 다음으로 부족한 부분이 남아 있었다. 크게 보면 초반 목표감, 넓어진 맵의 방향감, 적 역할 판독성, 그리고 커진 VFX가 너무 복잡해질 위험이다. 이번 작업은 직접 플레이가 필요한 VFX 감각 판단을 제외하고, Codex가 바로 고칠 수 있는 구조적 부족분을 먼저 보강했다.

2. 오늘 바뀐 것

  • 현재 부족한 부분을 정리했다:
  • 커진 VFX가 충분히 시원한지 혹은 과한지 직접 리뷰 필요.
  • 첫 보스 180초가 루즈할 수 있음.
  • 넓어진 맵에 방향을 잡을 랜드마크가 부족함.
  • 적/보스 스프라이트가 VFX 속에서 역할별로 묻힐 수 있음.
  • 여러 잔향이 동시에 터질 때 화면이 복잡해질 수 있음.
  • 첫 문지기 타이밍을 180초 -> 150초로 앞당겼다.
  • 전체 보스 스케줄을 150 / 300 / 450 / 600초로 조정했다.
  • 첫 문지기 HP를 2050 -> 1750으로 낮춰, 빨리 나오지만 너무 오래 막히지 않게 했다.
  • 문지기 등장 18초 전 붉은/금색 경고 VFX와 문지기 접근 콜아웃을 추가했다.
  • 넓은 맵에 기억석 랜드마크 5개를 추가했다.
  • 원거리 눈, 분열체, 공허 사제, 문지기에게 낮은 알파의 역할 표식을 추가했다.

3. 테스트 결과와 근거

  • dotnet build LETHE/Assembly-CSharp.csproj --nologo: 통과, warning 0개, error 0개.
  • Unity Assets/Refresh: 성공.
  • Unity compile error count: 0.
  • Unity Play Mode 진입: isPlaying=true 확인.
  • Unity console에는 MCP 서버 정보 로그만 있었고 게임 런타임 오류/경고는 없었다.

4. 결정한 것

초반 3분을 기다리는 구조는 현재 개발 단계에서는 판단이 너무 늦어진다. 첫 보스는 150초에 만나게 하고, 대신 HP를 줄여 "목표가 생긴다" 쪽으로 가져간다. 맵과 적 판독은 새 스프라이트를 더 만들기 전에 낮은 비용의 시각 표식으로 먼저 검증한다.

5. 문제 또는 리스크

문지기가 150초에 나오는 것이 너무 빠르게 느껴질 수 있다. 역할 표식도 도움이 되면 좋지만, VFX가 커진 상태에서는 표식까지 더해져 화면이 복잡해질 수 있다. 이건 직접 플레이에서 바로 판정해야 한다.

6. GPT/Claude 인계 요약

Codex가 부족분을 다섯 축으로 정리하고, 그중 초반 목표감/맵 방향감/적 판독성을 먼저 보강했다. 다음 판단 기준은 "첫 보스가 더 재밌어졌는가", "맵이 덜 비어 보이는가", "적 역할 표식이 도움이 되는가"다.
