2026-07-10-05 - 무기별 잔향 VFX 런타임 차별화

1. 현재 빌드 상태

Dev_Prototype_v1에 무기별 잔향 VFX 차별화 1차 구현을 적용했다. 설계 보드에서 정한 쌍검/대검 차이를 실제 런타임 VFX와 판정 감각으로 옮긴 상태다.

2. 오늘 바뀐 것

  • 쌍검 계열은 작은 다단, 체인, 표식, 빠른 점멸이 더 보이도록 보강했다.
  • 대검 계열은 큰 반달, 전방 균열, 단두대, 시계장, 방벽, 붕괴 구덩이가 더 보이도록 보강했다.
  • 주요 변경:
  • 혈반: 쌍검은 혈실/봉합 바늘, 대검은 더 큰 이중 혈월.
  • 파문: 쌍검은 작은 파문 체인, 대검은 전방 지면 균열.
  • 처형: 쌍검은 처형 표식 연쇄, 대검은 떨어지는 단두대 판결.
  • 추적: 쌍검은 짧은 추적칼 부채.
  • 정지: 대검은 시계바늘 판결 참격.
  • 잿빛: 쌍검은 패링 불꽃, 대검은 전방 방벽과 방출 원뿔.
  • 낙인: 쌍검은 낙인 전이 룬, 대검은 붕괴 구덩이.

3. 테스트 결과와 근거

  • dotnet build LETHE/Assembly-CSharp.csproj --nologo: 성공, 기존 경고 7개, 오류 0개.
  • Unity compilation errors: 0.
  • Unity console errors: 0.
  • MCP의 LETHE/V1 QA/Echo Matrix Dual Blades 메뉴 실행은 세 번 시도했지만 Error polling queue: fetch failed로 실패했다.

4. 결정한 것

  • 잔향의 기본 테마는 유지하되 인게임 VFX는 무기별로 나눈다.
  • 쌍검은 “작은 효과를 여러 번 빠르게”, 대검은 “큰 효과를 적게, 늦게, 넓게”를 기준으로 한다.

5. 문제 또는 리스크

  • 자동 QA 메뉴를 끝까지 돌리지 못했기 때문에 직접 플레이 판단이 더 중요하다.
  • 쌍검은 VFX를 늘리면 다시 렉/가시성 문제가 생길 수 있다.
  • 대검은 VFX를 크게 만들수록 몹과 보스 판독을 가릴 수 있다.

6. GPT/Claude 인계 요약

무기별 잔향 차별화는 단순 크기 조절이 아니라 발동 방식, 실루엣, 판정 언어를 분리하는 방향으로 갔다. 다음 판단은 수치보다 감각이다.

7. 다음 Codex 작업

  • jaewoo 직접 플레이 피드백을 기준으로 약한 잔향을 하나씩 골라 조정한다.
  • 우선 확인 순서: 대검 혈반, 대검 처형, 대검 잿빛, 쌍검 혈반/처형/낙인, 쌍검 밀집전 프레임 저하.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 잔향이 무기와 무관하게 비슷해 보여 빌드 선택의 의미가 약했다.
  • 방향: 무기별 전투 리듬을 잔향 VFX와 판정에 직접 반영한다.
  • 행동: 같은 잔향이라도 쌍검은 체인/다단, 대검은 큰 범위/지연 강타로 구현했다.
  • 결과: 플레이어가 같은 잔향을 다른 무기에서 써도 다른 빌드처럼 느낄 수 있는 평가판이 생겼다.
