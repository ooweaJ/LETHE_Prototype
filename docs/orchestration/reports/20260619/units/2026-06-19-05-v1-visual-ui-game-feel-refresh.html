2026-06-19-05 - v1 Visual/UI/game-feel refresh

1. 현재 빌드 상태

Dev_Prototype_v1은 여전히 _dev 검증 빌드다. 이번 작업은 새 시스템 추가가 아니라, 첫 화면/이동/무기/지형/HUD가 더 게임답게 읽히도록 하는 시각 정리 패스다.

2. 오늘 바뀐 것

  • 플레이어 본체의 V1BillboardPulse를 제거해 이동할 때 몸이 좌우로 흔들리는 느낌을 줄였다.
  • 플레이어 스프라이트를 PlayerVisual 자식 오브젝트로 분리해 루트 이동과 시각 애니메이션을 분리했다.
  • 새 플레이어 시트 sheet_player_v1_4dir.png를 생성/임포트했다.
  • 새 8x4 플레이어 시트를 실제 4방향 idle/walk 애니메이션으로 사용한다.
  • 무기 앵커를 중앙 하단으로 옮겨 무기 움직임 때문에 캐릭터 위치가 밀려 보이는 문제를 줄였다.
  • 새 대검 스프라이트를 생성/임포트하고 런타임에서 우선 로드한다.
  • 지형 타일에 회전, 색, 크기 변화를 줘 평평한 반복 격자 느낌을 줄였다.
  • HUD를 HP/XP/기억 슬롯/궁극 상태 중심의 작은 패널로 정리했다.

4. 결정한 것

뱀서류의 핵심 참고점은 특정 UI/아트를 복제하는 것이 아니라, 안정적인 캐릭터 위치, 즉시 읽히는 방향 애니메이션, 반복 플레이에 부담 없는 HUD 밀도, 전장 바닥의 판독성으로 잡았다.

5. 문제 또는 리스크

자동 카메라 스크린샷은 단색 이미지로 저장되어 증거에서 제외했다. 실제 시각 만족도는 Unity Play Mode에서 직접 봐야 한다.

6. GPT/Claude 인계 요약

Codex가 v1의 기본 시각 셸을 정리했다. 플레이어 본체 흔들림을 제거하고 새 4방향 애니메이션 시트, 대검 스프라이트, 지형 변화, 컴팩트 HUD를 적용했다. 기술 검증은 통과했지만, 최종 판정은 jaewoo 직접 플레이 리뷰가 필요하다.

7. 다음 Codex 작업

직접 플레이 후 가장 거슬리는 축 하나만 고른다: 캐릭터 애니메이션 프레임/방향, 시작 HUD, 전투 VFX 밀도, 지형 밝기, 또는 보상 카드 UI.

8. 포트폴리오 메모

  • 문제: 기능은 늘었지만 화면 첫인상과 이동감이 아직 프로토타입처럼 보였다.
  • 방향: 생존 액션 게임의 기본 판독성부터 안정화한다.
  • 행동: 캐릭터 루트/비주얼 분리, 새 4방향 캐릭터 시트, 대검 스프라이트, 지형 변주, HUD 압축을 적용했다.
  • 결과: 기술 검증은 통과했고, 이제 사람 눈으로 시각 품질을 판단할 수 있는 상태가 됐다.
