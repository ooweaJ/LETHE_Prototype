# 2026-06-19-04 - 시작 UI 무기 선택 정정

## 1. 현재 빌드 상태

`Dev_Prototype_v1`의 시작 화면은 다시 무기 2종 선택 구조가 되었습니다. 첫 화면에서는 `절단쌍검` 또는 `장송대검`만 고르고, `굶주린 칼무리`와 `피의 반사`는 첫 레벨업 보상에서 선택합니다.

## 2. 오늘 바뀐 것

- 시작 화면의 4개 `무기 + 기억` 빌드 카드를 제거했습니다.
- 시작 화면을 2개 무기 카드로 줄였습니다.
- `BeginRun(V1WeaponId)`가 더 이상 시작 기억을 자동 지급하지 않습니다.
- Unity 스모크 메뉴도 `Start Dual Blades`, `Start Greatsword`, `M2 Loop` 기준으로 정리했습니다.

## 3. 테스트 결과와 근거

- `dotnet build LETHE/Assembly-CSharp.csproj --nologo`: 통과, legacy warning 7개, error 0개.
- Unity compile error count: 0.
- `Start Dual Blades` 스모크: weapon `절단쌍검`, memories `[]`, death `False`.
- 강제 첫 레벨업 선택지: `굶주린 칼무리 | 피의 반사 | 칼날 가속`.
- Unity console error count: 0.

## 4. 결정한 것

첫 화면은 빌드 선택이 아니라 무기 선택으로 둡니다. 기억 선택은 첫 보상 카드가 담당하게 해서, 플레이어가 “왜 혈반이 무기에 붙어 있지?”라고 느끼지 않게 합니다.

## 5. 문제 또는 리스크

시작 직후 기억이 없으므로 첫 보상이 너무 늦게 느껴지면 다시 보상 속도를 봐야 합니다. 현재 XP 보정상 첫 보상은 빠르게 뜨는 편입니다.

## 6. GPT/Claude 인계 요약

Codex가 시작 UI를 4개 빌드 카드에서 2개 무기 카드로 되돌렸다. 칼무리/혈반은 첫 보상에서 선택되며, 첫 레벨업 카드에 두 기억이 정상 노출되는 것을 확인했다.

## 7. 다음 Codex 작업

- 직접 플레이로 첫 보상까지의 시간이 자연스러운지 확인합니다.
- 너무 밋밋하면 시작 적/XP 리듬만 미세 조정합니다.
- 무기 카드의 시각 디자인은 이후 아트/VFX 패스에서 정리합니다.

## 8. 포트폴리오 메모

- 문제: 초반 재미를 빨리 보여주려다 첫 선택이 과도한 빌드 선택처럼 보였다.
- 방향: 첫 선택은 단순하게, 빌드 정체성은 보상 루프에서 열리게 한다.
- 행동: 시작 UI를 무기 2장으로 정리하고 기억 지급을 첫 레벨업으로 이동했다.
- 결과: 시작 선택의 의미가 명확해지고, 첫 보상이 자연스럽게 빌드 방향을 정하게 됐다.
