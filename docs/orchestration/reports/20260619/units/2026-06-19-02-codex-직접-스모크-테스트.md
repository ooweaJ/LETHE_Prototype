# 2026-06-19-02 - Codex 직접 스모크 테스트

## 1. 현재 빌드 상태

`Dev_Prototype_v1`은 120초 초반 루프 5패스가 적용된 상태이며, 이번에는 Codex가 Unity Editor 메뉴 기반 스모크 테스트를 직접 추가하고 실행했습니다.

## 2. 오늘 바뀐 것

- `LETHE/V1 Smoke/*` 에디터 메뉴를 추가했습니다.
- 메뉴에서 네 가지 시작 빌드와 M2 루프를 자동으로 Play Mode에서 실행하고 `DebugSnapshot()` 결과를 콘솔에 남기게 했습니다.
- Unity 도메인 리로드 후에도 테스트 요청이 사라지지 않도록 `SessionState`를 사용했습니다.

## 3. 테스트 결과와 근거

- `dotnet build LETHE/Assembly-CSharp.csproj --nologo`: 통과, warning 0, error 0.
- Unity compile error count: 0.
- 시작 빌드 4종 스모크: 모두 지정한 무기와 시작 기억으로 초기화됨.
- M2 스모크: `HungryBlades:5`, `BloodReflection:5`, `storm=True`, enemies `10`, death `False`.
- Unity console error count: 0.
- Unity missing references: scene 0, assets 0.

## 4. 결정한 것

수동 클릭이 어려운 Unity MCP 환경에서는 에디터 메뉴 기반 스모크 테스트를 유지합니다. 앞으로 시작 빌드나 M2 루프를 건드릴 때 이 메뉴로 최소 기술 검증을 빠르게 반복합니다.

## 5. 문제 또는 리스크

이번 테스트는 기술 연결 검증입니다. 실제로 뱀파이어 서바이벌처럼 초반이 재미있는지, 타격/보상/망각이 기분 좋게 읽히는지는 jaewoo 직접 플레이 리뷰가 필요합니다.

## 6. GPT/Claude 인계 요약

Codex가 네 가지 시작 빌드와 M2 루프를 직접 실행하는 Unity 에디터 스모크 메뉴를 추가했습니다. 모든 시작 경로는 지정한 무기와 기억으로 초기화됐고, M2 루프는 Hungry/Blood 잔향 +5와 Blood Blade Storm 준비 상태까지 확인됐습니다. 기술 오류는 없으며 남은 판단은 플레이 감각 리뷰입니다.
