# 2026-06-22-05 - 타격 지점 팬텀 무기 연출 적용

## 1. 현재 빌드 상태

`Dev_Prototype_v1`의 무기 표현을 캐릭터 몸에 붙은 실루엣에서, 실제 공격 순간 타격 위치에 잠깐 나타나는 팬텀 무기 방식으로 바꿨다. 이제 평소에는 쌍검/대검이 플레이어 몸에 붙어 보이지 않고, 히트가 발생할 때 베기 VFX 위치에 무기 스프라이트가 짧게 나온다.

## 2. 오늘 바뀐 것

- `LeftBlade` / `RightBlade` 렌더러를 일반 플레이 중에는 항상 비활성으로 유지한다.
- 쌍검 공격은 타겟 양옆에 `DualBladePhantomLeft`, `DualBladePhantomRight` 두 칼날이 짧게 교차해서 나타난다.
- 대검 공격은 cleave 중심에 `GreatswordPhantomStrike`와 옅은 `GreatswordPhantomAfterimage`가 나타난다.
- 무기 스프라이트는 원본 이미지 크기가 아니라 목표 월드 높이 기준으로 스케일링한다.

## 4. 결정한 것

몸에 상시 붙어 있는 무기 실루엣은 이번 방향에서는 제거한다. 현실적인 착용감보다, 공격 위치에 자동으로 칼이 스윽 나타나 베는 마법사형 연출을 우선한다.

## 5. 문제 또는 리스크

Codex는 컴파일, 콘솔 에러, 런타임 오브젝트 생성, 대략적인 bounds까지 검증할 수 있다. 하지만 “보는 맛”, “너무 마법 같아서 가벼운지”, “베기 VFX와 위치가 자연스러운지”는 jaewoo가 직접 플레이해서 판단해야 한다. 현재 스크린샷 경로도 계속 불안정해서 최종 시각 판단에는 적합하지 않다.

## 6. GPT/Claude 인계 요약

무기 표현은 held weapon에서 hit-point phantom weapon으로 바뀌었다. 다음 리뷰는 무기 크기 자체보다, 팬텀 무기가 slash VFX와 잘 맞아 보이는지, 쌍검은 빠르고 선명한지, 대검은 묵직하지만 플레이어를 가리지 않는지에 집중하면 된다.

## 7. 다음 Codex 작업

jaewoo 직접 리뷰 후 하나만 고른다: 팬텀 무기 위치/크기/알파, 베기 VFX 타이밍, 걷기 애니메이션, 피의 칼폭풍 payoff, 보상 속도, 망각 UX.

## 8. 포트폴리오 메모: 문제, 방향, 행동, 결과

- 문제: 캐릭터 몸에 붙은 무기가 오류처럼 보이고, 대검은 시야를 가렸다.
- 방향: 무기를 상시 장착물이 아니라 타격 순간의 마법적 칼날 연출로 바꿨다.
- 행동: held weapon 렌더링을 끄고, 히트 위치에 쌍검/대검 팬텀 스프라이트를 생성했다.
- 결과: 기술 검증상 오류 없이 쌍검/대검 팬텀이 생성되며, 크기는 이전 대검 가림 문제보다 작게 제한됐다. 최종 감각은 직접 플레이 리뷰가 필요하다.
