2026-06-22-06 - 팬텀 무기 선행 모션과 베기 VFX 지속시간 보정

1. 현재 빌드 상태

Dev_Prototype_v1의 무기 타격 연출은 이제 “무기 팬텀 스윽 → 베기 VFX / 스파크 / 히트 확인” 순서로 바뀌었다. 이전에는 타격 순간 모든 것이 거의 동시에 짧게 튀어나와서, 유저가 어느 적을 공격했는지와 어떤 무기인지 읽기 어려웠다.

2. 오늘 바뀐 것

  • 팬텀 무기가 먼저 나타나 타격 지점을 가로지르며 회전한다.
  • 쌍검은 약 0.13~0.14s 동안 빠르게 교차하고, 전체 표시 시간은 0.24~0.26s로 늘렸다.
  • 대검은 약 0.20~0.22s 동안 더 묵직하게 쓸고, 전체 표시 시간은 0.34~0.38s로 늘렸다.
  • slash VFX는 무기 모션보다 약간 늦게 나온다:
  • 쌍검: 0.055s 뒤.
  • 대검: 0.075s 뒤.
  • 무기 slash VFX 지속시간은 1.45x로 늘렸고, 최소 지속시간을 넣었다:
  • 쌍검 최소 0.34s.
  • 대검 최소 0.48s.
  • 히트 스파크와 히트 확인 링/코어도 slash VFX와 같은 딜레이 뒤에 나오게 바꿨다.

4. 결정한 것

타격 연출은 동시에 번쩍이는 방식이 아니라, 무기 모션이 먼저 읽히고 그 다음 베기 자국과 히트 피드백이 남는 순서로 간다. 현실적인 손동작보다 “마법 칼날이 적을 베고 지나가는” 판독성을 우선한다.

5. 문제 또는 리스크

Unity MCP Play Mode 시간이 이번 세션에서도 안정적으로 흐르지 않았다. 그래서 실제 코루틴 시간 경과는 즉시 상태 확인과 enumerator 직접 진행으로 검증했다. 최종적으로 0.055~0.075s 딜레이가 너무 늦거나 빠른지는 jaewoo가 직접 플레이해서 봐야 한다.

6. GPT/Claude 인계 요약

무기 타격 연출은 이제 팬텀 무기 sweep이 먼저 나오고, slash/spark/confirm VFX가 짧게 뒤따른다. 다음 리뷰는 “적절한 45도 베기 모션처럼 보이는지”, “공격 대상이 읽히는지”, “VFX가 너무 늦게 터지는 느낌은 없는지”를 중심으로 보면 된다.

7. 다음 Codex 작업

jaewoo 직접 리뷰 후, 필요하면 다음 중 하나만 조정한다: sweep 각도, slash 지연 시간, VFX 지속시간, 무기 팬텀 알파/크기, 또는 쌍검/대검별 타격 위치 보정.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 무기 종류와 공격 대상이 너무 짧은 VFX 때문에 읽히지 않았다.
  • 방향: 무기 모션을 먼저 보여주고, 베기 VFX를 뒤에 남겨 판독 시간을 늘린다.
  • 행동: 팬텀 무기에 sweep 컴포넌트를 추가하고, slash/spark/confirm VFX를 지연 및 수명 연장했다.
  • 결과: 기술 검증상 무기 팬텀이 먼저 생성되고, VFX는 뒤따르는 순서로 정렬됐다. 최종 감각은 직접 플레이 리뷰가 필요하다.
