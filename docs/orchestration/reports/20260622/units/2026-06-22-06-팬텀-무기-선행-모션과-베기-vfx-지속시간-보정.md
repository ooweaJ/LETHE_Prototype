# 2026-06-22-06 - 팬텀 무기 선행 모션과 베기 VFX 지속시간 보정

## 1. 현재 빌드 상태

`Dev_Prototype_v1`의 무기 타격 연출은 이제 “무기 팬텀 스윽 → 베기 VFX / 스파크 / 히트 확인” 순서로 바뀌었다. 이전에는 타격 순간 모든 것이 거의 동시에 짧게 튀어나와서, 유저가 어느 적을 공격했는지와 어떤 무기인지 읽기 어려웠다.

## 2. 오늘 바뀐 것

- 팬텀 무기가 먼저 나타나 타격 지점을 가로지르며 회전한다.
- 쌍검은 약 `0.13~0.14s` 동안 빠르게 교차하고, 전체 표시 시간은 `0.24~0.26s`로 늘렸다.
- 대검은 약 `0.20~0.22s` 동안 더 묵직하게 쓸고, 전체 표시 시간은 `0.34~0.38s`로 늘렸다.
- slash VFX는 무기 모션보다 약간 늦게 나온다:
  - 쌍검: `0.055s` 뒤.
  - 대검: `0.075s` 뒤.
- 무기 slash VFX 지속시간은 `1.45x`로 늘렸고, 최소 지속시간을 넣었다:
  - 쌍검 최소 `0.34s`.
  - 대검 최소 `0.48s`.
- 히트 스파크와 히트 확인 링/코어도 slash VFX와 같은 딜레이 뒤에 나오게 바꿨다.

## 4. 결정한 것

타격 연출은 동시에 번쩍이는 방식이 아니라, 무기 모션이 먼저 읽히고 그 다음 베기 자국과 히트 피드백이 남는 순서로 간다. 현실적인 손동작보다 “마법 칼날이 적을 베고 지나가는” 판독성을 우선한다.

## 5. 문제 또는 리스크

Unity MCP Play Mode 시간이 이번 세션에서도 안정적으로 흐르지 않았다. 그래서 실제 코루틴 시간 경과는 즉시 상태 확인과 enumerator 직접 진행으로 검증했다. 최종적으로 `0.055~0.075s` 딜레이가 너무 늦거나 빠른지는 jaewoo가 직접 플레이해서 봐야 한다.
