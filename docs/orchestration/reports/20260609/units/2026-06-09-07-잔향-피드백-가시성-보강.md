# 2026-06-09-07 - 잔향 피드백 가시성 보강

## 1. 현재 빌드 상태

- v0.12는 사람 테스트 후보 상태를 유지한다.
- 밸런스 수치는 바꾸지 않았다.
- 잔향이 약하게 느껴지는 문제는 먼저 "효과가 안 보이는 문제"로 보고 UI/피드백을 보강했다.

## 2. 오늘 바뀐 것

- 망각 결과 카드에 `전투 변화` 설명을 추가했다.
- 무기 잔향 카드와 좌측 잔향 목록에 기억별 전투 요약을 넣었다.
- 결손 구간 시작 로그가 이제 "어떤 잔향이 무기에 남았는지"를 말한다.
- 결손 구간 시작 시 `잔향 각인` 텍스트와 burst 이펙트가 뜬다.
- Windows에서 identity QA가 Chrome 임시 폴더 삭제 `EPERM`으로 실패하던 부분을 안정화했다.

## 3. 테스트 결과와 근거

- `npm run qa:postloss`: 처음에는 `CHROME_PATH` 미설정으로 중단.
- `CHROME_PATH=C:\Program Files\Google\Chrome\Application\chrome.exe` 지정 후 `npm run qa:postloss`: 통과, failures `[]`.
- `npm run qa:identity`: QA 본문은 통과했지만 임시 폴더 삭제 `EPERM`으로 실패.
- cleanup retry 보강 후 `CHROME_PATH` 지정 `npm run qa:identity`: 통과, failures `[]`.
- `npm run playtest:package:dry`: 통과.
- `npm run playtest:package`: 통과, `dist\lethe-v0.12-playtest` 재생성.

## 4. 결정한 것

- 잔향이 약하게 느껴진다는 피드백은 바로 수치 버프로 처리하지 않는다.
- 현재는 잔향 효과가 실제로 존재하지만 플레이어에게 잘 읽히지 않는 문제로 본다.
- 수치 상향은 사람 테스트에서 여전히 약하다고 확인된 뒤, 한 번에 하나의 작은 레버로만 검증한다.

## 5. 문제 또는 리스크

- 시각/문구 보강만으로 충분하지 않을 수 있다.
- 사람 테스트에서 여전히 잔향이 약하게 느껴지면 발동 빈도, 이펙트 크기, echo power 중 하나를 골라 별도 검증해야 한다.
- 이번 변경은 자동 밸런스 수치를 다시 돌린 것이 아니라 postloss/identity UI 흐름을 검증한 것이다.

## 6. GPT/Claude 인계 요약

잔향은 HTML 한계가 아니라 전달력 문제로 먼저 처리했다. 수치는 그대로 두고 결과 카드, 로그, 사이드바, 결손 시작 이펙트를 보강했다. 다음 사람 테스트에서는 "망각된 기억이 어떤 무기 잔향으로 남았는지 이해되는가"를 반드시 확인해야 한다.

## 7. 다음 Codex 작업

- `npm run report`와 `npm run report:check`로 보고서 HTML을 갱신한다.
- `npm run doctor`로 로컬 파이프라인 상태를 확인한다.
- 사람 테스트 후 잔향 이해도와 체감 강도를 노트에 반영한다.

## 8. 포트폴리오 메모

- 문제: 핵심 차별점인 잔향이 플레이어에게 잘 느껴지지 않았다.
- 방향: 밸런스 수치를 흔들기 전에 피드백과 설명을 먼저 강화한다.
- 행동: 결과 화면, HUD, 로그, 전투 시작 이펙트에 잔향 의미를 반복 노출했다.
- 결과: 사람 테스트 전에 "무엇이 남았는지 모르는 문제"를 줄인 빌드가 준비됐다.
