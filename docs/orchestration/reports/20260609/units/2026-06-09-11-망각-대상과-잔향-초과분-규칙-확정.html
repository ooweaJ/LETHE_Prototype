2026-06-09-11 - 망각 대상과 잔향 초과분 규칙 확정

1. 현재 빌드 상태

  • 게임 코드는 변경하지 않았다.
  • HTML v0.12는 evidence build로 유지한다.
  • Unity 첫 slice 기획에서 망각 대상, 잔향 초과분, 전투력 하락/회복 목표를 정리했다.

2. 오늘 바뀐 것

  • 망각 대상은 최고 레벨 활성 기억으로 확정했다.
  • 최고 레벨 기억이 여러 개면 플레이어가 그중 하나를 선택한다.
  • 잔향 +5 초과분은 폐기하지 않고 잔향 과부하 또는 궁극 잔향 게이지로 전환한다.
  • 첫 망각 후 전투력 하락 목표를 30~40%, 최대 허용 45%로 조정했다.
  • 보충 후 회복 목표를 95~105%로 좁혔다.
  • Unity 첫 slice 명세에 루프 강제 시연 debug UI를 추가했다.

3. 테스트 결과와 근거

  • 이번 작업은 기획 문서 갱신이다.
  • npm run report: 통과, 2026-06-09 HTML 보고서와 11개 unit report 재생성.
  • npm run report:check: 통과, 11개 unit heading 확인.
  • npm run doctor: 통과, 50 pass / 0 warn / 0 fail.

4. 결정한 것

  • 플레이어는 망각 대상을 직접 고르는 대신, 어떤 기억을 최고 레벨로 키울지로 다음 상실 위험을 설계한다.
  • 강한 기억을 잃는 선택이 잔향 cap 때문에 손해가 되지 않도록 초과분 보상을 둔다.
  • 첫 slice의 손실은 아파야 하지만 허탈하면 안 되므로, 회복 밴드는 좁게 관리한다.

5. 문제 또는 리스크

  • 잔향 과부하가 즉시 폭발인지, 궁극 게이지인지, 둘의 혼합인지는 backlog에서 한 번 더 좁혀야 한다.
  • 최고 레벨 기억 동률 선택 UI가 없으면 디버그 빌드에서는 최근 강화 기억 fallback이 필요하다.
  • 95~105% 회복 밴드는 실제 전투 감각에 따라 미세 조정될 수 있다.

6. GPT/Claude 인계 요약

망각은 최고 레벨 활성 기억을 대상으로 한다. 동률이면 플레이어가 선택한다. 잔향은 +5 cap을 유지하지만 초과분은 버리지 않고 과부하 보상으로 전환한다. 첫 slice 목표는 망각 직후 30~40% 하락, 보충 후 95~105% 회복이다.

7. 다음 Codex 작업

  • debug UI에 즉시 망각, 잔향 +5, 궁극 잔향 시연 버튼을 작업 단위로 넣는다.
