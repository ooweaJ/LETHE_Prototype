2026-06-09-08 - Unity 전환용 핵심 시스템 기획

1. 현재 빌드 상태

  • HTML v0.12는 자동 밸런스 게이트를 통과한 evidence build로 남긴다.
  • 다음 초점은 HTML 추가 튜닝이 아니라 Unity로 가져갈 핵심 시스템 기획이다.
  • 새 기준 문서는 docs/design/LETHE_CORE_SYSTEMS_UNITY_PLAN.md에 만들었다.

2. 오늘 바뀐 것

  • 기억과 잔향이 모두 +5 max라는 규칙을 문서화했다.
  • 기억을 잃으면 해당 레벨만큼 같은 잔향에 누적되는 규칙을 정리했다.
  • +3 잔향이 있는 상태에서 +2 기억을 잃으면 +5 만렙 잔향이 되는 예시를 넣었다.
  • 잔향이 남아 있는 기억을 다시 얻으면 그 기억이 공명으로 강화되는 규칙을 제안했다.
  • 잔향 +5는 각성 잔향이 되어 강하고 눈에 띄는 효과를 가져야 한다고 정리했다.
  • 두 개의 +5 잔향이 궁극 잔향 시너지를 여는 구조를 추가했다.
  • 원거리 적은 계속 도망치는 적이 아니라, 짧게 위치를 바꾸고 멈춰서 쏘는 압박 적으로 정의했다.

3. 테스트 결과와 근거

  • 이번 작업은 코드 구현이 아니라 시스템 기획 문서화다.
  • 이전 검증 근거는 v0.12의 balance:loop 2회 연속 GO_BALANCE_BASELINE과 잔향 피드백 QA 통과다.
  • 보고서 생성 후 npm run report, npm run report:check, npm run doctor로 문서 파이프라인을 확인한다.

4. 결정한 것

  • 다음 단계는 HTML 밸런스 추가 조정이 아니라 Unity vertical slice 기획이다.
  • 잃은 기억은 단순 삭제가 아니라 잔향 성장으로 전환한다.
  • 같은 기억을 다시 얻으면 남아 있는 잔향과 공명해 더 강한 기억이 된다.
  • Unity 첫 slice는 모든 콘텐츠가 아니라, 각성 잔향 2개와 궁극 잔향 1개를 제대로 보여주는 방향이 좋다.

5. 문제 또는 리스크

  • 재획득 공식은 아직 확정이 필요하다. 현재 제안은 base offered level + floor(echo level / 2)다.
  • 궁극 잔향 조합을 너무 많이 만들면 Unity 첫 slice 범위가 커진다.
  • +5 잔향이 강해야 하지만, 너무 자동 승리처럼 느껴지면 빌드 선택 재미가 줄어들 수 있다.

6. GPT/Claude 인계 요약

LETHE는 Unity 전환 전 핵심 시스템을 기억 -> 잔향 -> 공명 -> 각성 잔향 -> 궁극 잔향 구조로 재정의했다. 기억과 잔향은 모두 +5 max이며, 잃은 기억 레벨은 같은 잔향에 누적된다. 같은 기억을 다시 얻으면 잔향을 소비하지 않고 기억이 강화된다. 다음 작업은 이 문서를 Unity vertical-slice backlog로 쪼개는 것이다.

7. 다음 Codex 작업

  • 첫 구현 대상으로 각성 잔향 2개와 궁극 잔향 1개를 고른다.
  • Unity data model, combat prefab, UI/debug panel, enemy AI 기준을 작업 단위로 나눈다.

8. 포트폴리오 메모

  • 문제: HTML에서 잔향이 약하게 느껴진 이유는 단순 UI 문제가 아니라, 잔향이 성장 시스템으로 충분히 설계되지 않았기 때문이다.
  • 방향: 잃은 기억이 새로운 힘으로 누적되고, 다시 얻은 기억과 공명하는 구조로 바꾼다.
  • 행동: 기억/잔향 레벨, 재획득 공명, 각성 잔향, 궁극 잔향, 원거리 적 행동 원칙을 시스템 문서로 정리했다.
  • 결과: Unity 전환을 위한 첫 핵심 시스템 계약이 생겼다.
