패널 닫기

2026-07-09-01 - 기본 칼무리 잔향 굶주림 이펙트 전환

1. 현재 빌드 상태

Dev_Prototype_v1의 기본 칼무리 잔향이 선택 후보용 VFX가 아니라 실제 런타임 기본 효과로 교체됐다. 아직 _dev 검증 단계이며 Assets/Lethe 승격은 하지 않았다.

2. 오늘 바뀐 것

  • 기본 칼무리 잔향을 무기 궤적 -> 피 냄새/상처 끌림 -> 안쪽으로 뜯어먹는 칼무리 구조로 바꿨다.
  • 쌍검은 빠른 이빨, 짧은 갉힘 자국, 여러 개의 작은 물어뜯기 표식으로 보이게 했다.
  • 대검은 큰 상처 웅덩이, 닫히는 턱, 상처로 빨려 들어가는 궤적, 무거운 균열 자국으로 보이게 했다.
  • 기본 +5 칼무리 각성은 더 이상 예전처럼 칼 하나가 날아가는 투사체를 만들지 않고, 맞은 지점에서 상처 포식 폭발을 만든다.
  • F12 K1-K4 후보 모드는 남겨두었지만, 아무 후보도 켜지 않은 기본 상태가 새 칼무리 런타임이다.

3. 테스트 결과와 근거

  • Unity compilation errors: 0.
  • Kalmuri Perf Matrix: PASS, totalKalmuri=340.
  • Dense Dual Blades Perf Matrix: PASS, hits=18, suppressed=15, transient=134, activeVfx=52, ms=8.56.
  • Echo Matrix Dual Blades: PASS, total=231, K=8, state=87.
  • Echo Matrix Greatsword: PASS, total=207, K=8, state=58.

4. 결정한 것

칼무리 잔향의 기본 방향은 K2 피 냄새 추적 + K1 상처 포식 하이브리드로 고정했다. 핵심은 새 칼이 몸에서 튀어나가는 것이 아니라, 무기가 남긴 상처에서 칼무리들이 냄새를 맡고 몰려와 뜯어먹는 느낌이다.

5. 문제 또는 리스크

자동 QA는 오브젝트 수와 기본 판정만 본다. 실제로 쌍검이 너무 잔잔한지, 대검이 충분히 묵직한지, 빨강/주황 계열이 혈반 잔향과 너무 겹치는지는 직접 플레이 평가가 필요하다.

6. GPT/Claude 인계 요약

기본 칼무리 잔향은 후보 비교 단계를 지나 실제 런타임으로 들어갔다. 다음 리뷰는 K1-K4 후보 선택이 아니라, F12 후보 모드를 끈 기본 칼무리 효과가 충분히 칼무리다운지 평가해야 한다.

7. 다음 Codex 작업

jaewoo 직접 플레이 결과에 따라 이펙트 세기, 상처 유지 시간, 이빨 수, 대검 턱 닫힘 타이밍, 쌍검 밀집전 가독성을 조정한다. 문제가 없으면 다른 유틸리티 잔향에도 같은 수준의 무기별 개성을 적용한다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 칼무리 잔향이 몸에서 칼 하나가 또 나가는 것처럼 보여 컨셉과 어긋났다.
  • 방향: 굶주린 칼무리는 상처와 궤적을 먹는 효과로 재정의했다.
  • 행동: 기본 잔향 VFX와 +5 각성 VFX를 상처 포식형으로 교체하고, 쌍검/대검 실루엣을 나눴다.
  • 결과: Unity 컴파일과 핵심 칼무리/밀집/무기별 Echo QA가 모두 PASS했다.

2026-07-09-02 - 푸른 칼무리 기억 계열 VFX 보정

1. 현재 빌드 상태

기본 칼무리 잔향은 이제 붉은 상처 효과가 아니라, 원래 굶주린 칼무리 기억의 푸른 칼날 톤을 이어받는 방향으로 보정됐다. _dev 검증 단계는 유지한다.

2. 오늘 바뀐 것

  • 기본 칼무리 잔향 색을 빨강/주황에서 청백/푸른색 계열로 바꿨다.
  • 쌍검 기본 칼무리에서 큰 붉은 원형 상처 웅덩이 느낌을 제거했다.
  • 안쪽으로 물어뜯는 파츠를 다이아몬드 이빨이 아니라 실제 칼무리 칼날 스프라이트로 바꿨다.
  • 쌍검은 빠른 푸른 균열, 칼날 끌림, 작은 칼날 물어뜯기 표식 중심으로 보이게 했다.
  • 대검은 푸른 기억 웅덩이, 칼날 파동 링, 큰 칼날 턱 닫힘, 푸른 파편 균열 중심으로 보이게 했다.
  • +5 각성도 푸른 칼날들이 상처 지점으로 빨려 들어가는 형태로 맞췄다.

3. 테스트 결과와 근거

  • Unity compilation errors: 0.
  • Kalmuri Perf Matrix: PASS, totalKalmuri=356.
  • Dense Dual Blades Perf Matrix: PASS, hits=18, suppressed=15, transient=120, activeVfx=58, ms=86.60.
  • Echo Matrix Dual Blades: PASS, total=232, K=8, state=86.
  • Echo Matrix Greatsword: PASS, total=207, K=8, state=58.

4. 결정한 것

칼무리 잔향의 메인 색감은 혈반처럼 붉게 가지 않고, 원래 기억의 푸른 칼무리 계보를 유지한다. 포식감은 남기되 피색이 아니라 차가운 칼날들이 상처로 끌려 들어가는 느낌으로 표현한다.

5. 문제 또는 리스크

자동 QA는 색감과 쾌감을 판단하지 못한다. 실제 플레이에서 푸른 칼날이 충분히 화려한지, 너무 얇거나 빨리 사라지는지, 대검이 충분히 묵직한지는 직접 평가가 필요하다.

6. GPT/Claude 인계 요약

기본 칼무리 잔향은 이제 붉은 상처 포식이 아니라 푸른 칼날 기억의 잔향이다. 다음 판단은 “칼무리답냐”뿐 아니라 “원래 기억이 잔향으로 남았다는 계보가 읽히냐”를 중심으로 봐야 한다.

7. 다음 Codex 작업

직접 플레이 결과에 따라 칼날 잔상 개수, 푸른 발광 강도, 상처선 지속시간, 대검 칼날 턱 닫힘 타이밍을 조정한다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 새 칼무리 잔향이 붉은 상처 효과처럼 보여 원래 푸른 칼무리 기억과 연결이 약했다.
  • 방향: 색과 형태를 원래 기억의 푸른 칼날 계열로 되돌리고, 디테일은 실제 검 파츠로 강화했다.
  • 행동: 기본 잔향과 +5 각성 VFX를 청백색 칼날 끌림/칼날 물어뜯기 형태로 수정했다.
  • 결과: 핵심 Unity QA가 모두 PASS했고, 이제 직접 플레이로 감성 평가할 수 있는 상태가 됐다.

2026-07-09-03 - LETHE 인트로 무기 선택 화면

1. 현재 빌드 상태

Dev_Prototype_v1 첫 화면에 레테 감성의 인트로 무기 선택 화면을 추가했다. 아직 _dev 검증 단계이며 Assets/Lethe 승격은 하지 않았다.

2. 오늘 바뀐 것

  • 기존 시작 무기 선택 오버레이 대신 새 DrawLetheIntroOverlay()를 사용한다.
  • 화면은 어두운 강, 낮은 물안개, 푸른 기억 조각 느낌의 배경으로 구성했다.
  • 첫 선택은 단순 클릭 시작이 아니라 무기 선택으로 유지했다.
  • 절단쌍검 카드는 빠른 2연 베기와 잦은 잔향 리듬을 보여준다.
  • 장송대검 카드는 느리지만 무거운 강타와 큰 잔향 리듬을 보여준다.
  • 숫자 1, 2와 카드 클릭 모두 기존 BeginRun 경로로 런을 시작한다.

3. 테스트 결과와 근거

  • Unity compilation errors: 0.
  • Play Mode 첫 상태: weaponSelectOverlay=True, runStarted=False, GameplayPaused=True.
  • 선택 호출 후 상태: weaponSelectOverlay=False, runStarted=True, GameplayPaused=False.
  • LETHE/V1 QA/Start Dual Blades: 정상 호출 및 시작 스냅샷 확인.
  • LETHE/V1 QA/Start Greatsword: 호출은 됐지만 liveEnemies=2로 현재 QA 기준에 실패했다. 인트로 라우팅 문제라기보다 대검 시작 smoke 기대치 불일치로 분리해서 봐야 한다.

4. 결정한 것

인트로는 별도의 Start 버튼만 두지 않고, 무기 선택 자체를 시작 의식으로 사용한다. 레테에서는 “어떤 무기로 강을 건너는가”가 첫 감정 선택이기 때문이다.

5. 문제 또는 리스크

  • MCP 상태 검증은 화면이 뜨고 시작되는 흐름만 보장한다. 실제 Game view에서 구도와 분위기가 충분한지는 직접 눈으로 봐야 한다.
  • 대검 시작 QA가 현재 실패하므로, 다음 작업에서 QA 기대치 또는 대검 시작 smoke 시뮬레이션을 정리해야 한다.

6. GPT/Claude 인계 요약

첫 화면은 무기 선택형 인트로로 정했다. 다음 리뷰는 “레테 감성”, “무기 카드 가독성”, “시작 의식으로 충분한가”를 보면 된다. 대검 시작 QA 실패는 인트로와 분리해서 처리한다.

7. 다음 Codex 작업

jaewoo가 인트로 화면을 직접 보고 유지/수정/재설계를 판단한다. 이후 대검 시작 QA 실패를 별도 작은 작업으로 고친다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 기존 시작 화면은 기능은 있었지만 첫인상이 게임의 감성과 충분히 연결되지 않았다.
  • 방향: 시작 버튼보다 무기 선택을 첫 의식으로 만들었다.
  • 행동: 어두운 강 배경과 두 무기 카드, 전투 리듬 설명, 기존 시작 라우팅을 연결했다.
  • 결과: 컴파일과 상태 흐름은 통과했고, 이제 직접 화면 감성 검수가 가능한 상태가 됐다.
