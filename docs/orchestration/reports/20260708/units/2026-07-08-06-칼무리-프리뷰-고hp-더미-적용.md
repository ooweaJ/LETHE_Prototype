# 2026-07-08-06 - 칼무리 프리뷰 고HP 더미 적용

## 1. 현재 빌드 상태

`Dev_Prototype_v1`의 F12 칼무리 프리뷰가 다시 보강됐다. 이제 `K1`~`K4`를 누르면 고HP 더미가 생성되고, 화면에 최신 버전 라벨이 뜬다.

## 2. 오늘 바뀐 것

- 칼무리 프리뷰 전용 더미 적 HP를 `9999`로 올렸다.
- 프리뷰 데미지는 `1`로 제한했다.
- `K1`~`K4`를 누르면 `K Preview v2 / high HP dummies` 라벨이 뜨게 했다.

## 3. 테스트 결과와 근거

- `dotnet build LETHE/Assembly-CSharp.csproj --nologo`: 기존 legacy warning 7개, error 0개.
- `dotnet build LETHE/Assembly-CSharp-Editor.csproj --nologo`: 기존 legacy warning 7개, error 0개.
- Unity compilation errors: `0`.
- Unity console errors: `0`.

## 4. 결정한 것

프리뷰는 전투 밸런스가 아니라 VFX 비교 도구이므로, 더미가 죽지 않고 반복 확인 가능해야 한다.

## 5. 문제 또는 리스크

만약 `K Preview v2 / high HP dummies` 라벨이 안 뜨면 Play Mode가 이전 스크립트 어셈블리를 쓰고 있는 것이다. Play Mode를 나갔다가 다시 들어가야 한다.

## 6. GPT/Claude 인계 요약

칼무리 후보 비교는 이제 고HP 더미 기준으로 봐야 한다. 최신 반영 확인 신호는 `K Preview v2 / high HP dummies` 라벨이다.

## 7. 다음 Codex 작업

jaewoo가 F12에서 `K1`~`K4`를 다시 눌러 보고, 라벨이 보이는지와 각 후보가 충분히 다르게 보이는지 판단한다.

## 8. 포트폴리오 메모: 문제, 방향, 행동, 결과

- 문제: 더미가 빨리 죽어 VFX를 보기 어려웠고 업데이트 적용 여부도 헷갈렸다.
- 방향: 디버그 프리뷰를 전투가 아닌 관찰용으로 분리한다.
- 행동: 고HP 더미, 데미지 제한, v2 라벨을 추가했다.
- 결과: 프리뷰를 반복해서 확인할 수 있는 상태가 됐다.
