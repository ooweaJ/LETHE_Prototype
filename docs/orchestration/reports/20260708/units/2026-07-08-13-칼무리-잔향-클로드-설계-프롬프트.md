# 2026-07-08-13 - 칼무리 잔향 클로드 설계 프롬프트

## 1. 현재 빌드 상태

`Dev_Prototype_v1`의 칼무리 K1~K4 후보는 기술적으로 테스트 가능한 상태지만, jaewoo 기준으로 아직 `굶주린 칼무리의 잔향`답지 않다. 이번 작업은 코드 수정이 아니라 클로드에게 설계 판단을 맡기기 위한 프롬프트 정리다.

## 2. 오늘 바뀐 것

- `docs/orchestration/review_prompts/2026-07-08-kalmuri-echo-identity-redesign-claude.md`를 추가했다.
- 프롬프트에 현재 실패 맥락을 명시했다:
  - 오래된 플레이어 발사 칼날은 실패 방향.
  - K1~K4 wound/scent/feast/trail 후보도 아직 만족스럽지 않음.
  - 쌍검의 큰 빨간 원 판독은 잘못된 방향.
  - 잔향은 무기 타격 지점/상처에서 반응해야 함.
- 클로드가 단순 VFX 아이디어가 아니라 활성 기억, 잔향, +5 각성의 정체성 표를 먼저 제안하도록 요청했다.

## 3. 테스트 결과와 근거

문서/프롬프트 작성 작업이라 Unity 빌드나 Play Mode 검증은 새로 실행하지 않았다. 이전 최신 검증 기준은 빌드 에러 0, Unity compilation errors 0, console errors 0 상태다.

## 4. 결정한 것

다음 칼무리 작업은 후보를 더 칠하는 방식이 아니라, `굶주린 칼무리`라는 기억이 무엇이고 망각 뒤 어떤 습관이 잔향으로 남는지부터 다시 정한다.

## 5. 문제 또는 리스크

클로드 답변이 너무 추상적이면 바로 구현 후보로 쓰기 어렵다. 그래서 프롬프트에 후보별 VFX, 판정, 무기별 행동, 재미 포인트, 리스크까지 표로 요구했다.

## 6. GPT/Claude 인계 요약

클로드에게는 `굶주린 칼무리`의 핵심 이미지를 재정의하고, 활성 기억과 잔향의 차이를 표로 정리한 뒤, Codex가 구현 가능한 4개의 플레이어블 후보를 추천하라고 요청한다.

## 7. 다음 Codex 작업

클로드 답변을 받으면 가장 강한 후보 1개와 보조 후보 1개를 골라 `V1GameManager.cs`의 K1~K4 후보를 다시 설계하거나, 선택된 후보를 실제 칼무리 잔향 경로로 통합한다.

## 8. 포트폴리오 메모: 문제, 방향, 행동, 결과

- 문제: 칼무리 후보들이 기능적으로는 존재하지만 기억이 잔향으로 변한 감각이 부족했다.
- 방향: VFX보다 먼저 기억/잔향 정체성을 재정의한다.
- 행동: 클로드가 이해할 수 있도록 실패 이력, 제약, 출력 형식을 포함한 설계 프롬프트를 작성했다.
- 결과: 다음 리뷰는 감각적 불만을 구현 가능한 후보 표로 변환할 수 있는 상태가 됐다.
