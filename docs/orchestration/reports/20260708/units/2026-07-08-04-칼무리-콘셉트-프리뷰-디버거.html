2026-07-08-04 - 칼무리 콘셉트 프리뷰 디버거

1. 현재 빌드 상태

Dev_Prototype_v1에 칼무리 잔향 후보를 직접 비교하는 F12 디버그 프리뷰가 추가됐다. 아직 최종 전투안 확정이 아니라, jaewoo가 화면에서 어떤 방향이 가장 칼무리답게 읽히는지 고르는 단계다.

2. 오늘 바뀐 것

  • F12 디버그 패널에 K1, K2, K3, K4 버튼을 추가했다.
  • K1: 상처 포식형. 여러 칼날이 타격 상처로 모여든다.
  • K2: 궤적 증식형. 공격 궤적 잔상이 늦게 추가 절단으로 번진다.
  • K3: 군집 교차참격형. 대상 주변에서 칼들이 나타나 안쪽으로 교차 절단한다.
  • K4: 표식 폭주형. 굶주린 검흔 표식이 주변 적에게 갈라진다.
  • 각 버튼은 같은 더미 적 배치를 다시 만들고 해당 콘셉트만 보여줘 비교 조건을 맞춘다.

3. 테스트 결과와 근거

  • dotnet build LETHE/Assembly-CSharp.csproj --nologo: 순차 재실행 기준 warning 0개, error 0개.
  • dotnet build LETHE/Assembly-CSharp-Editor.csproj --nologo: 기존 legacy warning 7개, error 0개.
  • Unity console errors: 0.
  • Unity EditorApplication.isCompiling 값이 refresh 이후 오래 true로 유지되어, 실제 버튼 클릭 검증은 직접 플레이에서 이어서 확인해야 한다.

4. 결정한 것

칼무리 잔향은 바로 하나로 확정하지 않고, 네 가지 화면 언어를 같은 조건에서 비교한 뒤 최종안을 고른다.

5. 문제 또는 리스크

현재 프리뷰는 비교용 샘플이다. 최종 전투 판정, 밀집 최적화, 쌍검/대검 분리 적용은 선택 이후 다시 구현해야 한다.

6. GPT/Claude 인계 요약

칼무리의 핵심 이슈는 수치가 아니라 화면 문법이다. K1~K4 중 어떤 것이 “칼무리 잔향”으로 가장 잘 읽히는지 선택한 뒤, 그 방향을 실제 잔향 로직에 이식하면 된다.

7. 다음 Codex 작업

jaewoo가 Play Mode에서 F12를 누르고 K1~K4를 차례로 눌러 본다. 선택 결과가 나오면 Codex가 해당 후보 또는 하이브리드를 실제 칼무리 잔향으로 변환한다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 기존 칼무리 잔향이 단일 투사체처럼 보여 콘셉트가 약했다.
  • 방향: 확정 구현 전에 여러 VFX 문법을 직접 비교한다.
  • 행동: F12 디버그 패널에 네 가지 칼무리 콘셉트 프리뷰를 추가했다.
  • 결과: 이제 플레이 화면에서 후보를 비교하고 최종 방향을 고를 수 있다.
