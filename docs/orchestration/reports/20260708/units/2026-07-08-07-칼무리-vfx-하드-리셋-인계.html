2026-07-08-07 - 칼무리 VFX 하드 리셋 인계

1. 현재 빌드 상태

현재 K1~K4 프리뷰는 비교 도구로 충분하지 않다. 더미 HP나 색 조정 문제가 아니라, 기존 칼무리 칼날/청록/짧은 폭발 언어를 계속 재사용해서 jaewoo가 보기에는 거의 안 바뀐 것처럼 느껴진다.

2. 오늘 바뀐 것

  • 다음 작업 방향을 문서에 고정했다.
  • 현재 K1~K4 프리뷰를 더 다듬는 대신, 칼무리 VFX를 하드 리셋하기로 했다.
  • 다음 세션에서는 기존 칼무리 스프라이트와 헬퍼를 재사용하지 않는 새 후보를 만들어야 한다.

3. 테스트 결과와 근거

이번 단위는 코드 구현이 아니라 인계/작업 방향 기록이다. 이전 검증 기준으로 현재 빌드는 컴파일 에러 0, Unity 콘솔 에러 0 상태다.

4. 결정한 것

다음 작업은 “색/수명/크기 조정”이 아니다. 기존 칼무리 VFX를 없애고 완전히 다른 실루엣으로 다시 만든다.

5. 문제 또는 리스크

지금 프리뷰는 계속 기존 칼날 느낌을 끌고 와서, 플레이어가 후보 차이를 판단할 수 없다. 이 상태로 실제 잔향에 이식하면 같은 문제가 반복된다.

6. GPT/Claude 인계 요약

다음 세션은 V1GameManager.cs의 현재 칼무리 프리뷰 헬퍼를 삭제하거나 우회하는 것부터 시작한다. 새 후보는 KalmuriBladeSprite()를 쓰지 않는다. 후보 실루엣은 아래처럼 완전히 달라야 한다.

  • K1: 상처 입/톱니 검흔.
  • K2: 긴 리본 궤적/잔상 띠.
  • K3: 큰 기하학적 X/교차 폭발.
  • K4: 저주 표식/사슬 링크 네트워크.

7. 다음 Codex 작업

1. docs/orchestration/state/CURRENT_TASK.md의 Kalmuri VFX Hard Reset Handoff부터 읽는다.

2. 현재 DebugPreviewKalmuriWoundFeast, DebugPreviewKalmuriTrailBloom, DebugPreviewKalmuriCrossSwarm, DebugPreviewKalmuriMarkFrenzy를 삭제하거나 새 구현으로 완전히 교체한다.

3. KalmuriBladeSprite()와 기존 칼무리 헬퍼를 새 프리뷰 후보에서 사용하지 않는다.

4. F12 프리뷰 라벨을 K Preview HARD RESET으로 바꾼다.

5. 각 후보가 최소 0.8초 이상 남아 화면에서 비교 가능하게 한다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 칼무리 프리뷰가 기존 VFX 재사용 때문에 거의 바뀌지 않은 것처럼 보였다.
  • 방향: 기존 칼날 언어를 버리고 완전히 다른 시각 문법으로 재설계한다.
  • 행동: 다음 세션용 하드 리셋 지시를 상태/작업 문서에 기록했다.
  • 결과: 집에서 이어 할 때 바로 현재 프리뷰 폐기와 새 실루엣 제작부터 시작할 수 있다.
