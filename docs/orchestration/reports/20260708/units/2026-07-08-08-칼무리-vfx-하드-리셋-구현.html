2026-07-08-08 - 칼무리 VFX 하드 리셋 구현

1. 현재 빌드 상태

Dev_Prototype_v1의 F12 칼무리 미리보기 K1부터 K4까지 하드 리셋 버전으로 교체했다. 아직 _dev 검증 단계이며 Assets/Lethe 승격은 하지 않았다.

2. 오늘 바뀐 것

  • K1: 상처 입 / 톱니 흉터 방향.
  • K2: 리본 궤적 / 매장 깃발 방향.
  • K3: 기하학적 X / 십자 폭발 방향.
  • K4: 저주 표식 / 체인 네트워크 방향.
  • 쌍검은 빠르고 얇은 다단 히트 느낌으로, 대검은 넓고 무거운 단발 충격 느낌으로 분리했다.
  • 미리보기 라벨을 K Preview HARD RESET / HP 9999로 바꿔 최신 버전 적용 여부를 바로 확인할 수 있게 했다.

4. 결정한 것

이번 단계에서는 실제 칼무리 Echo를 바로 교체하지 않고, 먼저 네 개 후보를 눈으로 비교하는 선택 도구로 만든다. jaewoo가 하나를 고르거나 섞을 방향을 정하면 그때 실제 전투 동작에 이식한다.

5. 문제 또는 리스크

자동 QA는 컴파일과 기본 성능만 보장한다. 최종 판단은 직접 플레이에서 “칼무리답게 보이는가”, “쌍검과 대검의 차이가 바로 읽히는가”를 보고 내려야 한다.

6. GPT/Claude 인계 요약

칼무리 후보는 이제 색만 다른 칼날 묶음이 아니라 완전히 다른 실루엣 네 개다. 다음 리뷰는 설명 없이 화면만 보고 K1-K4가 구분되는지, 그리고 쌍검/대검 분리가 유지되어야 하는지를 판단하면 된다.

7. 다음 Codex 작업

jaewoo가 F12에서 K1-K4를 직접 본 뒤 승자 또는 하이브리드를 지정하면, Codex는 해당 시각 문법을 실제 칼무리 Echo 런타임으로 변환한다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 기존 칼무리 미리보기는 후보끼리 너무 비슷해 선택 기준이 생기지 않았다.
  • 방향: 칼날 재사용을 줄이고 상처, 리본, 십자, 표식 네 가지 서로 다른 실루엣으로 재설계한다.
  • 행동: F12 K1-K4 후보를 하드 리셋하고 쌍검/대검 변형을 각각 만들었다.
  • 결과: 빌드와 Unity QA가 통과했고, 이제 직접 플레이로 후보를 고를 수 있는 상태가 됐다.
