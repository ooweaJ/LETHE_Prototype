2026-06-12-06 - 기본공격 VFX와 잔향 후속타 방향 정리

1. 현재 빌드 상태

이번 단위는 코드 구현이 아니라 다음 구현 방향 정리다. 현재 쌍검은 허공 베기가 제거됐지만, 기본공격 VFX가 플레이어 주변 부채꼴처럼 보이면 여전히 어색하다. 다음 구현은 기본공격 VFX를 적 위치 발도/절단선으로 바꾸고, 칼무리 잔향은 그 적중 위치에서 나오는 후속타로 만든다.

2. 오늘 바뀐 것

  • LETHE_DESIGN_02_COMBAT.md에 기본공격 VFX/판정 분리 규칙을 추가했다.
  • 판정은 range/arc/cleave를 쓸 수 있지만, 화면 VFX는 적 위치에 생기는 발도선으로 간다.
  • LETHE_DESIGN_03_MEMORY_ECHO.md에 온힛 잔향 후속타 규칙을 추가했다.
  • 잔향 origin은 player가 아니라 hit.position 또는 primaryTarget.position이다.
  • 칼무리 MultiSmall은 기본공격 절단선 뒤에 적중 지점 주변에서 작은 후속타로 나온다.
  • LETHE_DESIGN_06_BUILD_PLAN.md에 다음 회귀 체크를 추가했다: targetLocalSlash>0, playerFanArc=0, kalmuriFollowup>0.
  • CURRENT_TASK.md, NEXT_TASKS.md, STATUS.md를 다른 로컬에서 바로 이어받을 수 있게 갱신했다.

3. 테스트 결과와 근거

  • 문서 정리 단위라 Unity 실행은 하지 않았다.
  • 다음 검증 기준은 문서에 명시했다.
  • npm.cmd run report: 통과, 6개 unit report 생성.
  • npm.cmd run report:check: 통과, 6개 unit heading 확인.
  • npm.cmd run report:orchestrator:unit:dry: fetch failed로 실패.

4. 결정한 것

  • 기본공격 VFX는 플레이어 부채꼴이 아니라 대상 위치 발도/절단선이다.
  • 잔향은 기본공격 hit 이후 같은 위치에서 나오는 후속타다.
  • 쌍검 잔향은 캐릭터가 멈추는 느낌을 주면 안 된다.
  • 대검은 나중에 큰 hitstop과 큰 참격을 허용한다.

5. 문제 또는 리스크

  • 아직 코드가 이 방향을 완전히 따르지 않는다.
  • 현재 다음 구현에서 기존 player-origin/fan-like slash VFX를 제거해야 한다.
  • Project Orchestrator Discord intake가 현재 응답하지 않아 dry-run 보고가 실패했다.

6. GPT/Claude 인계 요약

다음 Codex는 V1GameManager의 쌍검 VFX를 target-local로 바꿔야 한다. 공격 판정은 현재 nearest/no-air-swing을 유지하되, VFX는 적 위치에 2개의 작은 발도선으로 만들고, 칼무리 MultiSmall은 그 hit origin에서 0~0.08초 후 후속타처럼 나오게 한다.

7. 다음 Codex 작업

  • player-origin/fan-like 쌍검 arc VFX 제거.
  • primary target 위치에 쌍검 발도선 2개 생성.
  • cleave 대상에는 작은 보조 절단선 생성.
  • 칼무리 MultiSmall을 hit origin 후속타로 이동.
  • Unity smoke에서 targetLocalSlash>0, playerFanArc=0, kalmuriFollowup>0 확인.

8. 포트폴리오 메모

  • 문제: 기본공격이 여전히 플레이어 주변 arc로 보이면 허공 베기 인상이 남는다.
  • 방향: 판정과 VFX를 분리해, 공격은 적에게 찍히고 잔향은 적중 지점에서 번지게 한다.
  • 행동: 전투/잔향/빌드/오케스트레이션 문서에 다음 구현 기준을 남겼다.
  • 결과: 다른 로컬에서도 바로 이어서 구현할 수 있는 다음 작업 계약이 생겼다.
