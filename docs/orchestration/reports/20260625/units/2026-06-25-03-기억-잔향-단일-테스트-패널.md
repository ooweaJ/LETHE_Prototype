# 2026-06-25-03 - 기억/잔향 단일 테스트 패널

## 1. 현재 빌드 상태

디버그 패널에서 8종 기억/잔향을 한 종류씩 골라 테스트할 수 있게 되었다. 이제 통합 화면에서 "뭔가 안 보인다"를 느끼는 단계에서 멈추지 않고, 같은 id의 기억과 잔향을 따로 켜서 비교할 수 있다.

## 2. 오늘 바뀐 것

- `Prev` / `Next`는 테스트할 기억/잔향 id를 선택한다.
- `Mem One` 버튼을 추가했다.
- `Mem One`은 주변 리뷰 적을 만들고, 잔향/궁극 상태를 지운 뒤 선택한 기억만 +5로 켠다.
- `Echo One`은 그대로 유지해서 선택한 잔향만 +5로 켠다.
- `DB Rev` / `GS Rev`는 전체 잔향 통합 리뷰용으로 유지했다.

## 3. 테스트 결과와 근거

- `dotnet build LETHE/Assembly-CSharp.csproj --nologo`: 통과, warning 0개, error 0개.

- Unity console에는 MCP 서버 정보 로그만 있었고 게임 오류/경고는 없었다.

## 4. 결정한 것

기억과 잔향의 가시성 문제는 한꺼번에 보면 원인을 분리하기 어렵다. 그래서 같은 선택 슬롯으로 `Mem One`과 `Echo One`을 번갈아 켜서 "기억 자체가 약한지", "잔향만 약한지", "둘이 너무 비슷한지"를 확인하는 흐름으로 바꿨다.

## 5. 문제 또는 리스크

기술 검증은 통과했지만, 실제 판단은 직접 플레이가 필요하다. 특히 각 id별로 색, 범위, 유지시간, 피해 가치가 기억과 잔향 사이에서 충분히 다르게 느껴지는지 봐야 한다.

## 6. GPT/Claude 인계 요약

Codex가 패널에 `Mem One`을 추가했고 기존 `Echo One`과 같은 선택 id를 공유하게 만들었다. 다음 리뷰는 8종을 하나씩 돌면서 기억/잔향의 차별성, 밸류, 가시성을 표로 적는 방식이 좋다.
