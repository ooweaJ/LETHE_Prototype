2026-06-25-04 - 전투 체감 통합 패스

1. 현재 빌드 상태

기억/잔향 단일 테스트는 나중으로 미루고, 지금은 대검 VFX 중복감과 전체 전투 진행감을 먼저 정리했다. 대검은 이제 프로필 참격이 메인이고, 보장형 VFX는 겹쳐 보이는 큰 두 번째 참격이 아니라 칼끝 잔광/컷라인으로 낮췄다.

2. 오늘 바뀐 것

  • 대검 보장형 VFX를 상황별로 나눴다.
  • 프로필 VFX가 있으면 얇은 tip afterglow와 cut line만 나온다.
  • 프로필 VFX가 없을 때만 fallback cleave가 크게 나온다.
  • 대검은 1.02s -> 0.92s로 조금 빨라졌고, 사거리/각도/넉백/히트스톱/흔들림은 더 묵직하게 정리했다.
  • 쌍검은 0.36s -> 0.32s로 더 빠르게 만들고, 단일 피해/범위/히트스톱은 줄여 빠르고 가벼운 무기 쪽으로 밀었다.
  • 첫 문지기 타이밍은 150s -> 135s, 보스 스케줄은 135 / 285 / 435 / 600으로 바꿨다.
  • 문지기 경고는 22초 전에 더 크고 오래 보이게 바꿨다.
  • 초반 120초 스폰 압력을 올리고, 적 역할 마커와 맵 랜드마크를 더 잘 보이게 했다.
  • 배경/바닥 톤을 조금 어둡게 내려 적과 VFX가 더 위에 뜨게 했다.

3. 테스트 결과와 근거

  • dotnet build LETHE/Assembly-CSharp.csproj --nologo: 통과, 기존 v0/debug 경고 7개, error 0개.
  • Unity compile error count: 0.
  • Unity Play Mode 진입: isPlaying=true 확인.
  • Unity console에는 MCP 서버 정보 로그만 있었고 게임 오류/경고는 없었다.

4. 결정한 것

대검 문제는 VFX 부재가 아니라 VFX 위계 문제로 본다. 그래서 대검의 “맛”은 프로필 참격과 무기 스윙이 맡고, 보장형 VFX는 안전장치 겸 칼끝 확인선으로 낮췄다. 초반 루즈함은 새 시스템을 추가하기보다 보스 타이밍, 스폰 압력, 역할 마커, 맵 대비를 먼저 조정했다.

5. 문제 또는 리스크

직접 플레이에서 135초 첫 보스가 너무 빠르게 느껴질 수 있다. 또 대검 fallback을 낮춘 만큼, 프로필 VFX가 실제 화면에서 충분히 선명한지 다시 봐야 한다.

6. GPT/Claude 인계 요약

Codex가 전투 체감 통합 패스를 적용했다. 다음 판단 기준은 대검이 한 번의 큰 베기로 읽히는지, 쌍검이 더 빠르고 가볍게 느껴지는지, 첫 문지기 135초가 목표감을 주는지, 어두워진 지형이 가시성을 살리는지다.

8. 포트폴리오 메모: 문제, 방향, 행동, 결과

  • 문제: 대검 VFX가 사라진 문제를 고쳤더니 이번에는 두 겹으로 나오는 느낌이 생겼다.
  • 방향: VFX를 더 넣는 대신 메인/보조 위계를 정하고, 초반 흐름을 출시 준비형 게임 템포에 가깝게 만든다.
  • 행동: 대검 fallback을 보조 잔광으로 낮추고, 무기 데이터/보스 타이밍/스폰/맵 대비/역할 마커를 조정했다.
  • 결과: 기술 검증은 통과했고, 이제 직접 플레이에서 전투 체감만 판단하면 된다.
