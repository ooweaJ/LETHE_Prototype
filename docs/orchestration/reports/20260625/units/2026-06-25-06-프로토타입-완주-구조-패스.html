2026-06-25-06 - 프로토타입 완주 구조 패스

1. 현재 빌드 상태

Dev_Prototype_v1은 이제 “재미 요소를 보는 전투 샌드박스”에서 한 단계 더 나아가, 죽거나 오래 버티거나 마지막 관문을 깨면 결과 화면으로 닫히는 한 판짜리 프로토타입 구조를 갖췄다.

2. 오늘 바뀐 것

  • 사망, 600초 생존, 모든 관문 돌파가 같은 결과 화면으로 이어지게 했다.
  • 네 번째 Gatekeeper를 잡으면 더 기다리지 않고 프로토타입 클리어로 종료되게 했다.
  • 결과 화면에 생존 시간, 처치 수, 관문 진행도, 무기, 레벨, 선택 횟수, 망각 횟수, 잔향 요약을 표시하게 했다.
  • Gatekeeper 처치 시 클리어 버스트 VFX와 카메라 흔들림을 추가했다.
  • HUD에 Gatekeeper 진행도를 추가했다.
  • 디버그 패널은 기본 숨김으로 바꾸고 F12로 열고 닫게 했다.
  • 공격, 피격, 처치, 레벨업, 보스 경고, 보스 처치, 클리어, 패배용 최소 절차적 SFX를 추가했다.

4. 결정한 것

기억/잔향 세부 밸런스와 지형 판단은 보류하고, 먼저 “한 판이 끝난다”는 구조를 완성하는 쪽을 우선했다. 네 번째 관문 처치를 프로토타입 클리어로 삼아 테스트 목표가 분명해지게 했다.

5. 문제 또는 리스크

절차적 SFX는 임시 사운드라 방향 확인용이다. 실제 출시감으로 가려면 나중에 사운드 에셋을 교체해야 한다. 또한 네 번째 Gatekeeper까지 실제 플레이로 도달 가능한지는 직접 테스트가 필요하다.

6. GPT/Claude 인계 요약

Codex가 프로토타입 완주 구조를 추가했다. 다음 판단 기준은 결과 화면이 만족스러운지, Gatekeeper 4관문 클리어 조건이 적절한지, 디버그 패널 숨김이 플레이테스트 빌드처럼 느껴지는지, 최소 SFX가 거슬리지 않는지다.
