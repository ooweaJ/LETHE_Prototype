# 2026-07-01-07 - 기술별 사운드 런타임 적용

## 1. 현재 빌드 상태

`Dev_Prototype_v1`에 무기, 기억, 잔향, 궁극, 픽업, 처치, UI 전투 순간별 원본 합성 사운드를 붙였다. 아직 `_dev` 검증 단계이며 `Assets/Lethe` 승격은 하지 않았다.

## 2. 오늘 바뀐 것

- 기존 sine-only 임시 톤을 더 다양한 절차 생성 사운드 팔레트로 교체했다.
- `sine`, `triangle`, `square`, `noise`, `blade` 계열 파형을 써서 짧고 읽히는 레트로 전투음을 만들었다.
- 반복 사운드가 겹치지 않도록 `PlaySfx(id, volumeMul, minInterval)` 재생 제한을 추가했다.
- 굶주린 칼무리에는 발사, 관통, 잔향 발동음을 따로 붙였다.
- 피의 반사에는 혈반, 회복, 피의 칼폭풍 사운드를 붙였다.
- 처형 섬광, 사냥꾼의 맹세, 파쇄의 파문, 멈춘 1초, 잿빛 방패, 망각의 낙인에도 각각 다른 발동음을 붙였다.
- XP, 처치, 피격, 경고, 레벨업, 클리어, 패배 사운드도 새 팔레트로 갱신했다.

## 3. 테스트 결과와 근거

- `dotnet build LETHE/Assembly-CSharp.csproj --nologo`: 통과, legacy 경고 7개 / 오류 0개.
- Unity MCP compile error count: `0`.
- Unity MCP console error count: `0`.
- Unity Play Mode 직접 `DebugRunM2Smoke()` 스냅샷: `HungryBlades:5`, `BloodReflection:5`, `storm=True`, `enemies=10`, `result=True`.

## 4. 결정한 것

뱀파이어 서바이벌 같은 자동 전투 게임의 방향은 참고하되, 실제 샘플은 복사하지 않는다. LETHE는 짧은 원본 합성음과 재생 제한으로 반복 전투에서도 귀가 피곤하지 않게 간다.

## 5. 문제 또는 리스크

자동 QA는 런타임 안정성만 확인한다. 최종 믹스는 jaewoo가 직접 스피커나 헤드폰으로 들어보고 칼무리 관통음, XP음, 피의 칼폭풍 볼륨이 과한지 판단해야 한다.

## 6. GPT/Claude 인계 요약

기술별 사운드가 들어갔다. 칼무리 계열은 빠른 blade/noise 계열, 피 계열은 낮고 젖은 pulse, 유틸리티 기억은 각 기능이 읽히는 짧은 발동음으로 분리했다. 모든 소리는 런타임 생성이며 외부 게임 음원을 복사하지 않았다.

## 7. 다음 Codex 작업

jaewoo 직접 플레이 후 너무 자주 들리는 소리, 너무 작은 소리, 구분이 안 되는 소리를 하나씩 받아서 볼륨과 `minInterval`을 조정한다.

## 8. 포트폴리오 메모

- 문제: VFX와 전투 리듬은 좋아졌지만 기술별 오디오 피드백이 거의 없었다.
- 방향: 자동 전투에 맞는 짧고 반복 가능한 원본 사운드 팔레트를 만든다.
- 행동: 절차 생성 클립과 재생 제한을 만들고 주요 기술 트리거에 연결했다.
- 결과: 기술별 사운드가 런타임에 적용됐고, 빌드/Unity 검증을 통과했다.
