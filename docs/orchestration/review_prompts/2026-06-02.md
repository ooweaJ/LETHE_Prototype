# LETHE v0.5 구현 후 평가 요청 - 2026-06-02

## 대상

Claude Code에게 v0.5 core-fun 보완 결과가 사람 테스트로 넘어갈 만한지 평가를 요청한다.

## 현재 빌드

- 현재 빌드: LETHE HTML alpha v0.5 core-fun human-test candidate.
- 핵심 루프: 기억 3개 선택 → 전투 → 처치 XP/런 중 성장 선택 → 보스 처치 → 사라질 기억 예측 → 의존도 기반 기억 상실 → 잔향 보상 → Q1/Q2/Q3 설문 → JSON 로그 다운로드.
- 구현 범위: 무기 2종, 기억 6종, 활성 기억 슬롯 3개, 적 4종, 3페이즈 보스 1종.
- 범위 제한은 유지한다: 메타 진행, 상점, 최종 보스, 신규 기억, 신규 적, 다중 지역 구조는 만들지 않는다.

## 사용자 피드백

v0.4는 망각/잔향 전달은 좋아졌지만, 실제 테스트에서 “기억이고 뭐고 초반이 너무 루즈하다”, “로그라이크 맛이 없다”, “Vampire Survivors처럼 적이 계속 몰려오고 성장하는 맛이 없다”는 피드백이 나왔다.

따라서 v0.5는 망각 감정 이전에 “재미있어서 9분 첫 망각까지 가는가”를 보완하는 데 집중했다.

## v0.5에서 구현한 것

- 초반 적 스폰을 pack 단위로 촘촘하게 바꿨다.
- 적 처치 시 XP를 준다.
- XP가 차면 전투를 잠시 멈추고 3택 1 런 성장 선택을 띄운다.
- 성장 선택은 이번 런 안에서만 유지된다.
- 성장 후보는 피해, 공격 속도/쿨다운, 범위/넉백, 생존, XP 획득 보너스다.
- JSON payload에 `runGrowth`를 추가했다.
- AI test에 early-fun 지표를 추가했다:
  - `Early Fun Score`,
  - `earlyKillTempo`,
  - `earlyLevelUps`.
- OpenAI API fallback은 제거했고, 기획 검토는 Claude Code 우선, Codex CLI fallback 순서로 정리했다.

## 알려진 리스크

- Browser plugin QA는 Windows sandbox startup 오류로 직접 확인하지 못했다.
- 로컬 브라우저에서 v0.5 레벨업 선택 화면과 JSON `runGrowth` payload는 아직 사람이 직접 확인해야 한다.
- 망각 직후 전투력 딥은 `27.8-28.0%`로 기존 목표 30%보다 낮다.
- 현재 런 성장은 공용 스탯 중심이다. “특정 기억에 투자했는데 그 기억이 사라져서 아깝다”는 상호작용은 아직 약하다.
