# LETHE v0.6 코어 런 구조 구현 결과 판단 요청 - 2026-06-02

## 목적

v0.6은 v0.5 사람 테스트를 보류하고, GPT/Claude가 공통으로 제안한 “빠른 첫 망각 + 결손 생존 + 잔향 피벗 + 기억 보충” 코어 사이클을 HTML 프로토타입에 구현한 버전이다.

Claude는 단순 리뷰어가 아니라 테스트 결과를 해석해 다음 자동 버전업 방향을 정하는 기획 파트너로 답한다. 지금 판단해야 할 것은 v0.6을 사용자 1인 체감 테스트로 넘겨도 되는지, 아니면 Codex가 v0.7 자동 보완을 한 번 더 해야 하는지다.

## v0.6에서 구현한 구조

- 브라우저 표기와 experiment version을 `v0.6`으로 변경했다.
- 총 런 구조를 20분 기준으로 잡았다.
- 보스 스케줄을 `4 / 8 / 12 / 16 / 20분`으로 바꿨다.
- 보스 처치 후 의존도 기반으로 기억 1개를 잃는다.
- 망각 후 바로 결과 종료가 아니라, 기억 2개 상태로 결손 생존 구간에 들어간다.
- 결손 생존 구간은 기본 105초다.
- 결손 구간을 버티면 새 기억 후보 3개 중 1개를 선택해 다시 기억 3개 상태로 복귀한다.
- 잃은 기억은 기존 echo/잔향 효과로 남아 빌드를 완전히 잃는 대신 다른 방향으로 피벗하게 했다.
- 최종 각인 시스템은 넣지 않았고, 결과 화면에는 단순 런 결산만 추가했다.
- `runTimeline` JSON payload에 boss schedule, cycle, forgotten history, refill choices를 기록한다.
- `?qa=fast,v06` 브라우저 QA 모드를 추가했다.

## 구현하지 않은 것

- 메타 진행 없음.
- 상점 없음.
- 신규 기억 없음.
- 신규 무기 없음.
- 복잡한 각인 시스템 없음.
- Unity 전환 없음.
- 5-8명 사람 테스트 없음.

## 알려진 리스크

- 실제 사람 플레이 감각은 아직 확인하지 않았다.
- 예측 일치율이 quick에서는 `92.5%`로 기존 상한보다 높아, 너무 정답 맞히기처럼 느껴질 가능성이 있다.
- 결손 생존 구간은 수치상 80% 전후라 적당해 보이지만, 실제로는 “흥미로운 압박”인지 “그냥 기다리는 구간”인지 확인이 필요하다.
- 잔향은 기존 echo 스탯 기반이라, 기억별 변형이 플레이어 눈에 충분히 강하게 보이는지는 아직 불확실하다.
- 최종 20분 클리어 결산은 구조만 있고, 아직 완성된 엔딩 감정은 없다.
