# LETHE v0.7.1 밸런스 현실 검증 요청 - 2026-06-02

## 배경

v0.7은 v0.6 사용자 피드백인 “기억 상실 후 너무 약해져 쫄몹을 못 잡는다”를 해결하기 위해 무기 기본 성능과 잃은 기억의 무기 잔향을 보강했다.

하지만 사용자가 실제로 플레이한 뒤 다음과 같이 피드백했다:

> 밸런스가 1도 안 맞는다. 어떤 밸런스를 피드백 받고 만든 거냐?

이 피드백은 v0.7 자동 밸런스 판정이 실제 플레이 체감과 크게 어긋났다는 뜻이다.

## 중요한 정정

v0.7에서 받은 Claude 피드백은 실전 플레이 밸런스 피드백이 아니었다.

- Codex가 Claude에게 넘긴 입력은 실제 플레이 로그가 아니라 AI 시뮬레이터 지표와 구현 요약이었다.
- Claude는 그 지표를 근거로 `GO_TO_SOLO_PLAYTEST`를 판단했다.
- 따라서 “Claude가 밸런스를 맞춰줬다”가 아니라 “부정확한 AI proxy 지표를 보고 사람 체감 테스트로 넘기라고 했다”가 정확하다.

## 현재 의심 원인

브라우저 실제 게임과 AI 시뮬레이터 사이에 다음 괴리가 있다.

1. 실제 게임의 결손 구간은 기억이 2개로 줄었을 때 오히려 스폰 압박이 올라간다.
   - `activeMemoryCount() < 3`이면 spawn rate가 빨라진다.
   - pack size도 3으로 고정된다.
2. 보스전에서 남은 적/소환몹/원거리 투사체/힐러 압박이 실제 체감에 크게 작용한다.
3. AI 시뮬레이터는 실제 조작, 적 누적, 화면 공간 압박, 접촉 피해, 투사체 회피, 힐러 회복을 충분히 재현하지 않는다.
4. v0.7의 무기 보강은 단순 DPS 보정에 가까웠고, 실제 플레이의 “몰려오는 적을 정리하는 맛”과 “위기에서 빠져나오는 수단”을 보장하지 못했다.
5. 현재 자동 QA는 기능 완료 여부를 확인할 뿐, 실제 밸런스 플레이를 자동으로 수행하지 않는다.

## 판단 요청

Claude/GPT는 이번에는 AI 지표를 그대로 신뢰하지 말고, “실제 플레이 체감이 틀렸다”는 사용자 피드백을 최상위 근거로 판단한다.

1. v0.7.1에서 가장 먼저 고쳐야 할 것은 무엇인가?
   - 결손 구간 스폰 압박
   - 적 HP/속도/접촉 피해
   - 무기 기본 성능
   - 무기별 특수기/위기 탈출 수단
   - 기억 상실 후 잔향 보상
   - 레벨업 성장 선택
   - 실제 브라우저 밸런스 QA 자동화
2. v0.7.1에서 새 무기/특수기를 넣어야 하는가, 아니면 현재 2무기와 6기억 안에서 먼저 조정해야 하는가?
3. AI test를 어떻게 바꿔야 “밸런스가 맞는 척하는 지표”가 아니라 실제 플레이 위험을 더 잘 잡는가?
4. 다음 Codex 구현은 어떤 순서로 해야 하는가?
