운영 절차

RUNBOOK.md

반복 명령은 Markdown 원본에서 관리하고, 이 화면은 사람이 빠르게 보고 실행 순서를 고르기 위한 요약입니다.

상태 점검

문서나 자동화가 깨졌는지 넓게 확인합니다.

npm run doctor
npm run report:check

플레이테스트 패키지

사람 테스트 전 패키지를 다시 만들거나 확인합니다.

npm run playtest:package:dry
npm run playtest:package

사람 테스트 요약

다운로드된 JSON 로그를 playtest_logs/에 둔 뒤 실행합니다.

npm run playtest:summary

밸런스 기준

현재 기준은 one-off가 아니라 loop 결과입니다.

npm run balance:loop
npm run qa:balance

보고서 생성

의미 있는 작업 단위가 끝났을 때 Markdown 원본에서 HTML을 재생성합니다.

npm run report
npm run report:check
npm run report:discord:unit:dry

실제 보고 전송

정책, webhook, 네트워크, 권한 문제로 실패하면 이유와 다음 명령을 devlog/report에 남깁니다.

npm run report:discord:unit

오케스트레이션 HTML

원본 Markdown 갱신 후 사람이 보는 HTML 3종도 같이 갱신합니다.

docs\orchestration\index.html
docs\orchestration\command.html
docs\orchestration\runbook.html

커밋 기준

검증과 보고가 끝난 coherent unit만 커밋/푸시합니다.

git status --short
git add <files>
git commit -m "docs: ..."
git push
