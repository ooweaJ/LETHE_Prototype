# AI Collaboration Docs

이 폴더는 LETHE에서 AI를 어떻게 통제하고 검증했는지 남기는 포트폴리오용 근거 문서다.

핵심 원칙은 “AI를 사용했다”가 아니라 “AI 제안을 개발자가 검증하고, 틀린 부분을 고치고, 테스트 결과로 다음 결정을 내렸다”를 증명하는 것이다.

## Files

- `ai-collaboration.md`: AI 협업 규약과 역할 분리.
- `ai-failure-cases.md`: AI 또는 자동화가 틀렸거나 막힌 사례와 개발자가 수정한 내용.

## Required Evidence Flow

기능 단위마다 아래 흐름 중 필요한 것을 남긴다.

1. 문제 정의.
2. 제약 조건.
3. AI에게 준 요청 또는 프롬프트.
4. AI 답변의 유효한 부분.
5. AI 답변의 문제, 과설계, 누락, 실패.
6. 개발자가 수정한 최종 판단.
7. 검증 결과.

모든 대화 원문을 저장하지 않는다. 설계가 바뀐 순간, AI가 틀린 순간, 테스트 기준이 바뀐 순간, 사람 테스트 결과로 기획이 바뀐 순간만 남긴다.
