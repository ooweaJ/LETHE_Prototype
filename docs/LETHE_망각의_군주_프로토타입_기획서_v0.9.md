# LETHE: 망각의 군주

HTML 프로토타입 기획서 v0.9

최종 갱신: 2026-06-04

이 문서는 기존 `LETHE_망각의_군주_완성형_기획서_v6.docx`를 현재 HTML Alpha v0.9 구현 기준으로 갱신한 원본이다. 현재 목표는 완성형 게임 설계 확정이 아니라, HTML 프로토타입에서 LETHE의 핵심 재미와 Unity 전환 가능성을 검증하는 것이다.

## 1. 현재 판정

- 현재 빌드: LETHE HTML Alpha v0.9.
- 구현 파일: `index.html`, `style.css`, `src/game.js`.
- 핵심 검증 질문: 플레이어가 선택한 기억 3개 중 가장 의존한 기억을 잃었을 때, 그 경험이 짜증보다 아쉬움, 납득, 피벗 욕구에 가까운가?
- 최신 상태: v0.9 release-feel 루프의 AI quick/full 기준과 WP2/WP3 trusted browser gates는 통과했다.
- 다음 판단 단계: controlled 1-person browser playtest. 아직 Unity 전환 판정이나 broad public test 단계가 아니다.
- 범위 원칙: 새 기억, 새 슬롯, 상점, 메타 진행, 새 지역, 새 무기, 새 적, 최종 보스, 다영역 구조는 추가하지 않는다.

## 2. 현재 플레이 루프

1. 시작 화면에서 무기 1개와 기억 3개를 고른다.
2. 전투 중 플레이어는 이동만 직접 조작한다. 기본 공격과 기억 발동은 자동이다.
3. 적을 처치하면 XP를 얻고, 런 중 레벨업 3지선다를 선택한다.
4. 보스 전까지 전투 압박은 `숨 고르기`, `압박 상승`, `망각 전조`로 상승한다.
5. 보스 처치 후 플레이어는 `가장 잃고 싶지 않은 기억`과 `레테가 가져갈 것 같은 기억`을 답한다.
6. 시스템은 의존도 계산으로 가장 기대고 싸운 기억을 망각시킨다.
7. 잃은 기억은 능동 발동을 멈추고, 전투 성향 일부가 잔향 패시브로 남는다.
8. 플레이어는 75초 동안 기억 2개와 잔향만으로 결손 생존 구간을 버틴다.
9. 결손 구간 후 후보 기억 중 1개를 보충해 다시 3슬롯으로 돌아온다.
10. 이 사이클을 반복하며 최종 결과 화면과 JSON 로그로 감정/공정성/기억 회상을 기록한다.

## 3. 조작과 전투 감각

- 이동: WASD 또는 방향키.
- 기본 공격: 선택 무기에 따라 자동 발동.
- 기억 발동: 기억별 쿨다운과 조건에 따라 자동 발동.
- 전술 집중: 전투 중 현재 활성 기억 슬롯을 클릭하거나 `Digit1`-`Digit3`을 눌러 한 기억을 짧게 집중시킨다.
- 전술 집중 효과: 쿨다운형 기억은 다음 발동이 당겨지고, 상시/온힛형 기억은 기존 효과만 짧게 강화된다.
- 설계 의도: 자동전투의 편안함은 유지하되, 플레이어가 자기 빌드를 읽고 작은 타이밍 선택을 할 수 있게 한다.

## 4. 무기 시스템

무기는 런 동안 삭제되지 않는 플레이어의 본질이다. 기억이 사라져도 무기는 전투의 기본 감각을 유지하고, 잔향에 따라 후반 운용이 달라진다.

| 무기 | 역할 | 현재 구현 의도 |
| --- | --- | --- |
| 절단쌍검 | 빠른 근접 / 온힛 | 빠른 타격과 넓어진 기본 베기로 기억 결손 구간에서도 잡몹을 정리하는 안정형 무기 |
| 장송대검 | 느린 강타 / 폭딜 | 긴 사거리와 강한 전방 정리로 결손 구간에서 한 줄을 뚫는 돌파형 무기 |

현재 프로토타입은 이 2종만 유지한다. 무기 확장은 코어 재미와 Unity 전환 근거가 충분할 때 다시 판단한다.

## 5. 기억 시스템

현재 기억은 6종이며, 플레이어는 시작 시 3개를 고른다. 기억은 능동 전투 행동이고, 망각되면 능동 발동은 사라지지만 잔향이 남는다.

| 기억 | 역할 | 태그 | 현재 전투 기능 | 잔향 방향 |
| --- | --- | --- | --- | --- |
| 처형자의 섬광 | 버스트 | 폭딜 | 위협 적 하나에 주기적 강타 | 치명타, 단발 폭딜, 무기 섬광 |
| 굶주린 칼무리 | 근접 도트 | 광역, 지속 | 근접 적을 계속 베는 칼날 | 공격속도, 지속 피해, 무기 출혈 |
| 추적자의 맹세 | 추적 다중 | 폭딜 | 위협 적을 쫓는 투사체 | 투사체 수, 속도, 무기 유도 |
| 파쇄의 파문 | 광역 / 넉백 | 광역, 제어 | 주변 적을 밀치고 충돌 피해 | 범위, 넉백, 피해 감소, 무기 충격파 |
| 피의 반사 | 온힛 증폭 | 생존, 지속 | 기본 공격마다 붉은 추가타 | 추가타, 온힛 피해, 무기 출혈 |
| 멈춘 초침 | 제어 | 제어, 지속 | 주변 둔화와 기억 쿨다운 단축 | 쿨다운 감소, 둔화 지속, 무기 둔화 |

## 6. 빌드 정체성과 시너지

v0.9의 핵심 보강은 플레이어가 자신이 만든 빌드를 빠르게 알아보는 것이다. 선택 화면과 HUD는 현재 빌드명, 활성 시너지, 의존 중인 기억을 보여준다.

현재 시너지 규칙:

| 시너지 | 필요 태그 | 효과 방향 |
| --- | --- | --- |
| 압축 파문 | 광역 + 제어 | 둔화되거나 밀린 적에게 광역 피해 증가 |
| 느린 출혈 | 지속 + 제어 | 둔화된 적에게 지속 피해 강화 |
| 피 묻은 결의 | 폭딜 + 생존 | 큰 피해 후 짧은 회복 |

시너지 목적은 단순 DPS 합산이 아니라, “내 빌드가 어떤 전투 문장으로 작동하는지”를 첫 망각 전 90초 안에 읽히게 하는 것이다.

## 7. 런 성장

런 중 성장은 메타 진행이 아니라 해당 런 안에서만 유지되는 선택이다.

현재 레벨업 선택:

| 선택 | 효과 |
| --- | --- |
| 칼날 가속 | 공격 속도 +11%, 기억 호흡 +4% |
| 검은 물의 힘 | 피해 +14% |
| 파문 확장 | 범위 +12%, 넉백 +8% |
| 가라앉지 않는 숨 | 최대 HP +12, 즉시 회복, 피해 감소 +4% |
| 기억 흡입 | 경험치 획득 +16% |

설계 목표는 초반 1-3분을 루즈하지 않게 만들고, 첫 망각 전에 플레이어가 빌드 성장 선택을 최소 2회 이상 체감하게 하는 것이다.

## 8. 적과 압박 리듬

현재 적은 4종이다.

| 적 | 역할 |
| --- | --- |
| 침식자 | 기본 추격 압박 |
| 떠도는 눈 | 느린 원거리/투사체 압박 |
| 쪼개진 자 | 중형 근접 압박과 분열 성향 |
| 공허 사제 | 후방 지원/회복 성향 |

보스 사이의 일반 전투는 `숨 고르기`, `압박 상승`, `망각 전조`로 나뉜다. 기억이 2개로 줄어든 결손 상태에서는 `결손 정비`와 `결손 압박`으로 별도 리듬을 사용한다.

## 9. 보스와 망각 사이클

현재 기본 런 길이는 540초이며, 보스 스케줄은 90초, 210초, 360초, 510초다. QA fast 모드는 별도 단축 스케줄을 사용한다.

보스 명칭:

- 1차: 작은 문지기.
- 중간 사이클: 기억을 씹는 자 2, 기억을 씹는 자 3.
- 마지막 사이클: 끝의 문지기.

보스는 3페이즈를 가지며, 전투 후 망각 질문과 의존도 계산을 트리거한다. 현재 기획의 목적은 보스 자체의 완성도가 아니라 “보스 직후 망각 판단이 납득되는가”를 검증하는 것이다.

## 10. 의존도 기반 망각

망각은 단순 딜 1위 삭제가 아니다. 현재 구현은 다음 성분을 합산한다.

- 전투 기여: 피해, 처치, 어시스트, 상태 기여.
- 보스 기여: 보스 피해, 그로기 피해, 보스 제어.
- 대체 불가능성: 총 피해/처치에서 차지한 비중.
- 존재감: 활성 횟수와 전투 중 존재 시간.
- 기억별 삭제 bias: 특정 기억 삭제 쏠림을 낮추기 위한 보정.

결과 화면은 사라진 행동, 남은 잔향, 의존도 상세, 예측 적중 여부를 분리해 보여준다. 목표는 “내가 기대고 싸운 기억이 사라졌다”가 읽히되, 플레이어가 억울함보다 후회와 피벗 욕구를 느끼게 하는 것이다.

## 11. 잔향과 기억 보충

망각된 기억은 능동 발동을 100% 잃는다. 대신 전투 성향 일부가 잔향으로 남아 다음 빌드 방향을 밀어준다.

현재 기본 잔향 파워 다이얼은 `echoPower = 0.50`이다. 잔향은 완전 보상이 아니라 상실 후 변형이다. 따라서 결과 화면은 “삭제 보상”보다 “사라진 행동과 남은 흔적”을 분리해 표현한다.

망각 후에는 75초 결손 생존을 진행한다. 이 구간은 기억 2개와 잔향만으로 버티는 post-loss challenge이며, 완료 후 후보 기억을 하나 골라 3슬롯으로 복귀한다. 복귀는 이전 빌드의 원상복구가 아니라 잔향 위에 새 방향이 붙는 피벗이어야 한다.

## 12. UI와 로그

현재 UI가 반드시 보여야 하는 정보:

- 선택 화면: 무기 역할, 기억 역할/짧은 설명/태그.
- 전투 HUD: 현재 빌드명, 활성 시너지, 의존 중인 기억.
- 기억 슬롯: 선명도, 의존도, 전술 집중 가능 상태.
- 망각 결과: 사라진 행동, 잔향 변형, 결손 생존 안내.
- 최종 결과: Q1 아쉬움, Q2 납득도, Q3 사라진 기억 이름 회상.

JSON 로그는 `playtest`, `runGrowth`, `runTimeline`, `buildIdentity`, `tacticalFocus`, `danger`, `death`, `echoTransformation`, 질문/예측/망각 결과를 포함한다.

## 13. 검증 기준

현재 자동 검증은 감정을 확정하지 않는다. AI 지표는 수렴/불공정/리듬 문제를 찾는 planning evidence이고, browser QA는 구현 흐름 증거다. 인간 감정과 Unity 전환 가능성은 사람 플레이테스트로 판단한다.

현재 통과한 주요 게이트:

- `npm run ai:test:quick`: GO_CANDIDATE, Alpha Fun Score 0.8904, earlyChoiceInterest 0.7684, tacticalFocusAgency 0.7309.
- `npm run ai:test`: GO_CANDIDATE, Alpha Fun Score 0.8937.
- `npm run qa:postloss:trusted`: post-loss challenge와 refill 흐름 통과.
- `npm run qa:tactical:trusted`: 전술 집중 사용, payload, UI 표시 통과.

다음 controlled 1-person test에서 볼 질문:

- 초반 1-3분이 적을 계속 잡고 싶게 만드는가?
- 전술 집중이 자동전투 안의 작은 선택으로 읽히는가?
- 망각 직후 결손 생존이 아쉬움/긴장으로 읽히는가, 짜증으로 읽히는가?
- 잔향과 기억 보충이 원상복구가 아니라 상실 후 피벗으로 이해되는가?
- HTML 프로토타입인데도 다른 빌드를 다시 해보고 싶은가?

## 14. 현재 제외 범위

다음 항목은 GPT나 사용자가 명시적으로 범위를 바꾸기 전까지 구현하지 않는다.

- 메타 진행.
- 상점 시스템.
- 최종 보스 완성.
- 현재 6개를 초과하는 기억.
- 3개를 초과하는 활성 기억 슬롯.
- 다영역 런 구조.
- 새 무기, 새 적, 새 지역.
- 기억 합성, 강화, 업그레이드 시스템.
- 해금/저장/캠페인 구조.
- 대형 튜토리얼 또는 스토리 컷신.

## 15. Unity 전환 게이트

Unity 전환은 현재 자동 지표만으로 결정하지 않는다. 최소 조건은 다음과 같다.

- 5-8명 사람 테스트에서 초반 1-3분 재미 반응이 대체로 긍정적이다.
- 첫 망각까지 도달한 플레이어가 “짜증”보다 “아까움/후회/납득”에 가깝게 말한다.
- 망각 후 잔향과 보충이 완전한 무효화가 아니라 상실 후 적응으로 읽힌다.
- 일부 플레이어가 다른 기억 조합이나 무기로 다시 해보고 싶다고 말한다.
- 위 결과를 Claude/GPT에 넘겨 Unity 전환 여부와 다음 설계 방향을 다시 판단한다.

## 16. 문서 차이 정리

기존 v6 DOCX와 현재 게임은 같지 않았다. 가장 큰 차이는 다음과 같다.

- 기존 문서는 v0.2 1회 망각 검증 중심이었고, 현재 게임은 v0.9 release-feel 루프다.
- 현재 게임에는 런 중 레벨업, 빌드 정체성, 압박 리듬, post-loss challenge, 기억 보충, 전술 집중이 구현되어 있다.
- 기존 문서에는 메타 진행과 완성형 확장 방향이 포함되어 있었지만, 현재 프로토타입 범위에서는 명시적으로 제외한다.
- 현재 검증 기준은 AI/browser gate 통과 후 controlled 1-person playtest이며, Unity 전환은 아직 판단하지 않는다.
