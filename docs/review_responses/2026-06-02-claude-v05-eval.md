## 결론

- `ITERATE_BEFORE_TEST` — 단, 좁은 범위의 검증 1건만 통과하면 즉시 `GO_TO_HUMAN_TEST`.

## 이유

- AI 지표는 사람 테스트 진입 조건을 모두 충족한다. Early Fun 0.87 / kill tempo 0.96 / 레벨업 4.08회는 "초반이 루즈하다"는 v0.4 피드백을 설계상 정면으로 해결한 신호다. 망각 감정(regret 81.6%, irritation 0.3%, prediction 84.8%)도 유지됐다.
- 그러나 **사람이 실제로 만질 v0.5 신규 화면(레벨업 3택, runGrowth payload)이 브라우저에서 한 번도 사람 눈으로 확인되지 않았다.** 이게 깨져 있으면 사람 테스트 세션 전체가 무효가 된다. 지표가 아무리 좋아도 이건 사람 테스트의 전제 조건이지 관찰 대상이 아니다.
- 따라서 막는 것은 "설계"가 아니라 "동작 확인 미완"이다. 그래서 FIX_CORE가 아니라 ITERATE이고, 검증만 통과하면 바로 GO다.
- 망각 직후 딥 28%(목표 30%)와 "기억별 투자-망각 아쉬움"은 둘 다 **지금 손대면 안 되는 항목**이다. 전자는 사람 인지 한계 안의 차이, 후자는 이번 라운드 검증 목표(초반 재미)와 다른 축이라 섞으면 무엇을 검증했는지 흐려진다.

## 앞으로 해야 할 일 (사람 테스트 전 필수 작업)

- [ ] 로컬 브라우저에서 v0.5 레벨업 3택 화면이 실제로 뜨고, 선택 후 전투가 정상 재개되는지 사람이 직접 1런 끝까지 확인한다. (Windows sandbox 우회: 일반 브라우저 수동 실행으로라도 확인)
- [ ] 한 런 완주 후 다운로드된 JSON에 `runGrowth` 필드가 실제 선택 내역과 일치하게 기록되는지 눈으로 확인하고, 깨지면 그것만 수정한다.
- [ ] 레벨업으로 전투가 멈출 때 적 스폰/타이머가 함께 일시정지되는지, 재개 시 입력이 먹는지(포커스/키 잔류) 확인한다. 이 흐름 끊김만 사람 테스트 직전 점검 항목으로 남긴다.

## 사람 테스트 관찰 기준

- 초반 재미: 첫 2-3분 안에 "적이 몰려오고 내가 강해진다"는 Vampire Survivors식 손맛을 입으로 말하는가. 9분 첫 망각까지 가는 동안 "지루하다/언제 끝나냐"가 나오는가.
- 성장 선택: 3택을 고민하고 고르는가, 아니면 아무거나 누르는가. 고른 성장이 "내 빌드"처럼 느껴진다고 말하는가(공용 스탯이라 무미건조하다는 반응이 나오는지 기록).
- 기억/망각: 사라질 기억 예측 시 진지하게 고민하는가. 망각 순간 "아 아깝다(후회)" vs "짜증난다"가 어느 쪽인가 — 표정/탄식까지 기록.
- 잔향/회복: 교체 직후 약해진 걸 체감하는가, 그리고 다시 강해졌다고 느끼는가. 28% 딥이 "거의 안 느껴진다"인지 "충분히 아쉽다"인지를 입으로 확인(이걸로 30% 보정 여부를 다음 라운드에 판단).

## 아직 만들지 말 것

- 기억별 투자-망각 상호작용 시스템(특정 기억 강화 → 그 기억 상실 시 손실). 방향은 맞지만 이번 라운드 검증 축이 아니다. 사람 테스트에서 "공용 스탯이라 아쉽다"는 반응이 실제로 나오는지부터 확인하고 v0.6 후보로 보류.
- post-forgetting power drop의 28%→30% 수치 튜닝. 사람 체감 결과를 먼저 받는다.
- 기존 범위 제한 유지: 메타 진행, 상점, 최종 보스, 신규 기억, 신규 적, 다중 지역 구조.
