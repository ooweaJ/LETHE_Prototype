## 결론

**BUILD_V09_RELEASE_FEEL_LOOP**

현재 자동화 파이프라인은 작동하지만 "목표 → 구현 → 검증 → 판단 → 다음 입력" 루프가 실제로 돌지 않았다. AI verdict는 planning pass로만 취급하고, 지금 당장 9분 플레이에서 출시작 감각을 만드는 데 집중한다. 자동화 수리나 추가 조사보다 구체적인 빌드 개선이 먼저다.

## v0.9 재미 목표

9분 안에 플레이어는 세 번의 "선택 고점"을 경험해야 한다. 첫 번째는 기억 3개를 처음 채웠을 때 빌드 방향이 보이는 순간, 두 번째는 첫 번째 망각 직전 "이 기억은 살리고 싶다"는 아쉬움, 세 번째는 망각 이후 결손 상태에서 echo zone이나 보스 tell을 활용해 한 번 이긴 순간이다. 이 세 고점이 없으면 망각은 그냥 페널티고, 있으면 망각이 다음 시도의 동기가 된다. AI verdict가 아니라 "처음 하는 사람이 스스로 한 판 더 하고 싶어지는가"가 유일한 기준이다.

## Codex 작업 단위

### A. 기억 식별성 · 시너지 (90초 이내 빌드 방향 인식)

- [ ] **A1** 기억 카드에 태그 1개 + 한 줄 시너지 힌트 추가 (예: `[압박]`, `[잔향]`, `[회복]` + "압박+잔향 → DPS 20% 증폭"). 텍스트만 수정, UI 구조는 건드리지 않는다.
- [ ] **A2** 슬롯 3개가 채워졌을 때 빌드 태그 조합 요약 한 줄을 화면 상단에 1.5초 표시 (예: "압박 특화 빌드"). 이 한 줄이 없으면 플레이어는 자기 빌드를 모른다.

### B. 망각 → 도전 욕구 전환

- [ ] **B1** 망각 발동 시 "결손 챌린지" 알림 추가: "기억 [X]가 사라졌다. 다음 60초 안에 echo zone을 밟으면 약화된 형태로 복구된다." 복구 성공 여부와 무관하게, 목표 자체가 짜증을 도전 욕구로 바꾼다.
- [ ] **B2** 망각 직후 10초간 적 스폰 밀도 20% 감소 ("숨 고르기 창"). 이 창이 없으면 망각과 압박이 동시에 오면서 좌절만 남는다.
- [ ] **B3** 결손 상태 전용 패시브 1개 추가: 기억 슬롯이 2개 이하일 때 이동속도 +10%, 공격 판정 +15%. 약해진 것이 아니라 "다른 방식으로 강해진 것"으로 느끼게 한다.

### C. 자동전투 수동 agency (최소 비용)

- [ ] **C1** echo zone 위치를 미니맵에 고정 표시, 진입 시 기억 회복/강화 pulse 발동. 플레이어가 직접 움직여야 하는 유일한 이유를 만든다.
- [ ] **C2** 보스 공격 전 1.5초 tell 애니메이션(붉은 원 범위 표시) 추가. 피하면 5초 보호막, 맞으면 기억 1개 즉시 약화. 이 tell이 있어야 "내가 판단해서 피했다"는 agency가 생긴다.

### D. 밤샘 루프 자동화

- [ ] **D1** `scripts/run_overnight_loop.js`에 루프 1회 단위 정의: ① 빌드 실행 → ② AI quick test (5회) → ③ 지표 수집 (생존시간, 망각 횟수, 기억 슬롯 평균) → ④ 임계값 비교 → ⑤ `docs/loop_runs/YYYY-MM-DD-HHmm.md` 생성 → ⑥ 다음 루프 입력 파일 업데이트.
- [ ] **D2** 루프 실패 감지 기준 파일 `docs/loop_runs/thresholds.json` 생성: `{ "min_survival_sec": 420, "max_amnesia_frustration_rate": 0.3, "min_echo_zone_use_rate": 0.4 }`. 이 수치를 넘지 못하면 다음 루프에서 해당 항목을 자동으로 review_prompt에 포함한다.
- [ ] **D3** 사람이 깨어났을 때 볼 문서를 `docs/loop_runs/LATEST_SUMMARY.md`로 고정. 이 파일만 열면 밤새 루프 결과, 실패 항목, 다음 권장 작업이 3단락 이내로 보여야 한다.

---

## 밤샘 루프 기준

**루프 1회 실행 내용:**
1. 현재 빌드로 AI quick test 5회 실행
2. 생존 시간 중앙값, 망각 발동 횟수, echo zone 사용 비율, 결손 챌린지 성공 비율 수집
3. `thresholds.json` 기준과 비교
4. 통과 항목 / 실패 항목 분리하여 `loop_runs/YYYY-MM-DD-HHmm.md` 기록
5. 실패 항목이 있으면 해당 항목을 다음 루프의 `review_prompts` 입력에 자동 포함

**루프 실패 시 다음 루프 입력으로 남겨야 할 것:**
- 실패한 지표 항목명 + 측정값 + 임계값 차이
- "이 항목이 실패한 원인 가설 1-2개" (AI가 생성, 사람이 검증 안 해도 됨)
- 다음 루프에서 시도할 변경사항 1개 (작업 단위 A-D 중 하나와 매핑)

**사람이 깨어났을 때 볼 문서:**
`docs/loop_runs/LATEST_SUMMARY.md` 하나만. 이 파일에는 ① 밤새 루프 횟수 ② 현재 통과/실패 지표 ③ 남은 작업 중 우선순위 1개가 들어 있어야 한다. 나머지 상세 로그는 링크로만 참조한다.

## 테스트 기준

**AI test:**
- 생존 시간 중앙값 ≥ 7분 (420초)
- 망각 이후 echo zone 진입 비율 ≥ 40%
- 기억 슬롯 평균 ≥ 1.8 (결손 상태가 지배적이지 않아야 함)
- 위 3개 모두 통과 → 다음 루프 유지 / 하나라도 실패 → 해당 항목 집중 수정

**Browser QA:**
- 태그+시너지 힌트가 모바일 해상도(375px)에서 잘리지 않는지 확인
- 망각 알림과 결손 챌린지 텍스트가 전투 중 읽히는지 확인 (3초 이내 인식 가능 여부)
- echo zone 표시가 미니맵과 화면 동시 표시될 때 겹치지 않는지 확인

**수동 1인 테스트 (사람이 깨어난 직후):**
- 처음 플레이하는 척 하고 3판 연속 플레이
- 망각 발동 후 "다시 하고 싶다" vs "짜증난다" 즉각 메모
- echo zone을 자발적으로 찾아갔는가 (지시 없이)
- 9분을 채우지 못했다면 어디서 멈췄는지 기록
