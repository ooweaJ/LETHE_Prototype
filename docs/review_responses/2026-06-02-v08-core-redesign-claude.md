## 결론

- **`FIX_TEST_MODEL_FIRST`** — 단, "전체 테스트 인프라를 다 갈아엎고 나서 기획" 이라는 뜻이 아니라, **HP 사망 버그 + 핵심 사망 지표 6종을 먼저 고치는 얇은 게이트(1~2일)를 통과시킨 뒤, 같은 v0.8 사이클 안에서 코어 재설계로 바로 이어간다**는 의미다.

## 이유

- 지금 모든 AI 생존 데이터는 "HP 1 고정" 버그로 오염돼 있다. 이건 밸런스 문제가 아니라 **정확성 버그**다. 이걸 고치기 전에 기억 예산/시너지/잔향을 재설계하면, 또 한 번 "AI proxy pass"라는 가짜 통과 신호로 v0.9에서 같은 실패를 반복한다.
- 반대로 "테스트 모델 전체를 완성한 뒤 기획"은 너무 오래 막힌다. 사용자가 직접 플레이로 이미 "밸런스가 무너졌다"는 명확한 신호를 줬으므로, 재설계 필요성은 검증 대상이 아니라 **확정**이다. 따라서 테스트 게이트는 최소한으로만 먼저 통과시키면 된다.
- 사망 버그를 고치는 순간 사망률/사망 시각/처치 공백/적 누적/HP 압박 시간이 측정 가능해지고, 이게 곧 "기억 6개 예산이 정말 비슷한가"를 검증하는 자(尺)가 된다. 자(尺) 없이 예산 재설계는 불가능하다.
- 사용자 피드백(초반 루즈, 기억 격차, 시너지 부재)은 전부 **코어 재설계 사안**이지 v0.7.1 수치 보정으로 안 풀린다. `SMALL_V071_ONLY`는 탈락.

## v0.8 핵심 설계

### 런 길이 — 8~9분 검증 런 (방향 OK)
20분은 프로토타입 검증용으로 과하다. 8~9분 권장 타임라인:

| 시각 | 이벤트 |
|---|---|
| 0:00–1:30 | 초반 전투 (몹 점증) |
| 1:30 | **미니보스** 등장 |
| ~2:00 | **첫 망각** (미니보스 처치 트리거, 기억 3→2) |
| 2:00–3:15 | 결손 생존 (≈75초, v0.7의 105초보다 단축) |
| 3:15 | 기억 보충 (후보 3 중 1, →3기억) |
| 3:15–5:15 | 보스1까지 (간격 2:00) |
| 5:15 | 보스1 |
| 5:15–7:45 | 보스2까지 (간격 2:30) |
| ~7:45–8:30 | 검증 런 종료점 |

- **첫 망각 타이밍**: 정확히 초로 고정하기보다 **"첫 미니보스 처치 = 첫 망각" 트리거**로 묶고, 미니보스를 1:30에 배치해 실제 망각이 **2:00 전후**에 오게 한다. 2분이면 플레이어가 3기억 빌드를 한 번은 체감한 뒤 잃는다.
- **미니보스/보스 구분 필요함**: 미니보스 = 망각 트리거 + 작은 체크포인트, 보스 = 큰 체크포인트. 구분이 있어야 "망각의 비중"을 보스 난이도와 분리해 튜닝할 수 있다.
- 보스 간격 선형 증가(2:00→2:30→3:00) 채택.

### 기억 성능 재설계 — "예산 바닥선" 도입
- 기준 단위: **단일 대상 10초 기준 DPS-equiv = 100**을 1예산으로 잡는다.
- **모든 기억은 자기 이상적 상황에서 90~110% 예산을 채워야 한다.** 단, 분배는 다르게:
  - 처형자(폭딜): 낮은 가동률 + 높은 순간딜 → 평균이 예산.
  - 굶주린(광역): 대상당 낮음, 적 3+에서 예산 달성.
  - 추적자(원거리 유도): 안정 중딜, 포지셔닝 비용 0.
  - 파쇄/멈춘 초침(제어): 여기가 핵심 문제. **"예산 바닥선 70%"** 규칙 — 어떤 기억도 순수 딜 기여가 예산의 70% 밑으로 못 내려간다. 제어 유틸은 **남는 30% 한도 안에서만** 지불한다. (지금 격차의 원인 = 제어 기억이 딜 거의 0이라 잃든 안 잃든 체감이 극단적)
- **역할 변경 권고 2개**:
  - **멈춘 초침**: 순수 둔화 → "둔화장 + 둔화된 적 피해 증가" 하이브리드로. 순수 유틸은 단독으로 쓸 이유가 안 생긴다.
  - **피의 반사**: 수동 반사는 지루하고 측정도 안 됨 → "흡혈 + 추가타" 능동형으로. 반사 수치는 빼고 그 예산을 추가타로 옮긴다.

### 시너지 시스템 — v0.8에 넣되 최소 3개만
빌드 재미 부재가 사용자 1순위 불만이므로 **넣는다**. 단 7태그 전부 시너지화하지 말고:
- **태그 5개로 축소**: 폭딜 / 광역 / 지속 / 제어 / 생존. (투사체·근접은 무기 서술자로만, 시너지 키로는 미사용)
- **먼저 구현할 시너지 3개** (제어를 연결 조직으로 깔고 보스축 1개):
  1. 광역 + 제어 → 둔화/넉백된 적에게 광역 피해 증가
  2. 지속 + 제어 → 느려진 적 출혈 tick 증가
  3. 폭딜 + 생존 → 보스에 큰 피해 시 짧은 보호막
- **UI**: 빌드 표시 옆 작은 아이콘 행으로 활성 시너지 노출, 발동 시 짧은 글로우 + 텍스트 라벨. 프로토타입은 텍스트+아이콘이면 충분.

### 잔향 시스템 — 태그 기반 + 순(純)손실 유지
- 태그 기반 전환 OK. 잃은 기억의 **주(主)태그가 무기 또는 남은 기억에 각인**되어 빌드 방향이 바뀐다.
- **보상 수준**: 잃은 기억 예산의 **최대 35%만** 다른 축으로 환원. 총 전투력은 반드시 줄어든다 → "잃어서 약해진 게 맞다, 다만 방향이 바뀐다."
- **보상처럼 안 느껴지게 할 제약**:
  - 새 슬롯/새 기억 금지 — 기존 무기·기억에 각인만.
  - 잔향이 **기존에 의존하던 시너지를 깨뜨릴 수 있게** 한다. (이게 "후회" 레버 — 망각이 빌드를 재구성하게 강제)

### 테스트 모델 — 게이트(먼저) + 본 측정(같이)
- **게이트(먼저, 필수)**: HP ≤ 0 실제 사망 처리, 그리고 6지표 기록 — 사망률 / 평균 사망 시각 / 첫 사망 구간 / 처치 공백 시간 / 화면 적 누적 / 저HP 압박 시간.
- **AI proxy 보강(재설계와 병행)**: 적 누적·접촉 피해·힐러 회복·공간 압박 반영. 투사체는 최소한 "노출/피격 확률"이라도 모델링.
- **Browser combat QA**: flow 확인만 하지 말고 일정 시간 실제 자동 전투 후 위험도 측정.

## 테스트 기준

- **AI proxy**: 사망률 **30~70%** 구간(죽을 수 있되 클리어도 가능). 첫 사망이 90초 이전에 몰리지 않음. 기억 6종 단독 클리어율 **최고–최저 편차 < 30%p** (예산 재설계 검증). 결손 구간 즉사율이 비결손 대비 과도하지 않음.
- **Browser combat QA**: 자동 전투 중 저HP 압박 시간 비율과 처치 공백이 임계 내. boss→forgetting→deficit→refill 흐름이 실제 사망 가능 상태에서도 완주.
- **User solo test**: 초반 1:30이 루즈하지 않은가, 망각이 "불합리"가 아니라 "후회+재구성"으로 읽히는가, 3기억 선택이 조합으로 느껴지는가.
