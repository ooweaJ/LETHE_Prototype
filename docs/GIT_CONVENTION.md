# Git Convention

이 프로젝트는 Conventional Commits 형식을 사용한다.

## Commit Message Format

```text
type: summary
```

예:

```text
feat: 기억 자동 발동 구현
fix: 보스 처치 후 질문 UI 오류 수정
docs: 오늘 개발 보고서 갱신
test: AI 알파테스트 시나리오 추가
chore: gitignore 정리
```

## Types

- `feat`: 사용자나 테스트 흐름에 영향을 주는 기능 추가
- `fix`: 버그 수정
- `docs`: 문서, 보고서, 개발일지 변경
- `test`: 테스트 코드, 테스트 데이터, 테스트 스크립트 변경
- `refactor`: 동작 변화 없는 구조 개선
- `chore`: 빌드, 설정, 저장소 관리
- `style`: CSS 또는 포맷 중심 변경

## Commit Timing

커밋은 “작업을 설명할 수 있는 최소 단위”로 한다.

좋은 커밋 타이밍:

- 기능 하나가 동작하고 검증됐을 때
- 테스트 도구 하나가 추가되고 실행 확인됐을 때
- 보고서/개발일지처럼 독립적인 문서 단위가 완성됐을 때
- 설정 변경이 다른 작업과 분리 가능할 때

피해야 할 커밋:

- 동작하지 않는 중간 상태
- 여러 목적이 섞인 큰 커밋
- 테스트하지 않은 구현 커밋
- 재생성 가능한 대용량 출력물 커밋

## Push Timing

push는 로컬 커밋이 다음 조건을 만족할 때 한다.

- `git status`가 의도한 상태다.
- 필요한 테스트를 실행했다.
- 커밋 메시지가 작업 목적을 설명한다.
- 원격에 공유해도 되는 단위다.

이 프로젝트에서는 기능 구현, 테스트 도구 변경, 보고서 갱신처럼 명확한 작업 단위가 끝날 때마다 push한다.
