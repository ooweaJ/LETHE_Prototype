# Codex-GPT 개발 워크플로우

이 프로젝트는 Codex를 구현/검증 담당으로, GPT를 기획 검토/우선순위 판단 담당으로 분리해 운영한다.

## 역할 분리

### Codex

- 코드 구현
- 테스트 실행
- AI 알파테스트 실행
- Git 커밋/push
- 현재 상태 보고서 갱신
- 구현 리스크와 기술 부채 기록

### GPT

- 기획 방향 검토
- 다음 작업 우선순위 결정
- 플레이테스트 결과 해석
- 망각/잔향/전투 감정 루프 판단
- 기능 추가 여부 판단

## 기본 루프

1. Codex가 구현한다.
2. Codex가 테스트와 AI 알파테스트를 실행한다.
3. Codex가 `docs/CODEX_STATUS.md`와 개발일지를 갱신한다.
4. 사용자가 `docs/GPT_REVIEW_PROMPT.md`와 `docs/CODEX_STATUS.md`를 GPT에 전달한다.
5. GPT가 다음 작업과 기획 판단을 정리한다.
6. 사용자가 GPT 답변을 `docs/NEXT_TASKS.md` 또는 Codex 대화에 붙여넣는다.
7. Codex가 다음 구현을 진행한다.

## 기록 원칙

- 하루 기록은 짧아야 한다.
- 기능 기록은 구현 결과보다 판단 근거를 남긴다.
- AI 테스트 수치는 원본 리포트 경로와 핵심 값만 남긴다.
- 사람 테스트 결과와 봇 프록시 결과를 섞지 않는다.
- 나중에 개발일지로 재가공할 수 있도록 날짜, 결정, 근거, 다음 행동을 유지한다.
